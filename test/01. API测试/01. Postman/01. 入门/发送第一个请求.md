Postman 使你能够创建并发送 API 请求。你可以向端点发送请求、从数据源检索数据或测试 API 的功能。你无需在终端中输入命令或编写任何代码。创建一个新请求并选择发送，API 的响应将直接显示在 Postman 中。  

# 定义 API 请求
  API 提供了一种结构化的方式，使一个应用程序能够访问另一个应用程序的功能。通常，这种通信是通过互联网上的 API 服务器进行的。客户端应用程序（如移动应用程序）向服务器发送请求，服务器在处理请求后将响应返回给客户端。  

  一个请求包括 API 端点的 URL 和一个 HTTP 请求方法。方法指示你希望 API 执行的操作。以下是一些常见的方法：  
  - GET：从 API 中检索数据。
  - POST：向 API 发送新数据。
  - PATCH和PUT：更新现有数据。
  - DELETE：删除现有数据。

# 发送请求
  1. 在工作空间中选择+以打开一个新的 选项卡 。
  2. 在请求 URL 中输入postman-echo.com/get。
  3. 选择发送。
  Postman 将显示从服务器发送的响应数据在下方的面板中。

# 发生了什么
  在这个示例中，Postman 充当客户端应用程序，与一个 API 服务器进行通信。当你选择发送时，发生了以下情况：  
  1. Postman 向位于postman-echo.com的 Postman Echo API 服务器发送了一个GET请求。
  2. API 服务器接收到请求，处理它，并向 Postman 返回一个响应。
  3. Postman 接收到响应，并在响应面板中显示它。
