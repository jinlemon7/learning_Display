# 组件的概念与优势
  组件是 Vue 应用的核‌‌‌‌‌‌心构建块，它们是可复用的 Vue 实例，带有自⁢⁢⁢⁢⁢⁢定义的名称。在一个大型应用中，将整个应用拆分为‏‏‏‏‏‏多个组件是必不可少的，这样可以使我们的开发过程‍‍‍‍‍‍更加高效和有条理      

  组件化开发的主要优势包括：  
  - 代码复用性：一旦创建了一个组件，它就可以在应用的不同部分甚至不同项目中重复使用，大大提高了开发效率。
  - 关注点分离：将一个复杂的应用拆分为多个组件，每个组件专注于自己的功能，使代码更易于理解和维护.
  - 团队协作：不同的开发者可以同时开发不同的组件，提高团队的工作效率。
  - 测试友好：组件化使得单元测试更加容易，可以独立测试每个组件的功能。

# 组件的注册与使用
  在 Vue‌‌‌‌‌‌3 中，我⁢⁢们⁢可⁢以⁢通⁢过‏‏两种‏方式‏注册‏‍‍组件‏‍：全局‍注⁡⁡册和局‍‍⁡部注册⁡。  

## 全局注册
  全局注册的组件可以在应用的任何地方使用，无需额外导入。使用 app.component() 方法进行全局注册：  
  ```javascript
  // main.js
  import { createApp } from 'vue'
  import App from './App.vue'
  
  const app = createApp(App)
  
  // 注册一个全局组件
  app.component('MyButton', {
    template: `
      <button class="my-button">
        <slot></slot>
      </button>
    `,
    // 组件选项
  })
  
  app.mount('#app')
  ```

## 局部注册
  局部注册的‌‌‌‌‌‌组件只能在注⁢册⁢它⁢的⁢父组件中⁢⁢‏‏使用。‏在单‏‍文件组‍件中，‏‏‍我⁡们通‍常⁡使用局部⁡注册： ‍‍   
  ```vue
  <script>
  import MyButton from './components/MyButton.vue'
  
  export default {
    components: {
      MyButton
    }
  }
  </script>
  <template>
    <div>
      <MyButton>点击我</MyButton>
    </div>
  </template>
  ```

  在 Vue3 的组合式 API 中，使用 <script setup> 可以更简洁地实现局部注册：  
  ```vue
  <script setup>
  import MyButton from './components/MyButton.vue'
  // 无需显式注册，导入的组件可直接在模板中使用
  </script>
  <template>
    <div>
      <MyButton>点击我</MyButton>
    </div>
  </template>
  ```

# Props 属性传递
  Props 是父组件向子组件传递数据的主要方式。在 Vue3 中，我们可以使用 defineProps 宏来声明 props：  
  ```vue
  <!-- ChildComponent.vue -->
  <script setup>
  const props = defineProps({
    title: String,
    likes: {
      type: Number,
      default: 0
    },
    isPublished: Boolean,
    commentIds: Array,
    author: Object
  })
  
  console.log(props.title) // 访问 props
  </script>
  <template>
    <h1>{{ title }}</h1>
    <p>点赞数: {{ likes }}</p>
  </template>
  ```

  在父组件中‌‌使用：  
  ```vue
  <!-- ParentComponent.vue -->
  <script setup>
  import ChildComponent from './ChildComponent.vue'
  </script>
  <template>
    <ChildComponent
      title="Vue3 组件教程"
      :likes="42"
      :is-published="true"
      :comment-ids="[234, 266, 273]"
      :author="{ name: '张三', bio: '前端开发者' }"
    />
  </template>
  ```

# 组件事件与自定义事件
  子组件可以通过触发事件的方式向父组件传递信息。在 Vue3 的组合式 API 中，我们使用 defineEmits 宏来声明组件可以触发的事件：  
  ```vue
  <!-- ChildComponent.vue -->
  <script setup>
  const emit = defineEmits(['update', 'delete'])
  
  function submitForm() {
    // 做一些处理...
    emit('update', { id: 1, data: 'new data' })
  }
  
  function deleteItem() {
    emit('delete', 1) // 传递 ID 为 1 的项
  }
  </script>
  <template>
    <div>
      <button @click="submitForm">更新</button>
      <button @click="deleteItem">删除</button>
    </div>
  </template>
  ```

  在父组件中‌监听‌‌‌这些事件：  
  ```vue
  <!-- ParentComponent.vue -->
  <script setup>
  import ChildComponent from './ChildComponent.vue'
  
  function handleUpdate(payload) {
    console.log('更新项目', payload)
  }
  
  function handleDelete(id) {
    console.log('删除项目', id)
  }
  </script>
  <template>
    <ChildComponent
      @update="handleUpdate"
      @delete="handleDelete"
    />
  </template>
  ```

# 组件插槽的使用
  插槽（Sl‌‌‌‌‌o‌ts）允许父组件⁢⁢⁢⁢⁢向子组件传递内容⁢，‏‏‏‏‏使组件更加灵活。V‍‍‍‍‍u‏e3 支持默认插⁡⁡⁡⁡⁡槽、具名插槽和作‍用域插槽  

## 默认插槽
  ```vue
  <!-- MyCard.vue -->
  <template>
    <div class="card">
      <div class="card-header">
        <h3>{{ title }}</h3>
      </div>
      <div class="card-body">
        <slot>默认内容，当没有提供内容时显示</slot>
      </div>
    </div>
  </template>

  <script setup>
  defineProps({
    title: String
  })
  </script>
  ```

  使用默认插槽：  
  ```vue
  <MyCard title="公告">
    <p>这是插入到卡片中的内容</p>
  </MyCard>
  ```

## 具名插槽
  当需要在组‌‌‌件中‌‌‌定义多个插槽位置⁢⁢⁢时，可以使用具名插⁢⁢⁢槽:  
  ```vue
  <!-- BaseLayout.vue -->
  <template>
    <div class="container">
      <header>
        <slot name="header"></slot>
      </header>
      <main>
        <slot></slot>
      </main>
      <footer>
        <slot name="footer"></slot>
      </footer>
    </div>
  </template>
  ```

  使用具名插‌‌‌‌‌槽：  
  ```vue
  <BaseLayout>
    <template #header>
      <h1>网站标题</h1>
    </template>
    <template #default>
      <p>主要内容区域</p>
    </template>
    <template #footer>
      <p>版权信息 © 2023</p>
    </template>
  </BaseLayout>
  ```

## 作用域插槽
  作用域插槽‌允许‌‌‌‌子组件向插槽内⁢容传递数据：   
  ```vue
  <!-- TodoList.vue -->
  <template>
    <ul>
      <li v-for="item in items" :key="item.id">
        <slot :item="item" :index="index">
          {{ item.text }}
        </slot>
      </li>
    </ul>
  </template>

  <script setup>
  defineProps({
    items: Array
  })
  </script>
  ```

  使用作用域‌‌‌‌‌‌插槽：   
  ```vue
  <TodoList :items="todos">
    <template #default="{ item, index }">
      <span>{{ index + 1 }}. {{ item.text }}</span>
      <button @click="removeTodo(item.id)">删除</button>
    </template>
  </TodoList>
  ```

# 动态组件与异步组件
## 动态组件
  动态组件允许我们根据条件动态切换不同的组件。Vue 提供了 <component> 元素和特殊的 is 属性来实现这一功能：  
  ```vue
  <script setup>
  import { ref } from 'vue'
  import TabHome from './TabHome.vue'
  import TabPosts from './TabPosts.vue'
  import TabArchive from './TabArchive.vue'
  
  const currentTab = ref('Home')
  const tabs = {
    Home: TabHome,
    Posts: TabPosts,
    Archive: TabArchive
  }
  </script>

  <template>
    <div class="demo">
      <button
        v-for="(_, tab) in tabs"
        :key="tab"
        :class="['tab-button', { active: currentTab === tab }]"
        @click="currentTab = tab"
      >
        {{ tab }}
      </button>
      <component :is="tabs[currentTab]" class="tab"></component>
    </div>
  </template>
  ```

## 异步组件
  对于大型应用，我们可能需要将应用拆分为更小的代码块，并且只在需要时才加载组件。Vue 提供了 defineAsyncComponent 函数来实现这一点：  
  ```vue
  <script setup>
  import { defineAsyncComponent } from 'vue'
  
  const AsyncComp = defineAsyncComponent(() =>
    import('./components/HeavyComponent.vue')
  )
  </script>
  <template>
    <AsyncComp />
  </template>
  ```

  你也可以添‌加加载和错误状态：⁢     
  ```javascript
  const AsyncComp = defineAsyncComponent({
    loader: () => import('./components/HeavyComponent.vue'),
    loadingComponent: LoadingComponent,
    errorComponent: ErrorComponent,
    delay: 200,           // 展示加载组件前的延迟时间，默认为 200ms
    timeout: 3000,        // 超时时间，默认是无穷大
    suspensible: false,   // 控制组件是否可挂起，默认为 false
    onError(error, retry, fail, attempts) {
      if (error.message.match(/fetch/) && attempts <= 3) {
        // 在发生 fetch 错误时重试，最多重试 3 次
        retry()
      } else {
        // 注意，retry/fail 就像 promise 的 resolve/reject 一样：
        // 必须调用其中一个才能继续错误处理。
        fail()
      }
    }
  })
  ```

# 练习题
## 练习 1：创建一个简单的按钮组件
  创建一个可复‌‌用的按钮组件，支持自定⁢⁢义文本、颜色和点击‏‏事件。  

  答案：  
  ```vue
  <!-- MyButton.vue -->
  <script setup>
  defineProps({
    text: {
      type: String,
      default: '按钮'
    },
    color: {
      type: String,
      default: 'primary'
    }
  })
  
  const emit = defineEmits(['click'])
  
  function handleClick() {
    emit('click')
  }
  </script>
  <template>
    <button 
      :class="[`btn-${color}`]"
      @click="handleClick"
    >
      {{ text }}
    </button>
  </template>
  <style scoped>
  button {
    padding: 8px 16px;
    border: none;
    border-radius: 4px;
    cursor: pointer;
  }
  .btn-primary {
    background-color: #3498db;
    color: white;
  }
  .btn-success {
    background-color: #2ecc71;
    color: white;
  }
  .btn-danger {
    background-color: #e74c3c;
    color: white;
  }
  </style>
  ```
