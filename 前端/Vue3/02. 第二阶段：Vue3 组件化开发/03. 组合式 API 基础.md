在本章中，你将学习 Vu‌‌‌e3‌ 最重要的新特性——组合式 API（Com⁢⁢⁢position ⁢API）的基础知识。通过组合‏‏‏式 API，你可以更灵活地组织组‏件逻辑，提高代‍‍‍码的可复用性和可维护性。掌握这些知识后，你能够‍⁡⁡⁡编写出更加清晰、模块化的 Vue 组件      

# 组合式 API 介绍与优势
  组合式 API ‌‌‌‌是 Vue3 引入的一种新的编⁢⁢⁢⁢写组件的方式，它与 Vue2 ‏‏‏‏中的选项式 API（Optio‍‍‍‍ns API）并存，但提供了更⁡⁡⁡⁡好的逻辑组织和代码复用能力。  

## 什么是组合式 API
  在 Vue‌2 ‌‌‌中，我们使用选⁢项式 API 来⁢⁢⁢组‏织组件代码，将数据‍、方法、‏‏‏计算属性等⁡分散在不同的选项中：‍‍‍   
  ```javascript
  export default {
    data() {
      return {
        user: { name: '张三', age: 30 },
        searchQuery: ''
      }
    },
    computed: {
      fullName() {
        return this.user.firstName + ' ' + this.user.lastName
      }
    },
    methods: {
      updateUser() {
        // 更新用户信息
      },
      search() {
        // 搜索逻辑
      }
    },
    mounted() {
      // 组件挂载后执行
    }
  }
  ```

  而组合式 AP‌‌‌I ‌允许我们按照逻辑关注点来⁢⁢⁢组织代码，将相关的⁢数据、方法‏‏‏和生命周期钩子放在一起：  
  ```javascript
  import { ref, computed, onMounted } from 'vue'
  
  export default {
    setup() {
      // 用户相关逻辑
      const user = ref({ name: '张三', age: 30 })
      const fullName = computed(() => {
        return user.value.firstName + ' ' + user.value.lastName
      })
      function updateUser() {
        // 更新用户信息
      }
      
      // 搜索相关逻辑
      const searchQuery = ref('')
      function search() {
        // 搜索逻辑
      }
      
      // 生命周期钩子
      onMounted(() => {
        // 组件挂载后执行
      })
      
      // 返回需要暴露给模板的内容
      return {
        user,
        fullName,
        updateUser,
        searchQuery,
        search
      }
    }
  }
  ```

## 组合式 API 的优势  
  1. 更好的逻辑组织：可以将相关的代码放在一起，而不是分散在不同的选项中，使代码更容易理解和维护。
  2. 更强的代码复用：可以将逻辑提取到独立的函数中（组合式函数），然后在多个组件中复用。
  3. 更好的类型推导：对 TypeScript 的支持更好，提供更完善的类型推导。
  4. 更小的打包体积：通过 tree-shaking，只包含实际使用的 API，减小打包体积。

# setup 函数
  setup 函数是组合式 API 的入口点，它在组件实例创建之前执行，因此在 setup 中没有 this。  

## setup 函数的基本使用
  ```javascript
  import { ref } from 'vue'
  
  export default {
    setup() {
      // 声明响应式状态
      const count = ref(0)
      
      // 声明方法
      function increment() {
        count.value++
      }
      
      // 返回值会暴露给模板和其他选项
      return {
        count,
        increment
      }
    }
  }
  ```

## setup 函数的参数
  setup 函数接收两个参数：  
  1. props：组件接收的 props
  2. context：一个包含特定属性的上下文对象

  ```javascript
  export default {
    props: {
      title: String
    },
    setup(props, context) {
      // props 是响应式的，不能解构
      console.log(props.title)
      
      // context 包含三个属性
      const { attrs, slots, emit } = context
      
      // attrs 是非响应式的，包含没有在 props 中声明的属性
      console.log(attrs.class)
      
      // slots 是一个函数，用于访问插槽
      console.log(slots.default)
      
      // emit 用于触发事件
      function handleClick() {
        emit('click', 'some data')
      }
      
      return {
        handleClick
      }
    }
  }
  ```

## setup 语法糖
  Vue3.2 引入了 <script setup> 语法糖，它是在单文件组件 (SFC) 中使用组合式 API 的编译时语法糖，使代码更加简洁：  
  ```vue
  <script setup>
  import { ref } from 'vue'
  
  // 声明响应式状态
  const count = ref(0)
  
  // 声明方法
  function increment() {
    count.value++
  }
  
  // 无需返回，所有顶层变量和导入都会自动暴露给模板
  </script>
  <template>
    <button @click="increment">{{ count }}</button>
  </template>
  ```

  使用 <script setup> 的优势：  
  - 更少的样板代码，更简洁的语法
  - 能够使用纯 TypeScript 声明 props 和 emits
  - 更好的运行时性能
  - 更好的 IDE 类型推导性能

# 响应式 API (ref, reactive)
  Vue3 的响‌应式系‌统是基‌于 ES6 的 ⁢Proxy 实现的，⁢提供了更‏强大和灵⁢活的响应式能力。  

## ref
  ref 用于创建一个包含单一值的响应式引用：  
  ```javascript
  import { ref } from 'vue'
  
  const count = ref(0)
  console.log(count.value) // 0
  
  count.value++
  console.log(count.value) // 1
  ```

  ref 可以包装任何类型的值，包括对象和数组：  
  ```javascript
  const user = ref({
    name: '张三',
    age: 30
  })
  
  console.log(user.value.name) // 张三
  user.value.age = 31
  ```

  在模板中使用 ref 时，不需要使用 .value，Vue 会自动解包：  
  ```vue
  <template>
    <div>
      <p>Count: {{ count }}</p>
      <p>User name: {{ user.name }}</p>
    </div>
  </template>
  ```

## reactive
  reactive 用于创建一个响应式对象或数组：  
  ```javascript
  import { reactive } from 'vue'
  
  const state = reactive({
    count: 0,
    user: {
      name: '张三',
      age: 30
    }
  })
  
  console.log(state.count) // 0
  state.count++
  console.log(state.count) // 1
  
  console.log(state.user.name) // 张三
  state.user.age = 31
  ```

  与 ref 不同，reactive 只能用于对象类型（对象、数组和集合类型如 Map、Set），不能用于基本类型如字符串、数字或布尔值。另外，使用 reactive 时不需要使用 .value。  

## ref vs reactive
  两者的主要区别：  
  1. 类型支持：
     - ref 可以包装任何类型的值
     - reactive 只能用于对象类型
  2. 访问方式：
     - ref 需要使用 .value 访问值（在模板中会自动解包）  
     - 问值（在模板中会自动解包）
  3. 解构和传递：
     - ref 可以单独传递和解构，保持响应性
     - reactive 对象解构后会失去响应性
    
  ```javascript
  // 使用 ref
  const user = ref({
    name: '张三',
    age: 30
  })
  
  // 可以保持响应性地传递
  function updateUser(userRef) {
    userRef.value.age = 31
  }
  updateUser(user)
  
  // 使用 reactive
  const state = reactive({
    user: {
      name: '张三',
      age: 30
    }
  })
  
  // 解构后会失去响应性
  const { user } = state
  user.age = 31 // 不会触发更新
  
  // 正确的做法是使用计算属性或直接访问
  const userAge = computed(() => state.user.age)
  // 或者
  function updateAge() {
    state.user.age = 31 // 会触发更新
  }
  ```

## toRef 和 toRefs
  toRef 和 toRefs 用于将 reactive 对象的属性转换为 ref，使其可以被解构而保持响应性：  
  ```javascript
  import { reactive, toRef, toRefs } from 'vue'
  
  const state = reactive({
    name: '张三',
    age: 30
  })
  
  // 转换单个属性
  const nameRef = toRef(state, 'name')
  nameRef.value = '李四' // state.name 也会变成 '李四'
  
  // 转换所有属性
  const { name, age } = toRefs(state)
  name.value = '王五' // state.name 也会变成 '王五'
  age.value = 31 // state.age 也会变成 31
  ```

# 生命周期钩子
  Vue3 的组合式 API 提供了一系列生命周期钩子函数，它们与 Vue2 中的生命周期钩子对应，但需要在 setup 函数中使用。  

## 生命周期钩子对应关系
  | Vue2 选项式 API | Vue3 组合式 API |
  |----------------|----------------|
  | beforeCreate | setup() 函数本身 |
  | created | setup() 函数本身 |
  | beforeMount | onBeforeMount |
  | mounted | onMounted |
  | beforeUpdate | onBeforeUpdate |
  | updated | onUpdated |
  | beforeDestroy | onBeforeUnmount |
  | destroyed | onUnmounted |
  | errorCaptured | onErrorCaptured |

  Vue3 ‌还新增了两个调试钩⁢子：  
  - onRenderTracked：当组件渲染过程中追踪到响应式依赖时调用
  - onRenderTriggered：当响应式依赖触发组件重新渲染时调用

## 使用生命周期钩子
  ```javascript
  import { ref, onMounted, onBeforeUnmount } from 'vue'
  
  export default {
    setup() {
      const timer = ref(null)
      const count = ref(0)
      
      onMounted(() => {
        console.log('组件已挂载')
        timer.value = setInterval(() => {
          count.value++
        }, 1000)
      })
      
      onBeforeUnmount(() => {
        console.log('组件即将卸载')
        clearInterval(timer.value)
      })
      
      return {
        count
      }
    }
  }
  ```

  使用 <script setup> 语法糖：  
  ```vue
  <script setup>
  import { ref, onMounted, onBeforeUnmount } from 'vue'
  
  const timer = ref(null)
  const count = ref(0)
  
  onMounted(() => {
    console.log('组件已挂载')
    timer.value = setInterval(() => {
      count.value++
    }, 1000)
  })
  
  onBeforeUnmount(() => {
    console.log('组件即将卸载')
    clearInterval(timer.value)
  })
  </script>
  <template>
    <div>计数器：{{ count }}</div>
  </template>
  ```

## 在组合式函数中使用生命周期钩子
  生命周期钩‌‌‌子可以在组合式⁢函⁢数⁢中使用，这‏使得‏我们‏可以将‍相关逻‍辑封装‍到⁡可复用的⁡函数中：  
  ```javascript
  import { ref, onMounted, onBeforeUnmount } from 'vue'
  
  export function useCounter() {
    const count = ref(0)
    const timer = ref(null)
    
    function startCounter() {
      timer.value = setInterval(() => {
        count.value++
      }, 1000)
    }
    
    function stopCounter() {
      clearInterval(timer.value)
    }
    
    onMounted(() => {
      startCounter()
    })
    
    onBeforeUnmount(() => {
      stopCounter()
    })
    
    return {
      count,
      startCounter,
      stopCounter
    }
  }
  ```

  在组件中使‌‌用：  
  ```vue
  <script setup>
  import { useCounter } from './composables/useCounter'
  
  const { count, startCounter, stopCounter } = useCounter()
  </script>

  <template>
    <div>
      <p>计数器：{{ count }}</p>
      <button @click="startCounter">开始</button>
      <button @click="stopCounter">停止</button>
    </div>
  </template>
  ```

## 练习 2：创建一个使用 reactive 和 ref 的用户信息表单
  答案：  
  ```vue
  <script setup>
  import { reactive, ref, computed } from 'vue'
  
  const formData = reactive({
    name: '',
    email: '',
    age: ''
  })
  
  const errors = reactive({
    name: '',
    email: '',
    age: ''
  })
  
  const isSubmitting = ref(false)
  const isFormValid = computed(() => {
    return formData.name && formData.email && formData.age && 
           !errors.name && !errors.email && !errors.age
  })
  
  function validateName() {
    if (!formData.name) {
      errors.name = '姓名不能为空'
    } else if (formData.name.length < 2) {
      errors.name = '姓名至少需要 2 个字符'
    } else {
      errors.name = ''
    }
  }
  
  function validateEmail() {
    const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/
    if (!formData.email) {
      errors.email = '邮箱不能为空'
    } else if (!emailRegex.test(formData.email)) {
      errors.email = '请输入有效的邮箱地址'
    } else {
      errors.email = ''
    }
  }
  
  function validateAge() {
    const age = parseInt(formData.age)
    if (!formData.age) {
      errors.age = '年龄不能为空'
    } else if (isNaN(age) || age <= 0) {
      errors.age = '请输入有效的年龄'
    } else if (age < 18) {
      errors.age = '年龄必须大于或等于 18'
    } else {
      errors.age = ''
    }
  }
  
  async function submitForm() {
    validateName()
    validateEmail()
    validateAge()
    
    if (isFormValid.value) {
      isSubmitting.value = true
      try {
        // 模拟 API 请求
        await new Promise(resolve => setTimeout(resolve, 1500))
        alert(`表单提交成功！\n姓名: ${formData.name}\n邮箱: ${formData.email}\n年龄: ${formData.age}`)
        
        // 重置表单
        formData.name = ''
        formData.email = ''
        formData.age = ''
      } catch (error) {
        alert('提交失败: ' + error.message)
      } finally {
        isSubmitting.value = false
      }
    }
  }
  </script>
  <template>
    <div class="form-container">
      <h2>用户信息表单</h2>
      <form @submit.prevent="submitForm">
        <div class="form-group">
          <label for="name">姓名</label>
          <input 
            id="name" 
            v-model="formData.name" 
            @blur="validateName"
            :class="{ 'is-invalid': errors.name }"
          />
          <div v-if="errors.name" class="error-message">{{ errors.name }}</div>
        </div>
        
        <div class="form-group">
          <label for="email">邮箱</label>
          <input 
            id="email" 
            type="email" 
            v-model="formData.email" 
            @blur="validateEmail"
            :class="{ 'is-invalid': errors.email }"
          />
          <div v-if="errors.email" class="error-message">{{ errors.email }}</div>
        </div>
        
        <div class="form-group">
          <label for="age">年龄</label>
          <input 
            id="age" 
            type="number" 
            v-model="formData.age" 
            @blur="validateAge"
            :class="{ 'is-invalid': errors.age }"
          />
          <div v-if="errors.age" class="error-message">{{ errors.age }}</div>
        </div>
        
        <button type="submit" :disabled="!isFormValid || isSubmitting">
          {{ isSubmitting ? '提交中...' : '提交' }}
        </button>
      </form>
    </div>
  </template>
  ```

## 练习 3：创建一个使用生命周期钩子的组件
  创建一个组‌‌‌‌件，在挂载时从 ⁢A⁢PI⁢⁢ 获取数据‏，并‏在卸载前‏‏保‍存状态。    

  答案：  
  ```vue
  <script setup>
  import { ref, reactive, onMounted, onBeforeUnmount } from 'vue'
  
  const posts = ref([])
  const isLoading = ref(true)
  const error = ref(null)
  
  // 模拟本地存储键
  const STORAGE_KEY = 'user-scroll-position'
  
  // 获取帖子数据
  async function fetchPosts() {
    isLoading.value = true
    error.value = null
    
    try {
      // 模拟 API 请求
      const response = await new Promise(resolve => {
        setTimeout(() => {
          resolve({
            ok: true,
            json: () => Promise.resolve([
              { id: 1, title: 'Vue3 组合式 API 简介', content: '组合式 API 是 Vue3 的新特性...' },
              { id: 2, title: '使用 ref 和 reactive', content: '在 Vue3 中，我们可以使用 ref 和 reactive...' },
              { id: 3, title: '生命周期钩子', content: 'Vue3 提供了一系列生命周期钩子...' },
              { id: 4, title: '组合式函数', content: '组合式函数是一种复用有状态逻辑的方式...' },
              { id: 5, title: 'Vue3 性能优化', content: 'Vue3 相比 Vue2 有显著的性能提升...' }
            ])
          })
        }, 1000)
      })
      
      if (response.ok) {
        posts.value = await response.json()
      } else {
        throw new Error('Failed to fetch posts')
      }
    } catch (err) {
      error.value = err.message || '获取数据失败'
    } finally {
      isLoading.value = false
    }
  }
  
  // 保存滚动位置
  function saveScrollPosition() {
    const scrollPosition = window.scrollY || document.documentElement.scrollTop
    localStorage.setItem(STORAGE_KEY, scrollPosition.toString())
    console.log('保存滚动位置:', scrollPosition)
  }
  
  // 恢复滚动位置
  function restoreScrollPosition() {
    const savedPosition = localStorage.getItem(STORAGE_KEY)
    if (savedPosition) {
      window.scrollTo(0, parseInt(savedPosition))
      console.log('恢复滚动位置:', savedPosition)
    }
  }
  
  onMounted(() => {
    console.log('组件已挂载，开始获取数据')
    fetchPosts()
    restoreScrollPosition()
    
    // 添加滚动事件监听器
    window.addEventListener('scroll', saveScrollPosition)
  })
  
  onBeforeUnmount(() => {
    console.log('组件即将卸载，保存状态')
    saveScrollPosition()
    
    // 移除滚动事件监听器
    window.removeEventListener('scroll', saveScrollPosition)
  })
  </script>

  <template>
    <div class="posts-container">
      <h2>博客文章列表</h2>
      
      <div v-if="isLoading" class="loading">
        加载中...
      </div>
      
      <div v-else-if="error" class="error">
        错误: {{ error }}
        <button @click="fetchPosts">重试</button>
      </div>
      
      <div v-else>
        <article v-for="post in posts" :key="post.id" class="post">
          <h3>{{ post.title }}</h3>
          <p>{{ post.content }}</p>
        </article>
        
        <div v-if="posts.length === 0" class="no-posts">
          暂无文章
        </div>
      </div>
    </div>
  </template>

  <style scoped>
  .posts-container {
    max-width: 800px;
    margin: 0 auto;
    padding: 20px;
  }
  
  .loading, .error, .no-posts {
    text-align: center;
    padding: 20px;
    margin: 20px 0;
    border-radius: 8px;
  }
  
  .loading {
  ```
