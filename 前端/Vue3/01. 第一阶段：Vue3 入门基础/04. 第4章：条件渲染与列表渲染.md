# 条件渲染指令 v-if、v-else 和 v-show
  条件渲染是‌构建‌‌‌‌‌‌‌动态 UI 的⁢基础，Vue ⁢⁢⁢⁢⁢⁢提供‏了几种不⁢同的指令来‍根据‏‏‏‏‏‏条件控制元素的显⁡示与隐藏。  

## v-if 指令
  v-if 指令用于条件性地渲染一块内容。这块内容只会在指令的表达式返回真值时被渲染：  
  ```vue
  <script setup>
  import { ref } from 'vue'
  
  const isVisible = ref(true)
  
  function toggleVisibility() {
    isVisible.value = !isVisible.value
  }
  </script>
  <template>
    <button @click="toggleVisibility">
      {{ isVisible ? '隐藏' : '显示' }}
    </button>
    
    <p v-if="isVisible">现在你能看到我</p>
  </template>
  ```

  当 isVisible 为 false 时，包含 v-if 指令的元素会从 DOM 中移除，而不仅仅是隐藏。  

## v-else 和 v-else-if 指令
  v-else 和 v-else-if 指令可以配合 v-if 使用，实现更复杂的条件渲染：  
  ```vue
  <script setup>
  import { ref } from 'vue'
  
  const type = ref('A')
  </script>
  <template>
    <div>
      <div v-if="type === 'A'">
        类型是 A
      </div>
      <div v-else-if="type === 'B'">
        类型是 B
      </div>
      <div v-else-if="type === 'C'">
        类型是 C
      </div>
      <div v-else>
        不是 A/B/C 中的任何一种
      </div>
    </div>
  </template>
  ```

## v-show 指令
  另一个用于条件性展示元素的指令是 v-show：  
  ```vue
  <script setup>
  import { ref } from 'vue'
  
  const isVisible = ref(true)
  </script>
  <template>
    <p v-show="isVisible">使用 v-show 控制显示</p>
  </template>
  ```

  与 v-if 不同的是，v-show 只是简单地切换元素的 CSS display 属性。元素始终会被渲染并保留在 DOM 中，只是视觉上的显示和隐藏。  

## v-if vs v-show
  如果你需要切换多个元素，又不想引入额外的包装 div，可以使用 <template> 元素搭配 v-if：  
  ```vue
  <template>
    <template v-if="isVisible">
      <h1>标题</h1>
      <p>第一段</p>
      <p>第二段</p>
    </template>
  </template>
  ```

  <template> 标签不会在最终的 DOM 中渲染，它只作为包装多个元素的不可见容器。   

# 列表渲染指令 v-for
## 基本用法
  v-for 指令用于基于数组或对象渲染列表：  
  ```vue
  <script setup>
  import { ref } from 'vue'
  
  const items = ref([
    { id: 1, text: '学习 JavaScript' },
    { id: 2, text: '学习 Vue' },
    { id: 3, text: '构建一个项目' }
  ])
  </script>
  <template>
    <ul>
      <li v-for="item in items" :key="item.id">
        {{ item.text }}
      </li>
    </ul>
  </template>
  ```

  v-for 也可以接受第二个参数，即当前项的索引：  
  ```vue
  <ul>
    <li v-for="(item, index) in items" :key="item.id">
      {{ index }} - {{ item.text }}
    </li>
  </ul>
  ```

## 使用 key 属性
  当使用 v-for 时，建议始终提供一个 key 属性，除非遍历输出的 DOM 内容非常简单。key 帮助 Vue 识别节点，从而高效地更新 DOM：  
  ```vue
  <div v-for="item in items" :key="item.id">
    <!-- 内容 -->
  </div>
  ```

  选择 ke‌‌‌‌‌‌‌y 的原则： ⁢⁢⁢⁢⁢ ⁢ ⁢   
  - 使用唯一的标识符，如 id
  - 避免使用索引(index)作为 key（除非你确定列表不会重新排序或过滤）
  - 避免使用随机数或随机字符串

## 遍历对象
  你也可以使用 v-for 来遍历对象的属性：  
  ```vue
  <script setup>
  import { reactive } from 'vue'
  
  const userInfo = reactive({
    name: '张三',
    age: 30,
    occupation: '工程师'
  })
  </script>
  <template>
    <ul>
      <li v-for="(value, key, index) in userInfo" :key="key">
        {{ index }}. {{ key }}: {{ value }}
      </li>
    </ul>
  </template>
  ```

  在遍历对象时，顺序是根据 Object.keys() 的枚举顺序确定的。  

## 使用范围值
  v-for 也可以接受整数作为范围值：  
  ```vue
  <template>
    <ul>
      <li v-for="n in 5" :key="n">{{ n }}</li>
    </ul>
  </template>
  ```

  这会渲染 ‌‌‌‌‌1 到‌ 5 ‌‌的列⁢⁢⁢⁢表  

## 在 template 上使用 v-for
  类似于 v-if，你也可以在 <template> 标签上使用 v-for 来渲染一组元素，而无需引入额外的包装元素：  
  ```vue
  <template>
    <template v-for="item in items" :key="item.id">
      <div>{{ item.title }}</div>
      <p>{{ item.description }}</p>
    </template>
  </template>
  ```

#  数组更新检测
## Vue 对数组变更方法的响应式处理  
  Vue 能‌够‌检测到‌调用响‌‌‌‌‌应式⁢数组的变更⁢方法时的‏变化，并⁢触发视‍图‏更新。这⁢⁢⁢⁢⁢些方法包括：   
  - push()
  - pop()
  - shift()
  - unshift()
  - splice()
  - sort()
  - reverse()

  ```vue
  <script setup>
  import { ref } from 'vue'
  
  const items = ref(['苹果', '香蕉', '橙子'])
  
  function addItem() {
    items.value.push('葡萄')  // 视图会自动更新
  }
  
  function removeItem() {
    items.value.pop()  // 视图会自动更新
  }
  
  function sortItems() {
    items.value.sort()  // 视图会自动更新
  }
  </script>
  <template>
    <ul>
      <li v-for="(item, index) in items" :key="index">{{ item }}</li>
    </ul>
    
    <div>
      <button @click="addItem">添加水果</button>
      <button @click="removeItem">移除水果</button>
      <button @click="sortItems">排序水果</button>
    </div>
  </template>
  ```

## 使用非变更方法
  不修改原数组的方法（如 filter()、concat() 和 slice()）不会触发视图更新，需要用返回的新数组替换旧数组：  
  ```vue
  <script setup>
  import { ref } from 'vue'
  
  const numbers = ref([1, 2, 3, 4, 5])
  
  function filterEvenNumbers() {
    // 创建新数组并赋值给 numbers.value
    numbers.value = numbers.value.filter(n => n % 2 === 0)
  }
  </script>
  <template>
    <ul>
      <li v-for="n in numbers" :key="n">{{ n }}</li>
    </ul>
    
    <button @click="filterEvenNumbers">只显示偶数</button>
  </template>
  ```

# 列表渲染中的 key 属性
## key 的作用
  key 是 Vue 识别节点的一种方式，特别是在列表渲染中。当列表更新时，Vue 会优先复用相同 key 的元素，这让更新更加高效。  
  ```vue
  <script setup>
  import { ref } from 'vue'
  
  const users = ref([
    { id: 1, name: '张三' },
    { id: 2, name: '李四' },
    { id: 3, name: '王五' }
  ])
  </script>
  <template>
    <div v-for="user in users" :key="user.id">
      <!-- 内容 -->
    </div>
  </template>
  ```

  没有 key 或使用不当的情况可能导致以下问题：  
  - 无法正确识别组件实例
  - 表单状态错误复用
  - 动画问题
  - 子组件无法正确更新

## 使用 index 作为 key 的问题
  在某些情况下，使用索引作为 key 可能导致意外行为，特别是当列表会重新排序或有项目被插入/删除时：  
  ```vue
  <!-- 不推荐的做法 -->
  <div v-for="(item, index) in items" :key="index">
    {{ item.text }}
  </div>
  ```

  问题实例：当列‌‌表改‌变时‌‌，索引‌‌‌与特定项目的⁢⁢关联也会改变，导⁢致 Vue‏‏ ⁢⁢无法正确追踪每个项目⁢⁢⁢，可‍‏‍能引发不必要的 DOM 重‏‏⁡⁡新渲染或组件状‍态错误。    

# 过滤与排序显示
  在 Vue‌‌‌‌‌‌‌‌3 的组合式 AP⁢⁢⁢⁢⁢⁢⁢⁢I 中，我们可以使‏‏‏‏‏‏‏‏用计算属性来展示过‍‍‍‍‍‍‍‍滤或排序后的列表，⁡⁡⁡⁡⁡⁡⁡⁡而不需要改变原始数据：  
  ```vue
  <script setup>
  import { ref, computed } from 'vue'
  
  const items = ref([
    { id: 1, name: '苹果', price: 5 },
    { id: 2, name: '香蕉', price: 3 },
    { id: 3, name: '橙子', price: 4 },
    { id: 4, name: '葡萄', price: 8 }
  ])
  
  const searchQuery = ref('')
  
  // 过滤后的列表
  const filteredItems = computed(() => {
    return items.value.filter(item => 
      item.name.toLowerCase().includes(searchQuery.value.toLowerCase())
    )
  })
  
  // 排序后的列表（按价格从低到高）
  const sortedItems = computed(() => {
    return [...filteredItems.value].sort((a, b) => a.price - b.price)
  })
  </script>
  <template>
    <input v-model="searchQuery" placeholder="搜索水果...">
    
    <h3>过滤后的列表</h3>
    <ul>
      <li v-for="item in filteredItems" :key="item.id">
        {{ item.name }} - ¥{{ item.price }}
      </li>
    </ul>
    
    <h3>排序后的列表（价格从低到高）</h3>
    <ul>
      <li v-for="item in sortedItems" :key="item.id">
        {{ item.name }} - ¥{{ item.price }}
      </li>
    </ul>
  </template>
  ```

  注意，我们使用了计算属性而不是直接在 v-for 表达式中进行过滤和排序，这样能提高效率，避免每次重新渲染时都执行这些操作。  

# 实用示例：动态任务列表
  下面是一个结合‌‌条件渲‌‌染和‌‌列表‌渲染的‌实用示例⁢⁢ - 一个任务列表，⁢⁢用户可以‏‏添加⁢⁢、完成和过滤任务⁢：  
  ```vue
  <script setup>
  import { ref, computed } from 'vue'
  
  // 响应式状态
  const newTask = ref('')
  const tasks = ref([
    { id: 1, text: '学习 Vue3 基础', completed: false },
    { id: 2, text: '练习组合式 API', completed: true },
    { id: 3, text: '构建一个小项目', completed: false }
  ])
  let nextTaskId = 4
  
  // 过滤类型
  const filterType = ref('all') // 'all', 'active', 'completed'
  
  // 过滤后的任务列表
  const filteredTasks = computed(() => {
    switch (filterType.value) {
      case 'active':
        return tasks.value.filter(task => !task.completed)
      case 'completed':
        return tasks.value.filter(task => task.completed)
      default:
        return tasks.value
    }
  })
  
  // 统计信息
  const statistics = computed(() => {
    const total = tasks.value.length
    const completed = tasks.value.filter(task => task.completed).length
    const active = total - completed
    return { total, completed, active }
  })
  
  // 方法
  function addTask() {
    if (newTask.value.trim()) {
      tasks.value.push({
        id: nextTaskId++,
        text: newTask.value,
        completed: false
      })
      newTask.value = ''
    }
  }
  
  function toggleTask(task) {
    task.completed = !task.completed
  }
  
  function removeTask(taskId) {
    tasks.value = tasks.value.filter(task => task.id !== taskId)
  }
  
  function clearCompleted() {
    tasks.value = tasks.value.filter(task => !task.completed)
  }
  </script>
  <template>
    <div class="task-app">
      <h1>任务列表</h1>
      
      <!-- 添加新任务 -->
      <div class="task-form">
        <input 
          v-model="newTask" 
          placeholder="添加新任务..." 
          @keyup.enter="addTask"
        >
        <button @click="addTask">添加</button>
      </div>
      
      <!-- 过滤选项 -->
      <div class="task-filters">
        <button 
          :class="{ active: filterType === 'all' }" 
          @click="filterType = 'all'"
        >
          所有
        </button>
        <button 
          :class="{ active: filterType === 'active' }" 
          @click="filterType = 'active'"
        >
          未完成
        </button>
        <button 
          :class="{ active: filterType === 'completed' }" 
          @click="filterType = 'completed'"
        >
          已完成
        </button>
      </div>
      
      <!-- 任务列表 -->
      <div v-if="filteredTasks.length > 0" class="task-list">
        <div 
          v-for="task in filteredTasks" 
          :key="task.id" 
          class="task-item"
          :class="{ completed: task.completed }"
        >
          <input 
            type="checkbox" 
            :checked="task.completed"
            @change="toggleTask(task)"
          >
          <span class="task-text">{{ task.text }}</span>
          <button @click="removeTask(task.id)" class="delete-btn">
            删除
          </button>
        </div>
      </div>
      <div v-else class="empty-message">
        <p>没有符合条件的任务</p>
      </div>
      
      <!-- 底部工具栏 -->
      <div class="task-footer">
        <span>{{ statistics.active }} 个待办, {{ statistics.completed }} 个已完成</span>
        <button 
          v-if="statistics.completed > 0" 
          @click="clearCompleted"
        >
          清除已完成
        </button>
      </div>
    </div>
  </template>
  <style scoped>
  .task-app {
    max-width: 500px;
    margin: 0 auto;
    padding: 20px;
    font-family: sans-serif;
  }
  
  .task-form {
    display: flex;
    margin-bottom: 20px;
  }
  
  .task-form input {
    flex: 1;
    padding: 10px;
    font-size: 16px;
    border: 1px solid #ddd;
    border-radius: 4px 0 0 4px;
  }
  
  .task-form button {
    padding: 10px 15px;
    font-size: 16px;
    background-color: #42b983;
    color: white;
    border: none;
    border-radius: 0 4px 4px 0;
    cursor: pointer;
  }
  
  .task-filters {
    display: flex;
    margin-bottom: 15px;
  }
  
  .task-filters button {
    flex: 1;
    padding: 8px;
    background-color: #f5f5f5;
    border: 1px solid #ddd;
    cursor: pointer;
  }
  
  .task-filters button.active {
    background-color: #42b983;
    color: white;
  }
  
  .task-list {
    margin-bottom: 20px;
  }
  
  .task-item {
    display: flex;
    align-items: center;
    padding: 10px;
    border-bottom: 1px solid #eee;
  }
  
  .task-item.completed .task-text {
    text-decoration: line-through;
    color: #999;
  }
  
  .task-text {
    flex: 1;
    margin: 0 10px;
  }
  
  .delete-btn {
    padding: 3px 8px;
    background-color: #ff6b6b;
    color: white;
    border: none;
    border-radius: 3px;
    cursor: pointer;
  }
  
  .task-footer {
    display: flex;

    padding-top: 15px;
    border-top: 1px solid #eee;
  }
  
  .empty-message {
    text-align: center;
    color: #999;
    padding: 20px 0;
  }
  </style>
  ```

## 练习 2：列表渲染 - 创建一个可排序的表格
  创建一个可排序的表格组件，可以根据不同的列进行排序。  
  答案：  
  ```vue
  <script setup>
  import { ref, computed } from 'vue'
  
  const users = ref([
    { id: 1, name: '张三', age: 28, occupation: '开发工程师' },
    { id: 2, name: '李四', age: 34, occupation: '产品经理' },
    { id: 3, name: '王五', age: 22, occupation: '设计师' },
    { id: 4, name: '赵六', age: 31, occupation: '数据分析师' },
    { id: 5, name: '钱七', age: 26, occupation: '市场专员' }
  ])
  
  const sortBy = ref('id')
  const sortDirection = ref('asc') // 'asc' 或 'desc'
  
  // 排序后的用户列表
  const sortedUsers = computed(() => {
    const direction = sortDirection.value === 'asc' ? 1 : -1
    
    return [...users.value].sort((a, b) => {
      if (a[sortBy.value] < b[sortBy.value]) return -1 * direction
      if (a[sortBy.value] > b[sortBy.value]) return 1 * direction
      return 0
    })
  })
  
  // 排序方法
  function sort(column) {
    if (sortBy.value === column) {
      // 如果已经按这列排序，则切换方向
      sortDirection.value = sortDirection.value === 'asc' ? 'desc' : 'asc'
    } else {
      // 否则，按新列排序，默认升序
      sortBy.value = column
      sortDirection.value = 'asc'
    }
  }
  
  // 获取排序图标
  function getSortIcon(column) {
    if (sortBy.value !== column) return '⇅'
    return sortDirection.value === 'asc' ? '↑' : '↓'
  }
  </script>
  <template>
    <div class="sortable-table">
      <table>
        <thead>
          <tr>
            <th @click="sort('id')">
              ID {{ getSortIcon('id') }}
            </th>
            <th @click="sort('name')">
              姓名 {{ getSortIcon('name') }}
            </th>
            <th @click="sort('age')">
              年龄 {{ getSortIcon('age') }}
            </th>
            <th @click="sort('occupation')">
              职业 {{ getSortIcon('occupation') }}
            </th>
          </tr>
        </thead>
        <tbody>
          <tr v-for="user in sortedUsers" :key="user.id">
            <td>{{ user.id }}</td>
            <td>{{ user.name }}</td>
            <td>{{ user.age }}</td>
            <td>{{ user.occupation }}</td>
          </tr>
        </tbody>
      </table>
    </div>
  </template>
  <style scoped>
  .sortable-table {
    max-width: 700px;
    margin: 0 auto;
    font-family: sans-serif;
  }
  
  table {
    width: 100%;
    border-collapse: collapse;
  }
  
  th, td {
    padding: 10px;
    text-align: left;
    border-bottom: 1px solid #ddd;
  }
  
  th {
    background-color: #f5f5f5;
    cursor: pointer;
  }
  
  th:hover {
    background-color: #e5e5e5;
  }
  
  tr:hover {
    background-color: #f9f9f9;
  }
  ```
