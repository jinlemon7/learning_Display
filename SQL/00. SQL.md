# SQL 知识体系
## SQL 分类
  1. DDL（Data Definition Language）数据定义语言
     用于定义和管理数据库对象（数据库、表、索引、视图等）。
     - CREATE：创建数据库对象
     ```sql
      -- 创建数据库
      CREATE DATABASE school;
      
      -- 创建表
      CREATE TABLE student (
          id INT PRIMARY KEY,
          name VARCHAR(50),
          age INT
      );
     ```

     - ALTER：修改数据库对象
     ```sql
      -- 添加列
      ALTER TABLE student ADD COLUMN email VARCHAR(100);
      
      -- 修改列类型
      ALTER TABLE student MODIFY COLUMN age TINYINT;
     ```

     - DROP：删除数据库对象
     ```java
      -- 删除表
      DROP TABLE student;
      
      -- 删除数据库
      DROP DATABASE school;
     ```

     - TRUNCATE：清空表数据（保留表结构）
     `TRUNCATE TABLE student;`

  2. DML（Data Manipulation Language）数据操作语言
     - INSERT：插入数据
     ```sql
      -- 插入单条数据
      INSERT INTO student (id, name, age) VALUES (1, '张三', 20);
      
      -- 插入多条数据
      INSERT INTO student VALUES 
          (2, '李四', 21),
          (3, '王五', 22);
     ```

     - UPDATE：更新数据
     ```sql
      -- 更新单个字段
      UPDATE student SET age = 21 WHERE id = 1;
      
      -- 更新多个字段
      UPDATE student SET name = '赵六', age = 23 WHERE id = 2;
     ```

     - DELETE：删除数据
     ```sql
      -- 删除指定数据
      DELETE FROM student WHERE id = 1;
      
      -- 删除所有数据（慎用！）
      DELETE FROM student;
     ```

  3. DQL（Data Query Language）数据查询语言
     - SELECT：查询数据
     ```sql
      -- 查询所有列
      SELECT * FROM student;
      
      -- 查询指定列
      SELECT name, age FROM student;
      
      -- 条件查询
      SELECT * FROM student WHERE age > 20;
      
      -- 排序查询
      SELECT * FROM student ORDER BY age DESC;
      
      -- 分组查询
      SELECT class, COUNT(*) as count FROM student GROUP BY class;
      
      -- 多表查询
      SELECT s.name, c.course_name 
      FROM student s 
      JOIN course c ON s.id = c.student_id;
     ```

  4. DCL（Data Control Language）数据控制语言
     - GRANT：授予权限
     ```sql
      -- 授予用户查询权限
      GRANT SELECT ON school.* TO 'user1'@'localhost';
      
      -- 授予所有权限
      GRANT ALL PRIVILEGES ON school.* TO 'admin'@'localhost';
     ```

     - REVOKE：撤销权限
     ```sql
      -- 撤销查询权限
      REVOKE SELECT ON school.* FROM 'user1'@'localhost';
     ```

  5. TCL（Transaction Control Language）事务控制语言
     - BEGIN/START TRANSACTION：开始事务
     - COMMIT：提交事务
     - ROLLBACK：回滚事务
     - SAVEPOINT：设置保存点
     ```sql
      -- 开始事务
      START TRANSACTION;
      
      -- 执行操作
      UPDATE account SET balance = balance - 100 WHERE id = 1;
      UPDATE account SET balance = balance + 100 WHERE id = 2;
      
      -- 提交事务（如果成功）
      COMMIT;
      
      -- 或回滚事务（如果失败）
      -- ROLLBACK;
     ```

# SQL 方言
  主流数据库‌及其方言：    
  1. MySQL：最流行的开源数据库，语法相对简单，适合入门
  2. PostgreSQL：功能强大的开源数据库，支持复杂查询和扩展
  3. Oracle：企业级商业数据库，功能完善，使用 PL/SQL
  4. SQL Server：微软的商业数据库，使用 T-SQL
  5. SQLite：轻量级嵌入式数据库，常用于移动端和小型应用

  方言差异例子：  
  分页查询：  
  - MySQL: LIMIT 10 OFFSET 0
  - PostgreSQL: LIMIT 10 OFFSET 0
  - Oracle: ROWNUM <= 10 或 FETCH FIRST 10 ROWS ONLY
  - SQL Server: TOP 10 或 OFFSET 0 ROWS FETCH NEXT 10 ROWS ONLY

  字符串拼接：
  - MySQL: CONCAT(str1, str2) 或 str1 || str2
  - PostgreSQL: str1 || str2
  - Oracle: str1 || str2
  - SQL Server: str1 + str2 或 CONCAT(str1, str2)

# SQL 进阶查询
  1. 聚合函数
     - COUNT()：计数
     - SUM()：求和
     - AVG()：平均值
     - MAX()：最大值
     - MIN()：最小值
     - GROUP BY：分组
     - HAVING：分组后的条件筛选
     ```sql
      -- 统计每个班级的学生人数
      SELECT class, COUNT(*) as student_count 
      FROM student 
      GROUP BY class;
      
      -- 查询平均成绩大于 80 分的班级
      SELECT class, AVG(score) as avg_score 
      FROM student 
      GROUP BY class 
      HAVING AVG(score) > 80;
     ```

  2. 多表查询
     - 内连接：INNER JOIN
     - 左外连接：LEFT JOIN
     - 右外连接：RIGHT JOIN
     - 全外连接：FULL OUTER JOIN（MySQL 不支持，可用 UNION 实现）
     - 交叉连接：CROSS JOIN
     - 自连接：表和自己连接
     ```sql
      -- 内连接：查询学生及其选课信息
      SELECT s.name, c.course_name 
      FROM student s 
      INNER JOIN course c ON s.id = c.student_id;
      
      -- 左连接：查询所有学生及其选课信息（包括没选课的学生）
      SELECT s.name, c.course_name 
      FROM student s 
      LEFT JOIN course c ON s.id = c.student_id;
     ```

  3. 子查询
     - 单行子查询
     - 多行子查询：IN、ANY、ALL
     - 相关子查询
     - EXISTS 和 NOT EXISTS
     ```sql
      -- 查询成绩高于平均成绩的学生
      SELECT * FROM student 
      WHERE score > (SELECT AVG(score) FROM student);
      
      -- 查询选了课程的学生
      SELECT * FROM student 
      WHERE id IN (SELECT DISTINCT student_id FROM course);
     ```

  4. 高级查询技巧
     - 联合查询：UNION、UNION ALL
     - 条件表达式：CASE WHEN
     - 窗口函数（排名、累计等）
       - ROW_NUMBER()：行号
       - RANK()：排名
       - DENSE_RANK()：密集排名
       - PARTITION BY：分区
     - 公共表表达式（CTE）：WITH ... AS
     ```sql
      -- CASE WHEN 示例
      SELECT name, 
          CASE 
              WHEN score >= 90 THEN '优秀'
              WHEN score >= 80 THEN '良好'
              WHEN score >= 60 THEN '及格'
              ELSE '不及格'
          END as grade
      FROM student;
      
      -- 窗口函数示例：查询每个班级成绩排名前 3 的学生
      SELECT * FROM (
          SELECT name, class, score,
              ROW_NUMBER() OVER (PARTITION BY class ORDER BY score DESC) as rank
          FROM student
      ) as ranked
      WHERE rank <= 3;
     ```

  5. 视图和索引
     - 视图：CREATE VIEW、DROP VIEW
     - 索引：CREATE INDEX、DROP INDEX
     - 索引类型：主键索引、唯一索引、普通索引、全文索引

# SQL 性能优化
  1. 查询优化基础
     - 使用 EXPLAIN 分析查询计划
     - 理解查询的执行过程
     - 避免全表扫描
     - 减少子查询的使用（能用 JOIN 就用 JOIN）
     - 避免在 WHERE 子句中使用函数或表达式
     - 使用 LIMIT 限制返回结果
    
  2. 索引优化
     - 索引的原理（B+ 树）
     - 何时需要创建索引
     - 索引的优缺点
     - 联合索引和最左前缀原则
     - 覆盖索引
     - 索引失效的场景
    
  3. SQL 编写最佳实践
     - 避免 SELECT *，只查询需要的列
     - 合理使用 EXISTS 代替 IN
     - 使用 UNION ALL 代替 UNION（如果不需要去重）
     - 批量操作代替单条操作
     - 避免使用 OR，考虑使用 IN 或 UNION
    
  1. 性能优‌‌‌‌化是经验积累⁢的⁢过⁢程⁢：不要‏死记‏硬背‏优化‏‍技巧，‍要理解‍背后‍⁡的原⁡理。
  2. 多使用‌‌‌‌ EXPLAIN 分⁢⁢⁢⁢析：建议每写一个复杂‏‏‏‏查询，都用 EXPL‍‍‍‍AIN 看看执行计划⁡⁡⁡⁡，观察是否用到索引。
  3. 索引不‌‌‌‌是越多越好：⁢过⁢多⁢的⁢索引会‏影响‏写入‏性能‏‍，要根‍据实际‍查⁡询需求合‍⁡理创建。
  4. 结合具体‌‌‌‌数据库学习：不同数据库⁢⁢⁢⁢的优化策略可能不同，建‏‏‏‏议结合 MySQL 或‍‍‍‍ PostgreSQL⁡⁡⁡⁡ 的优化专题深入学习。
  5. 关注慢‌‌‌‌查询日志：实⁢际⁢工⁢作⁢中，通‏过分‏析慢‏查询‏‍日志来‍定位性能⁡‍问题‍。
