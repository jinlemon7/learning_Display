# 单字段分组(GROUP BY)
  GROUP BY 子句允可将数据分成多个组，然后对每个组应用聚合函数（如 COUNT(), SUM(), AVG() 等）。  
  <img width="1908" height="2010" alt="image" src="https://github.com/user-attachments/assets/7ed71786-d1ae-4a8c-b238-5e7c8af3e4af" />  

## 基本语法
  ```sql
  SELECT 列1, 列2, ..., 聚合函数(列)
  FROM 表名
  GROUP BY 列1, 列2, ...;
  ```
  - GROUP BY 子句通常与聚合函数一起使用，使我们能够获取每个组的汇总信息。

## 单字段分组示例
  假设我们有一个 orders 表，记录了每天的订单信息：  
  | id | user_id | product_id	| product_name | category	| price	| quantity | order_date |  
  | --- | --- | --- | --- | --- | --- | --- | --- |  
  | 1	| 101 | 1001 | Java 从入门到精通	| 书籍 | 89 | 1 | 2025-05-10 |  
  | 2	| 102	| 2001 | 算法学习笔记 | 电子教程 | 59	| 1	| 2025-05-11 |  
  | 3	| 103	| 1002 | 前端开发实战 | 书籍	| 79 | 2 | 2025-05-12 |  
  | 4	| 101	| 3001 | 程序员定制键盘 | 周边	| 299	| 1	| 2025-05-15 |  
  | 5	| 104	| 1001 | Java 从入门到精通	| 书籍 | 89	| 1	| 2025-05-18 |

  按商品类别统计销售情况：
  ```sql
  SELECT 
      category,
      COUNT(*) AS order_count,
  ```

  按用户统计消费情况：
  ```sql
  SELECT 
      user_id,
      COUNT(*) AS order_count,
      SUM(price * quantity) AS total_spent
  FROM orders
  GROUP BY user_id;
  ```
  结果：  
  | user_id	| order_count	| total_spent |

  | 101	| 2	| 388 |  
  | 102	| 2	| 176 |

## 分组与排序结合
  通常，我们会‌‌‌将分组结果按照某个计算值⁢⁢⁢排序，以找出"最高"或"‏‏‏最低"的组：   
  ```sql
  SELECT 
      category,
      COUNT(*) AS order_count,
      SUM(price * quantity) AS total_revenue
  FROM orders
  GROUP BY category
  ORDER BY total_revenue DESC;
  ```
  结果：  
  | category | order_count |total_revenue |  
  | --- | --- | --- |  
  | 周边 | 2 | 416 |  
  | 书籍 | 3 | 336 |  
  | 电子教程	| 1	| 59 |  
  - 这个查询结‌‌‌果按照总销售额⁢降⁢序⁢排列，这样‏我们‏就可‏以快速‍识别销‍售额最‍高的商⁡品类⁡别。

## 使用 GROUP BY 时的注意事项
  1. SELECT 子句中的列：在使用 GROUP BY 时，SELECT 子句中的列要么包含在 GROUP BY 子句中，要么必须使用聚合函数。这是因为在分组后，每一组会得到一行结果，非分组列的值可能有多个，SQL 不知道该选择哪一个。
  2. 分组顺序：GROUP BY 不保证结果集的顺序。如果需要特定顺序，应该使用 ORDER BY。
  3. NULL 值处理：NULL 值在分组时被视为相同的值，所有 NULL 值会被分为一组。

# 多字段分组
  我们需要更‌‌‌细粒度的分组时，可以使⁢⁢⁢用多字段分组。多字段分‏‏‏组允许我们按照多个列的‍‍‍组合进行分组，这样可以⁡⁡⁡得到更详细的统计结果。  
  <img width="2176" height="2104" alt="image" src="https://github.com/user-attachments/assets/e5f88c84-7d3c-4882-bdc5-a6e92bb897c1" />  

## 基本语法
  ```sql
  SELECT 列1, 列2, ..., 聚合函数(列)
  FROM 表名
  GROUP BY 列1, 列2, ...;
  ```
  - 在多字段分‌‌‌组中，GROU⁢P⁢ ⁢BY 后列‏出的‏所有‏列的值‍组合会‍被视为‍一个唯⁡一的⁡组。

## 多字段分组示例
  继续使用上面的 orders 表：  

  按类别和订单日期统计：
  ```sql
  SELECT 
      category,
      DATE_FORMAT(order_date, '%Y-%m') AS month,
      COUNT(*) AS order_count,
      SUM(price * quantity) AS total_revenue
  FROM orders
  GROUP BY category, DATE_FORMAT(order_date, '%Y-%m');
  ```
  结果：  
  | category | month | order_count | total_revenue |  
  | --- | --- | --- | --- |  
  | 书籍 |2025-05 | 3	| 336 |  
  | 电子教程	| 2025-05	| 1	|59 |  
  | 周边 | 2025-05 | 2 | 416 |  
  - 在这个例子‌‌‌中，我们按商品类⁢别⁢和月份⁢进行分组‏，获‏得了更详细的‏‍时间维‍度统计。

  按用户和商品类别统计消费：
  ```sql
  SELECT 
      user_id,
      category,
      COUNT(*) AS order_count,
      SUM(price * quantity) AS total_spent
  FROM orders
  GROUP BY user_id, category;
  ```
  结果：  
  | user_id | category | order_count | total_spent |  
  |---------|----------|-------------|-------------|  
  | 101     | 书籍     | 1           | 89          |  
  | 101     | 周边     | 1           | 299         |  
  | 102     | 电子教程 | 1           | 59          |  
  | 102     | 周边     | 1           | 117         |  
  | 103     | 书籍     | 1           | 158         |  
  | 104     | 书籍     | 1           | 89          |  
  - 这样就可以查出每个用户在每个商品类别上的消费情况。

## 多字段分组的应用场景
  多字段分组在以下场景特别有用：  
  1. 多维度分析：例如，按地区、产品类别、时间段等多个维度分析销售数据。
  2. 交叉表：创建类似于透视表的结果，展示不同维度之间的关系。
  3. 时间序列分析：按时间段和其他维度分组，观察数据随时间的变化趋势。

  <img width="2184" height="320" alt="image" src="https://github.com/user-attachments/assets/c83c62ea-3c33-4e8c-88ef-74ab4242a657" />

  例如，我们可以生成按用户和月份的消费明细：  
  ```sql
  SELECT 
      user_id,
      DATE_FORMAT(order_date, '%Y-%m') AS month,
      COUNT(*) AS order_count,
      SUM(price * quantity) AS total_spent
  FROM orders
  GROUP BY user_id, DATE_FORMAT(order_date, '%Y-%m');
  ```
  - 这种分组方‌‌‌式可以帮助我们⁢跟⁢踪⁢用户消费行‏为的‏变化‏，识别消‍‍费模式。

# 分组过滤(HAVING)
  在 SQL 中‌，WH‌‌ERE 子句用于在分组⁢前过滤行，而 HA⁢⁢VING ‏子句则用于在分组后过滤结果。‍‏‏当我们需要基于聚合值进行筛选⁡时，就需要使‍‍用 HAVING  
  <img width="1840" height="196" alt="image" src="https://github.com/user-attachments/assets/287d3187-dd64-4363-91ef-00174d735f14" />  

## 基本语法
  ```sql
  SELECT 列1, 列2, ..., 聚合函数(列)
  FROM 表名
  GROUP BY 列1, 列2, ...
  HAVING 条件;
  ```
  - HAVING 子‌句必须‌‌放在 GROUP BY ⁢子句之后，它可以包含⁢⁢聚合函数，‏而 WHERE 子句不能。

## HAVING 子句示例
  这里继续使用 orders 表做演示：  

  找出总销售额超过 200 的类别：  
  ```sql
  SELECT 
      category,
      SUM(price * quantity) AS total_revenue
  FROM orders
  GROUP BY category
  HAVING total_revenue > 200;
  ```
  结果：  
  | category | total_revenue |  
  |----------|---------------|  
  | 书籍     | 336           |  
  | 周边     | 416           |  

  找出平均订单金额超过 100 且订单数大于 1 的用户：  
  ```sql
  SELECT 
      user_id,
      COUNT(*) AS order_count,
      AVG(price * quantity) AS avg_order_value
  FROM orders
  GROUP BY user_id
  HAVING order_count > 1 AND avg_order_value > 100;
  ```
  结果：  
  | user_id | order_count | avg_order_value |  
  |---------|-------------|-----------------|  
  | 101     | 2           | 194.0           |  

## WHERE 与 HAVING 的区别
  虽然 WH‌‌‌ERE 和 HA⁢V⁢ING⁢ 都用于‏过滤‏数据，但它们‏‍有很大‍的区别：     ‍ 
  1. 执行时机：
     - WHERE 在分组前过滤行
     - HAVING 在分组后过滤结果
  2. 聚合函数：
     - WHERE 不能包含聚合函数
     - HAVING 可以使用聚合函数
  3. 性能：
     - WHERE 通常比 HAVING 更高效，因为它减少了需要处理的行数

  例如，下面两个查询会产生相同的结果，但性能可能不同：  
  ```sql
  -- 使用 HAVING（先分组，再过滤）
  SELECT category, COUNT(*) AS order_count
  FROM orders
  GROUP BY category
  HAVING category = '书籍';
  
  -- 使用 WHERE（先过滤，再分组）
  SELECT category, COUNT(*) AS order_count
  FROM orders
  WHERE category = '书籍'
  GROUP BY category;
  ```
  - 第二个查询‌‌‌（使用 WHE⁢R⁢E⁢）通常更高‏效，‏因为‏它在分‍组前就‍筛选掉‍了不相⁡关的⁡行。

# 完整的查询顺序
   一个完整的‌‌‌ SQL 查询⁢（⁢包⁢含分组）包‏含以‏下子‏句，它‍们的执‍行顺序‍如⁡下：  
   1. FROM：指定要查询的表
   2. WHERE：在分组前过滤数据
   3. GROUP BY：对数据进行分组
   4. HAVING：过滤分组后的结果
   5. SELECT：选择要显示的列或表达式
   6. ORDER BY：对结果进行排序
   7. LIMIT：限制返回的行数

   <img width="2216" height="212" alt="image" src="https://github.com/user-attachments/assets/cbdb4ac0-96df-4256-974d-c88ffbaed150" />  

   下面是一个完整的示例：  
   ```sql
  SELECT 
      category, 
      COUNT(*) AS order_count, 
      SUM(price * quantity) AS total_revenue
  FROM orders
  WHERE order_date >= '2025-05-10'
  GROUP BY category
  HAVING total_revenue > 100
  ORDER BY total_revenue DESC
  LIMIT 2;
  ```
  - 这个查询首先‌‌‌过滤出特定日期之后的订单⁢⁢⁢，然后按类别分组，再过滤‏‏‏出总销售额超过 100 ‍‍‍的类别，最后按总销售额降⁡⁡⁡序排列并只返回前 2 个结果。

# 练习题

## 练习 1：单字段分组应用
  假设有一个 website_traffic 表，记录了网站各页面的访问情况，包含 page_url, user_id, visit_time, duration_seconds（停留时长）等字段。请编写一个 SQL 查询，按照页面 URL 分组，计算每个页面的访问次数、平均停留时长和最长停留时长。  
  答案：  
  ```sql
  SELECT 
      page_url,
      COUNT(*) AS visit_count,
      AVG(duration_seconds) AS avg_duration,
      MAX(duration_seconds) AS max_duration
  FROM website_traffic
  GROUP BY page_url;
  ```

## 练习 2：多字段分组应用
  使用练习 1 中的 website_traffic 表，按照页面 URL 和日期（从 visit_time 中提取）进行分组，统计每个页面每天的访问次数、平均停留时长，并按日期降序、访问次数降序排序。  
  答案：  
  ```sql
  SELECT 
      page_url,
      DATE(visit_time) AS visit_date,
      COUNT(*) AS visit_count,
      AVG(duration_seconds) AS avg_duration
  FROM website_traffic
  GROUP BY page_url, DATE(visit_time)
  ORDER BY visit_date DESC, visit_count DESC;
  ```

## 练习 3：HAVING 子句应用
  继续使用 website_traffic 表，找出平均停留时长超过 2 分钟（120 秒）且访问次数大于 100 次的热门页面，按平均停留时长降序排列。  
  答案：  
  ```sql
  SELECT 
      page_url,
      COUNT(*) AS visit_count,
      AVG(duration_seconds) AS avg_duration
  FROM website_traffic
  GROUP BY page_url
  HAVING avg_duration > 120 AND visit_count > 100
  ORDER BY avg_duration DESC;
  ```
