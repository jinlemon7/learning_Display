# 创建表(CREATE TABLE)
  CREATE TABLE 语句用于在数据库中创建新表。  

  基本语法：  
  ```sql
  CREATE TABLE 表名 (
      列名1 数据类型1 [约束],
      列名2 数据类型2 [约束],
      ...,
      [表级约束]
  );
  ```

## 常见数据类型
  不同的数据‌‌‌‌库系统支持不同⁢的⁢数⁢据类型，⁢以‏下是‏几个‏常见数‍据库系‍‏统中的‍主⁡要数据类⁡型：   ‍ 

### MySQL 数据类型
  数字类型：  
  - INT：整数（-2^31 到 2^31-1）
  - BIGINT：大整数（-2^63 到 2^63-1）
  - FLOAT：单精度浮点数
  - DOUBLE：双精度浮点数
  - DECIMAL(M,D)：定点数，M 是总位数，D 是小数位数

  字符串类型：  
  - CHAR(n)：固定长度字符串，最多 255 字符
  - VARCHAR(n)：可变长度字符串，最多 65,535 字符
  - TEXT：最大长度为 65,535 字符的文本
  - LONGTEXT：最大长度为 4,294,967,295 字符的文本

  日期和时间类型：  
  - DATE：日期（YYYY-MM-DD）
  - TIME：时间（HH:MM:SS）
  - DATETIME：日期和时间（YYYY-MM-DD HH:MM:SS）
  - TIMESTAMP：时间戳

  其他类型：  
  - BOOLEAN：布尔值（TRUE 或 FALSE）
  - BLOB：二进制数据
  - JSON：JSON 格式数据（MySQL 5.7+）

  - CHARACTER VARYING(n), VARCHAR(n)：可变长度字符串

## 列约束
  列约束是对‌‌表中列的限制条件，⁢⁢常见的约束包括：  
  1. NOT NULL：列不能包含 NULL 值
  2. UNIQUE：列的值在表中必须唯一
  3. PRIMARY KEY：列是主键，具有 NOT NULL 和 UNIQUE 特性
  4. FOREIGN KEY：列是外键，引用另一个表的列
  5. CHECK：列的值必须满足特定条件
  6. DEFAULT：未指定值时使用的默认值

## 创建简单的表
  ```sql
  CREATE TABLE users (
      user_id INT PRIMARY KEY AUTO_INCREMENT,
      username VARCHAR(50) NOT NULL UNIQUE,
      email VARCHAR(100) NOT NULL UNIQUE,
      password VARCHAR(255) NOT NULL,
      registration_date DATETIME DEFAULT CURRENT_TIMESTAMP,
      last_login DATETIME,
      status VARCHAR(20) DEFAULT 'active'
  );
  ```
  - 这个语句创建了一个 users 表，包含用户 ID、用户名、电子邮件、密码、注册日期、最后登录时间和状态等字段。

## 表级约束
  表级约束适‌‌用于‌整个表或多个列⁢⁢的组合：  
  ```sql
  CREATE TABLE orders (
      order_id INT AUTO_INCREMENT,
      user_id INT,
      product_id INT,
      quantity INT NOT NULL,
      order_date DATETIME DEFAULT CURRENT_TIMESTAMP,
      PRIMARY KEY (order_id),
      FOREIGN KEY (user_id) REFERENCES users(user_id),
      FOREIGN KEY (product_id) REFERENCES products(product_id),
      CONSTRAINT valid_quantity CHECK (quantity > 0)
  );
  ```
  - 这个语句创建了一个 orders 表，包含订单 ID、用户 ID、产品 ID、数量和订单日期等字段。它使用表级约束定义了主键、外键和检查约束。

## 使用 CREATE TABLE ... AS SELECT
  可以使用 ‌SEL‌‌‌ECT 语句的⁢结果创建一个新表：  
  ```sql
  CREATE TABLE active_users AS
  SELECT user_id, username, email, registration_date
  FROM users
  WHERE status = 'active';
  ```
  - 这个语句根据 users 表中所有活跃用户的数据创建一个新表 active_users。

## 创建临时表
  临时表是会话结束时自动删除的表：  
  ```sql
  CREATE TEMPORARY TABLE user_session_data (
      session_id VARCHAR(100) PRIMARY KEY,
      user_id INT,
      login_time DATETIME,
      data JSON
  );
  ```
  - 临时表可以存储会话期间的临时数据，不会与其他用户的表冲突。

# 修改表结构(ALTER TABLE)
  ALTER TABLE 语句用于修改现有表的结构，如添加、修改或删除列、约束等。  

## 添加列
  ```sql
  ALTER TABLE users
  ADD COLUMN phone_number VARCHAR(20);
  ```
  - 这个语句向 users 表添加了一个名为 phone_number 的新列。

  ```sql
  ALTER TABLE users
  ADD COLUMN birth_date DATE,
  ADD COLUMN address VARCHAR(255);
  ```
  - 这个语句一次向 users 表添加了多个列。

## 修改列
  ```sql
  -- MySQL 语法
  ALTER TABLE users
  MODIFY COLUMN username VARCHAR(100) NOT NULL;
  
  -- PostgreSQL 语法
  ALTER TABLE users
  ALTER COLUMN username TYPE VARCHAR(100),
  ALTER COLUMN username SET NOT NULL;
  ```
  - 这个语句修改了 username 列的数据类型和 NOT NULL 约束。

## 删除列
  ```sql
  ALTER TABLE users
  DROP COLUMN phone_number;
  ```
  - 这个语句从 users 表中删除了 phone_number 列。

## 添加约束
  ```sql
  -- 添加主键约束
  ALTER TABLE products
  ADD PRIMARY KEY (product_id);
  
  -- 添加唯一约束
  ALTER TABLE users
  ADD CONSTRAINT unique_email UNIQUE (email);
  
  -- 添加外键约束
  ALTER TABLE orders
  ADD CONSTRAINT fk_user_id FOREIGN KEY (user_id) REFERENCES users(user_id);
  
  -- 添加检查约束
  ALTER TABLE products
  ADD CONSTRAINT valid_price CHECK (price >= 0);
  ```

## 删除约束
  ```sql
  -- 删除主键约束
  ALTER TABLE products
  DROP PRIMARY KEY;
  
  -- 删除唯一约束
  ALTER TABLE users
  DROP CONSTRAINT unique_email;
  
  -- 删除外键约束
  ALTER TABLE orders
  DROP CONSTRAINT fk_user_id;
  
  -- 删除检查约束
  ALTER TABLE products
  DROP CONSTRAINT valid_price;
  ```

## 重命名表
  ```sql
  -- MySQL 语法
  RENAME TABLE old_table_name TO new_table_name;
  
  -- PostgreSQL 语法
  ALTER TABLE old_table_name RENAME TO new_table_name;
  ```

## 重命名列
  ```sql
  -- MySQL 语法
  ALTER TABLE users
  CHANGE COLUMN old_column_name new_column_name VARCHAR(50);
  
  -- PostgreSQL 语法
  ALTER TABLE users
  RENAME COLUMN old_column_name TO new_column_name;
  ```

## ALTER TABLE 的注意事项
  1. 数据丢失：修改列的数据类型可能导致数据丢失或截断。
  2. 性能影响：对大表进行 ALTER TABLE 操作可能会很耗时，影响系统性能。
  3. 锁定：某些 ALTER TABLE 操作可能会锁定表，阻止其他操作。
  4. 依赖关系：修改被其他对象（如视图或外键）引用的表时需要小心。

# 删除表(DROP TABLE)
  DROP TABLE 语句用于从数据库中删除表定义及其所有数据。  

## 基本语法
  ```sql
  DROP TABLE [IF EXISTS] 表名;
  ```

## 示例
  ```sql
  -- 删除单个表
  DROP TABLE users;
  
  -- 删除多个表
  DROP TABLE users, orders, products;
  
  -- 使用 IF EXISTS 避免错误
  DROP TABLE IF EXISTS temporary_data;
  ```

## 使用事务保护
  在删除表之前，最好使用事务来保护操作：  
  ```sql
  BEGIN TRANSACTION;
  DROP TABLE users;
  -- 如果有错误，可以回滚
  ROLLBACK;
  -- 如果确认无误，提交
  COMMIT;
  ```

## DROP TABLE 的注意事项
  1. 不可逆：DROP TABLE 操作是不可逆的，除非有备份。
  2. 外键约束：如果其他表通过外键引用了要删除的表，DROP TABLE 可能会失败。
  3. 级联删除：某些数据库支持级联删除选项，允许删除表及其相关的外键约束。
  4. 权限：执行 DROP TABLE 需要足够的权限。

# 索引创建与管理
  索引是数据‌‌‌‌库中用于提高查询⁢⁢⁢性⁢能的数据结构。‏‏‏索引‏允许数据库快‍‍‍速找到‍特定列值的⁡⁡⁡行，而不必扫⁡描整个表。  

## 创建索引
  ```sql
  -- 创建单列索引
  CREATE INDEX idx_username ON users(username);
  
  -- 创建复合索引
  CREATE INDEX idx_name_email ON users(last_name, first_name, email);
  
  -- 创建唯一索引
  CREATE UNIQUE INDEX idx_unique_email ON users(email);
  ```

## 删除索引
  ```sql
  -- MySQL 语法
  DROP INDEX idx_username ON users;
  
  -- PostgreSQL 语法
  DROP INDEX idx_username;
  ```

## 索引的类型
  不同的数据库系统支持不同类型的索引：  
  - B-tree 索引：最常见的索引类型，适用于等值查询、范围查询和排序。
  - 哈希索引：适用于等值查询，但不支持范围查询或排序。
  - 全文索引：用于全文搜索。
  - 空间索引：用于地理空间数据。

  <img width="2252" height="364" alt="image" src="https://github.com/user-attachments/assets/9457e2c0-8162-4061-a3aa-82b00f6cc512" />  

## 索引的注意事项
  1. 性能权衡：索引提高查询性能，但可能降低写入性能（插入、更新和删除）。
  2. 选择性：索引列的选择性（唯一值的比例）越高，索引效果越好。
  3. 复合索引顺序：在复合索引中，列的顺序很重要，应该考虑查询模式。
  4. 覆盖索引：如果索引包含查询所需的所有列，数据库可以直接从索引中获取数据，不需要访问表。

# 数据库对象命名和设计规范
  规范的命名和设计可以提高数据库的可维护性和性能。  

## 命名规范
  1. 表名：使用名词复数（如 users, products），或者在单数名词前加前缀（如 tbl_user）。
  2. 列名：使用小写和下划线（如 first_name）或驼峰命名法（如 firstName）。
  3. 主键：通常使用 id 或表名加 _id（如 user_id）。
  4. 外键：使用引用表名加 _id（如 user_id）。
  5. 索引：使用 idx_ 前缀和列名（如 idx_username）。
  6. 约束：使用约束类型和列名（如 pk_users, fk_orders_users）。

  <img width="1432" height="4446" alt="image" src="https://github.com/user-attachments/assets/2ba8c2bd-9498-4da9-9ab1-34f264872c96" />


## 设计规范
  1. 规范化：遵循数据库规范化原则，减少数据冗余。
  2. 适当的数据类型：选择合适的数据类型，如使用 VARCHAR 而非 CHAR 存储可变长度字符串。
  3. NOT NULL 约束：尽可能使用 NOT NULL 约束，避免 NULL 值带来的问题。
  4. 默认值：为经常有默认值的列设置 DEFAULT 约束。
  5. 主键：每个表都应该有一个主键。
  6. 外键：使用外键确保引用完整性，但注意性能影响。
  7. 索引策略：为经常用于查询条件、排序或连接的列创建索引。

## 练习 1：创建表
  编程导航网站需要一个新的表来存储课程的评论。请创建一个 course_comments 表，包含以下字段：  
  - 评论 ID（主键，自增）
  - 用户 ID（外键，引用 users 表）
  - 课程 ID（外键，引用 courses 表）
  - 评论内容（文本）
  - 评分（1-5 的整数）
  - 评论时间（默认为当前时间）
  - 状态（默认为'已发布'）
  - 同时，确保‌‌‌评分值在 1-5 ⁢⁢⁢的范围内。

  答案：  
  ```sql
  CREATE TABLE course_comments (
      comment_id INT PRIMARY KEY AUTO_INCREMENT,
      user_id INT NOT NULL,
      course_id INT NOT NULL,
      content TEXT NOT NULL,
      rating INT NOT NULL,
      comment_time DATETIME DEFAULT CURRENT_TIMESTAMP,
      status VARCHAR(20) DEFAULT '已发布',
      FOREIGN KEY (user_id) REFERENCES users(user_id),
      FOREIGN KEY (course_id) REFERENCES courses(course_id),
      CONSTRAINT valid_rating CHECK (rating BETWEEN 1 AND 5)
  );
  
  -- 为常用查询创建索引
  CREATE INDEX idx_course_comment ON course_comments(course_id, comment_time);
  CREATE INDEX idx_user_comment ON course_comments(user_id, comment_time);
  ```

## 练习 3：数据库设计
  现在需要新增‌一个‌新功能‌，允许用户创⁢建多个简历版本并⁢跟踪应用‏记录。设计⁢必要的表结‍构（‏包括表、列、约束和⁡索引），以支持‏以‍下功能：  
  1. 用户可以创建多个简历版本
  2. 用户可以使用特定的简历版本申请职位
  3. 系统需要跟踪每个申请的状态和时间线

  答案：  
  ```sql
  -- 简历版本表
  CREATE TABLE resume_versions (
      version_id INT PRIMARY KEY AUTO_INCREMENT,
      user_id INT NOT NULL,
      title VARCHAR(100) NOT NULL,
      content TEXT NOT NULL,
      created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
      updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
      is_active BOOLEAN DEFAULT TRUE,
      FOREIGN KEY (user_id) REFERENCES users(user_id),
      INDEX idx_user_versions (user_id, is_active)
  );
  
  -- 职位表
  CREATE TABLE jobs (
      job_id INT PRIMARY KEY AUTO_INCREMENT,
      company_name VARCHAR(100) NOT NULL,
      job_title VARCHAR(100) NOT NULL,
      description TEXT NOT NULL,
      location VARCHAR(100),
      posting_date DATETIME DEFAULT CURRENT_TIMESTAMP,
      deadline_date DATETIME,
      status VARCHAR(20) DEFAULT '开放中',
      INDEX idx_job_search (job_title, company_name, status)
  );
  
  -- 应用记录表
  CREATE TABLE job_applications (
      application_id INT PRIMARY KEY AUTO_INCREMENT,
      user_id INT NOT NULL,
      job_id INT NOT NULL,
      resume_version_id INT NOT NULL,
      applied_at DATETIME DEFAULT CURRENT_TIMESTAMP,
      status VARCHAR(30) DEFAULT '已申请',  -- 已申请、已查看、面试中、已拒绝、已录用
      last_status_change DATETIME DEFAULT CURRENT_TIMESTAMP,
      notes TEXT,
      FOREIGN KEY (user_id) REFERENCES users(user_id),
      FOREIGN KEY (job_id) REFERENCES jobs(job_id),
      FOREIGN KEY (resume_version_id) REFERENCES resume_versions(version_id),
      UNIQUE KEY unique_application (user_id, job_id),  -- 防止重复申请
      INDEX idx_application_status (user_id, status, applied_at)
  );
  
  -- 应用状态更新表（跟踪时间线）
  CREATE TABLE application_status_history (
      history_id INT PRIMARY KEY AUTO_INCREMENT,
      application_id INT NOT NULL,
      previous_status VARCHAR(30),
      new_status VARCHAR(30) NOT NULL,
      changed_at DATETIME DEFAULT CURRENT_TIMESTAMP,
      changed_by INT,  -- 用户ID或系统ID
      notes TEXT,
      FOREIGN KEY (application_id) REFERENCES job_applications(application_id),
      INDEX idx_application_history (application_id, changed_at)
  );
  
  -- 触发器：记录应用状态变更（MySQL语法）
  DELIMITER //
  CREATE TRIGGER trg_application_status_change
  AFTER UPDATE ON job_applications
  FOR EACH ROW
  BEGIN
      IF NEW.status <> OLD.status THEN
          INSERT INTO application_status_history 
              (application_id, previous_status, new_status, changed_by)
          VALUES
              (NEW.application_id, OLD.status, NEW.status, @current_user_id);
      END IF;
  END //
  DELIMITER ;
  ```
