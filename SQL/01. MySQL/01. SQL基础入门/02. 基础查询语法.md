#  全表查询(SELECT * FROM)
  `SELECT * FROM 表名;`  
  - SELECT：告诉数据库我们要检索数据
  - *：星号是一个通配符，表示"所有列"
  - FROM：指定我们要从哪个表中获取数据
  - 表名：要查询的数据表的名称
  <img width="2196" height="600" alt="image" src="https://github.com/user-attachments/assets/20dec7ab-e986-432a-8810-28eebb6dd432" />

  全表查询的‌‌‌‌‌‌优缺点：   
  - 简单直观，容易编写
  - 可以快速查看表中的所有数据
  - 当你不确定表结构时，可以用它来了解表中有哪些列

  缺点：  
  - 对于大型表，返回所有数据会导致查询速度非常缓慢
  - 可能会返回不必要的数据，增加网络传输负担
  - 在生产环境中，一般不推荐使用（除非确实需要所有数据）
  <img width="2240" height="1520" alt="image" src="https://github.com/user-attachments/assets/0bc19d4a-c033-4468-9b27-bc8b30db06fa" />

# 选择查询(SELECT 特定列)
  `SELECT 列名1, 列名2, ... FROM 表名;`  

  选择查询的优点：  
  1. 提高查询效率：只传输和处理必要的数据
  2. 减少网络流量：对于远程数据库，减少了数据传输量
  3. 代码可读性：明确表示了我们需要的数据
  4. 安全性：可以避免返回敏感数据（如密码）
  <img width="2244" height="960" alt="image" src="https://github.com/user-attachments/assets/1e07210a-2348-4e82-8d77-335aacf35a8c" />

# 使用别名(AS)
  `SELECT 列名 AS 别名 FROM 表名;`  
  
  AS 关键字用于指定列的别名。在一些数据库中，AS 关键字甚至可以省略：
  `SELECT 列名 别名 FROM 表名;`  

  别名不仅可‌‌以使‌‌‌‌‌用中文（取决于⁢⁢数据库的字符集支⁢⁢⁢⁢⁢持‏‏），还可以包含空格‍‍，但包‏‏‏‏‏含空格时通常⁡⁡需要用引号括起来：‍‍‍‍‍  
  ```sql
  SELECT question_text AS "问题 内容"
  FROM interview_questions;
  ```

  别名的主要用途:  
  - 提高可读性：使查询结果对非技术人员更友好
  - 处理复杂查询：在复杂查询中可以为中间结果提供清晰的标识
  - 区分同名列：当从多个表中查询相同名称的列时，别名可以避免混淆
  - 支持本地化：可以将英文列名转为本地语言

# 常量和简单运算
## 常量
  `SELECT 'Hello World' AS greeting;`  
  - 这将返回一‌‌‌‌‌‌‌行一列的结果，其⁢⁢⁢⁢⁢⁢中⁢包含文本 "H‏‏‏‏‏‏el‏lo Wor‍‍‍‍‍‍ld"‍，列名为 ⁡⁡⁡⁡⁡⁡"greet⁡ing"。

  ```sql
  SELECT username, 'Active Member' AS membership_type
  FROM users
  WHERE status = 'active';
  ```

  结果：  
  | username | membership_type |  
  |----------|-----------------|  
  | 鱼皮 | Active Member |  
  | 算法达人 | Active Member |  

## 简单运算
  SQL 支持基本的算术运算：加(+)、减(-)、乘(*)、除(/)。  
  <img width="2160" height="1404" alt="image" src="https://github.com/user-attachments/assets/633181af-524f-46cd-95ea-9e02b96850a4" />

  假设有一个 products 表：  
  | id | name | price | discount |  
  |----|------|-------|----------|  
  | 1 | 基础版 | 99.00 | 0.2 |  
  | 2 | 高级版 | 199.00 | 0.1 |  
  | 3 | 专业版 | 299.00 | 0.15 |  
  | 4 | 终身版 | 999.00 | 0.25 |  

  我们可以计‌‌‌‌算每个产品的折后⁢⁢⁢价⁢格：   
  ```sql
  SELECT 
      name, 
      price, 
      discount, 
      price * (1 - discount) AS discounted_price
  FROM products;
  ```

  结果：  
  | name | price | discount | discounted_price |  
  |------|-------|----------|------------------|  
  | 基础版 | 99.00 | 0.2 | 79.20 |  
  | 高级版 | 199.00 | 0.1 | 179.10 |  
  | 专业版 | 299.00 | 0.15 | 254.15 |  
  | 终身版 | 999.00 | 0.25 | 749.25 |  

  除了基本算术外，大多数 SQL 实现还支持各种函数：  
  - 数学函数：ROUND(), CEIL(), FLOOR(), ABS() 等
  - 字符串函数：CONCAT(), SUBSTRING(), UPPER(), LOWER() 等
  - 日期函数：NOW(), DATE(), DATEDIFF() 等

  例如，我们可以对折后价格四舍五入  
  ```sql
  SELECT 
      name, 
      ROUND(price * (1 - discount), 2) AS discounted_price
  FROM products;
  ```

# 结果集排序(ORDER BY)
  <img width="2096" height="440" alt="image" src="https://github.com/user-attachments/assets/538a04f0-892a-4f1d-945d-25741b51f74a" />  

  ```sql
  SELECT column1, column2, ...
  FROM table_name
  ORDER BY column_name [ASC|DESC];
  ```
  - ASC 表示升序（从小到大），这是默认选项
  - DESC 表示降序（从大到小）

  假设有一个 tutorials 表：  
  | id | title | difficulty | views | publish_date |  
  |----|-------|------------|-------|--------------|  
  | 1 | SQL基础入门 | 1 | 15000 | 2025-01-15 |  
  | 2 | JavaScript高级技巧 | 3 | 12000 | 2025-02-20 |  
  | 3 | Python数据分析 | 2 | 18000 | 2025-01-25 |  
  | 4 | React实战项目 | 3 | 20000 | 2025-03-05 |  

## 单列排序
  ```sql
  SELECT title, views
  FROM tutorials
  ORDER BY views DESC;
  ```

  结果：  
  | title | views |  
  |-------|-------|  
  | React实战项目 | 20000 |  
  | Python数据分析 | 18000 |  
  | SQL基础入门 | 15000 |  
  | JavaScript高级技巧 | 12000 |  

## 多列排序
  ```sql
  SELECT title, difficulty, views
  FROM tutorials
  ORDER BY difficulty ASC, views DESC;
  ```

  结果：  
  | title | difficulty | views |  
  |-------|------------|-------|  
  | SQL基础入门 | 1 | 15000 |  
  | Python数据分析 | 2 | 18000 |  
  | React实战项目 | 3 | 20000 |  
  | JavaScript高级技巧 | 3 | 12000 |  

## 使用列的位置排序

## 结果集去重(DISTINCT)

  去重查询的基本语法是：   
  ```sql
  SELECT DISTINCT column1, column2, ...

  ```

  | 5 | 广度优先搜索 | 搜索 |  
  | 6 | 归并排序 | 排序 |  

  如果我们想‌知道系统中有哪些不⁢同的算法标签：  
  
  `SELECT DISTINCT tag FROM algorithm_tags;`  
  结果：  
  | tag |   
  | --- |  
  | 排序 |  
  | 搜索 |

#  结果集截断和偏移(LIMIT、OFFSET)
  LIMIT 子句的基本语法是：  
  ```sql
  SELECT column1, column2, ...
  FROM table_name
  LIMIT row_count;
  ```

## 使用 OFFSET 进行结果偏移
  OFFSET 关键字可以与 LIMIT 一起使用，跳过指定数量的行后再开始获取数据。这在实现分页功能时非常有用：  
  ```sql
  SELECT column1, column2, ...
  FROM table_name
  LIMIT row_count OFFSET offset_value;
  ```

  例如，要实‌‌‌‌‌‌‌现编程导航网⁢⁢⁢⁢站⁢的⁢文⁢章分页‏‏‏‏，每‏页显‏示 ‏‍‍‍‍10 ‍篇，查‍询⁡⁡⁡⁡第 ‍3⁡ 页的文⁡章：  
  ```sql
  SELECT id, title, author, publish_date
  FROM articles
  ORDER BY publish_date DESC
  LIMIT 10 OFFSET 20;  -- 跳过前20条(前2页)，获取接下来的10条(第3页)
  ```

  在某些数据‌‌库（如 MySQL）⁢⁢中，还有一种简写形式‏‏:  
  ```sql
  SELECT column1, column2, ...
  FROM table_name
  LIMIT offset_value, row_count;
  ```

  上例可以写成：  
  ```sql
  SELECT id, title, author, publish_date
  FROM articles
  ORDER BY publish_date DESC
  LIMIT 20, 10;  -- 从第21行开始，获取10行
  ```

## 使用场景
  LIMIT 和 OFFSET 在实际开发中有很多应用：  
  1. 分页显示：几乎所有显示大量数据的应用都需要分页
  2. 性能优化：只获取需要的数据，减少网络传输和应用处理负担
  3. 数据采样：快速获取少量数据进行分析或测试
  4. 排行榜：如"前 10 名用户"、"最热门的 5 篇文章"等

  使用 LIMIT 和 OFFSET 时需要注意：
  - OFFSET 值较大时可能会导致性能问题，因为数据库必须先获取所有行，然后丢弃前面的行
  - 在频繁更新的表中，分页可能会导致某些数据被重复或遗漏
  - 对于大规模数据集，可能需要考虑其他分页策略（如基于游标的分页）

# 练习题
## 练习 1：基本查询
  假设有一个 employees 表，包含员工信息，请编写 SQL 查询获取所有员工的姓名和职位信息。  
  答案：  
  `SELECT name, position FROM employees;`  

## 练习 2：结果排序和限制
  有一个 interview_questions 表，包含了标题（title）、浏览数（views）、类别（category），请编写 SQL 查询获取浏览量最高的 5 道 Java 相关面试题的标题和浏览量。  
  答案：  
  ```sql
  SELECT title, views
  FROM interview_questions
  WHERE category = 'Java'
  ORDER BY views DESC
  LIMIT 5;
  ```

## 练习 3：综合应用
  有一个 courses 表记录教程信息，包括 id、title(标题)、price(价格)、discount(折扣)、category(分类)字段。请编写一个查询，获取"前端开发"分类下所有课程的标题、原价、折扣和最终价格(原价乘以折扣)，按最终价格从低到高排序，并为计算出的价格列设置一个友好的中文别名"最终价格"。  
  ```sql
  SELECT 
      title, 
      price, 
      discount, 
      price * discount AS "最终价格"
  FROM courses
  WHERE category = '前端开发'
  ORDER BY price * discount ASC;
  ```
