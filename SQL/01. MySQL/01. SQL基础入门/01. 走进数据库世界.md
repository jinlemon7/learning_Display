# 为什么要学习 SQL？
  SQL（St‌‌‌ructured Que⁢⁢⁢ry Language，‏‏‏结构化查询语言）是用来与‍‍‍数据库通信的标准语言。通⁡⁡⁡过 SQL，我们可以：  
  1. 访问数据：从数据库中检索我们需要的信息
  2. 操作数据：插入、更新和删除数据库中的记录
  3. 创建和修改数据结构：定义数据如何组织和存储
  4. 控制访问权限：决定谁可以访问或修改数据

# 关系型数据库简介
  关系型数据库的核心概念包括：  
  1. 表(Table)：存储数据的基本单位，由行和列组成。例如，一个"用户表"可能包含用户的基本信息。
  2. 行(Row)：也称为记录，表示表中的一个数据项。例如，用户表中的一行代表一个用户的完整信息。
  3. 列(Column)：也称为字段，表示数据的一个属性。例如，用户表中的列可能包括用户名、邮箱、注册时间等。
  4. 主键(Primary Key)：唯一标识表中每一行的字段。例如，用户 ID 可以作为用户表的主键。
  5. 外键(Foreign Key)：建立表之间关联的字段，它指向另一个表的主键。例如，订单表中的用户 ID 可以作为外键关联到用户表。
  6. 索引(Index)：提高数据检索速度的数据结构。
  <img width="2238" height="2408" alt="image" src="https://github.com/user-attachments/assets/04cab14e-d309-4525-99ba-87460c32485f" />

  关系型数据库的优势在于：  
  - 数据一致性：通过外键约束等机制，确保数据的完整性和一致性
  - ACID 特性：支持事务处理，确保数据操作的原子性、一致性、隔离性和持久性
  - 标准化：遵循一套严格的数据组织原则，减少数据冗余
  - SQL 支持：提供强大的查询语言，方便数据操作和分析

# SQL 语言的起源与发展
## 起源
  SQL 的故事始于 1970 年代‌‌‌。当时，IBM 研究员 Edgar F. Codd 发表了一篇关于⁢⁢⁢关系型数据库的论文，至此奠定了关系型数据库的理论基础。随后，IBM‏‏‏ 开发了一种名为 SEQUEL（Structured Englis‍‍‍h Query Language）的语言，用于操作关系型数据库。后⁡⁡⁡来，由于商标问题，SEQUEL 更名为 SQL。

## MySQL 特有语法：  
  ```sql
  -- 使用 LIMIT 进行分页查询
  SELECT * FROM users LIMIT 10, 5;  -- 跳过10条记录，获取接下来5条
  
  -- 使用 ON DUPLICATE KEY UPDATE 处理重复键
  INSERT INTO users (id, name) VALUES (1, '鱼皮')
  ON DUPLICATE KEY UPDATE name = '鱼皮';
  ```

## PostgreSQL 特有语法：  
  ```aql
  -- 使用 RETURNING 子句获取操作影响的行
  INSERT INTO users (name) VALUES ('鱼皮') RETURNING id;
  
  -- 使用 ILIKE 进行大小写不敏感的模糊匹配
  SELECT * FROM users WHERE name ILIKE '%皮%';
  ```

## Oracle 特有语法：  
  ```sql
  -- 使用 ROWNUM 进行分页查询
  SELECT * FROM (
    SELECT users.*, ROWNUM rn FROM users WHERE ROWNUM <= 15
  ) WHERE rn > 10;
  
  -- 使用 CONNECT BY 查询层级数据
  SELECT name FROM employees
  START WITH manager_id IS NULL
  CONNECT BY PRIOR employee_id = manager_id;
  ```

# 练习题
## 练习 1：概念理解
  请解释关系‌‌‌‌‌‌型数据库中"⁢⁢⁢表⁢"⁢、⁢"行"‏‏‏和"‏列"‏的概‏‍‍‍念，并‍举例说‍明⁡⁡⁡它们之间‍⁡的关系⁡。  
  
  答案：  
  在关系型数据库中：
  - 表(Table)：是存储相关数据的结构，由行和列组成。例如"用户表"存储所有用户信息。
  - 行(Row)：代表表中的一条记录，包含该记录的所有属性。例如用户表中的一行代表一个特定用户的全部信息。
  - 列(Column)：代表数据的一个属性或字段。例如用户表可能包括"用户名"、"邮箱"、"注册日期"等列。

## 练习 2：数据库选择
  假设你正在为一个小‌‌‌‌‌‌型电商网站选择数据库系统。该网站预⁢⁢⁢⁢⁢⁢计每天有约 5000 名访客，需要‏‏‏‏‏‏存储产品信息、用户账户和订单数据。‍‍‍‍‍‍请从上述介绍的数据库管理系统中选择⁡⁡⁡⁡⁡⁡一个，并说明你的选择理由。

## 练习 3：SQL 应用场景
  阅读以下场‌‌‌‌景，‌‌思考 SQ⁢L⁢ ⁢如何应⁢用，‏并尝‏试写⁢⁢‏出可能‍的 ‏S‍QL‍ 语⁡句。   

  场景：算法‌‌‌‌‌‌导航平台想要找出过⁢⁢⁢⁢⁢⁢去一个月内，被查看‏‏‏‏‏‏次数最多的前 10‍‍‍‍‍‍ 种算法类型，以便⁡⁡⁡⁡⁡⁡优先更新这些类型的内容。  

  答案：  
  ```sql
  SELECT 
      algorithm_type, 
      COUNT(*) as view_count
  FROM algorithm_views
  WHERE view_date >= CURRENT_DATE - INTERVAL 30 DAY
  GROUP BY algorithm_type
  ORDER BY view_count DESC
  LIMIT 10;
  ```
  - 从 algorithm_views 表选择数据，这张表记录了算法被查看的记录
  - 只选择过去 30 天内的记录
  - 按算法类型分组
  - 计算每种类型的查看次数
  - 按查看次数降序排序
  - 只返回前 10 条记录
