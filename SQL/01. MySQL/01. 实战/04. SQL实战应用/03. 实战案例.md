## 案例 1: 销售分析

### 计算各商品类别的销售额、平均订单金额和销量  
  编写思路：  
  1. 通过多表连接获取所需的商品类别和销售数据
  2. 使用 JOIN 连接 order_items、products 和 orders 表
  3. 过滤出已完成状态的订单和特定时间范围
  4. 按商品类别分组统计销售额、平均订单金额和销量
  5. 按销售额降序排序显示结果

  <img width="1238" height="3802" alt="image" src="https://github.com/user-attachments/assets/f6834c96-7097-4bb0-9ac9-22961971268b" />  

  ```sql
  SELECT 
      p.category,
      SUM(oi.price * oi.quantity) AS total_sales,
      AVG(o.total_amount) AS avg_order_value,
      SUM(oi.quantity) AS total_quantity_sold
  FROM order_items oi
  JOIN products p ON oi.product_id = p.product_id
  JOIN orders o ON oi.order_id = o.order_id
  WHERE o.status = 'completed'
  AND o.order_date BETWEEN '2025-01-01' AND '2025-12-31'
  GROUP BY p.category
  ORDER BY total_sales DESC;
  ```

### 查找销售额前 10 的产品  
  编写思路：  
  1. 连接 order_items 表与 products 表获取产品信息和销售数据
  2. 过滤已完成的订单
  3. 按产品分组并计算每个产品的销售总量和销售总额
  4. 使用 ORDER BY 和 LIMIT 获取销售额前 10 的产品
  5. 按销售额降序排列结果

  <img width="986" height="3692" alt="image" src="https://github.com/user-attachments/assets/7bc9942a-46c4-402b-ae70-0be84dafb2c1" />  

  ```sql
  SELECT 
      p.product_id,
      p.product_name,
      SUM(oi.quantity) AS total_quantity,
      SUM(oi.price * oi.quantity) AS total_sales
  FROM order_items oi
  JOIN products p ON oi.product_id = p.product_id
  JOIN orders o ON oi.order_id = o.order_id
  WHERE o.status = 'completed'
  GROUP BY p.product_id, p.product_name
  ORDER BY total_sales DESC
  LIMIT 10;
  ```

  ### 计算月度销售增长率  
   编写思路：  
   1. 创建 CTE 计算每个月的总销售额
   2. 使用 LEFT JOIN 将当前月份与前一个月的销售数据关联
   3. 使用 DATE_FORMAT 和 STR_TO_DATE 函数处理月份格式
   4. 计算月度增长率：(当月销售额-上月销售额)/上月销售额 * 100
   5. 按月份顺序排序显示结果

  <img width="704" height="2478" alt="image" src="https://github.com/user-attachments/assets/dcc147ee-37d0-4bf2-ab88-c69ab24c8029" />  

  ```sql
  WITH monthly_sales AS (
      SELECT 
          DATE_FORMAT(order_date, '%Y-%m') AS month,
          SUM(total_amount) AS sales
      FROM orders
      WHERE status = 'completed'
      GROUP BY DATE_FORMAT(order_date, '%Y-%m')
  )
  SELECT 
      current.month,
      current.sales AS current_month_sales,
      prev.sales AS previous_month_sales,
      (current.sales - prev.sales) / prev.sales * 100 AS growth_rate
  FROM monthly_sales current
  LEFT JOIN monthly_sales prev ON current.month = DATE_FORMAT(DATE_ADD(STR_TO_DATE(prev.month, '%Y-%m'), INTERVAL 1 MONTH), '%Y-%m')
  ORDER BY current.month;
  ```

## 案例 2: 库存管理

### 查找库存不足的商品  
  编写思路：  
  1. 连接 products 表与 order_items 表获取库存和订购数据
  2. 使用 LEFT JOIN 保证所有产品都包括在结果中，即使没有订单
  3. 计算每个产品的已订购数量总和
  4. 使用 COALESCE 函数处理 NULL 值（没有订单的情况）
  5. 最后使用 WHERE 子句过滤出库存小于已订购数量的产品

  <img width="1038" height="3604" alt="image" src="https://github.com/user-attachments/assets/0498367e-a5aa-432b-ab43-e9776d010f59" />  

  ```sql
  SELECT 
      p.product_id,
      p.product_name,
      p.stock,
      COALESCE(SUM(oi.quantity), 0) AS ordered_quantity
  FROM products p
  LEFT JOIN order_items oi ON p.product_id = oi.product_id
  LEFT JOIN orders o ON oi.order_id = o.order_id AND o.status = 'processing'
  GROUP BY p.product_id, p.product_name, p.stock
  HAVING p.stock - COALESCE(SUM(oi.quantity), 0) < 10
  ORDER BY p.stock - COALESCE(SUM(oi.quantity), 0);
  ```

### 计算商品周转率  
   编写思路：  
   1. 创建 CTE 计算每个商品过去一年的销售总量
   2. 连接商品表获取当前库存数据
   3. 计算周转率公式：年销售量/当前库存
   4. 处理库存为 0 的特殊情况，避免除以 0 的错误
   5. 按周转率降序排列，把周转率为 NULL 的商品排在最后

  <img width="824" height="3630" alt="image" src="https://github.com/user-attachments/assets/a7639156-ca0d-4d26-a43b-2ad4d6a7fc11" />  

  ```sql
  WITH annual_sales AS (
      SELECT 
          p.product_id,
          p.product_name,
          p.category,
          SUM(oi.quantity) AS total_sold
      FROM products p
      JOIN order_items oi ON p.product_id = oi.product_id
      JOIN orders o ON oi.order_id = o.order_id
      WHERE o.status = 'completed'
      AND o.order_date >= DATE_SUB(CURRENT_DATE, INTERVAL 1 YEAR)
      GROUP BY p.product_id, p.product_name, p.category
  )
  SELECT 
      a.product_id,
      a.product_name,
      a.category,
      a.total_sold,
      p.stock,
      CASE
          WHEN p.stock = 0 THEN NULL
          ELSE a.total_sold / p.stock
      END AS turnover_rate
  FROM annual_sales a
  JOIN products p ON a.product_id = p.product_id
  ORDER BY turnover_rate DESC;
  ```

## 案例 3: 消费者行为分析

### 用户购买路径分析  
  编写思路：  
  1. 创建 CTE 分析用户从浏览到购买的转化过程
  2. 连接 users、page_views 和 orders 表获取完整的用户行为数据
  3. 筛选出用户浏览产品后7天内购买的记录
  4. 计算每个产品的查看用户数、购买用户数和转化率
  5. 筛选出有足够样本量的产品(至少 100 个查看者)
  6. 按转化率降序排序查看效果最好的产品

  <img width="746" height="3634" alt="image" src="https://github.com/user-attachments/assets/c6910e8d-5022-4083-955e-c3f4f5b6c460" />  

  ```sql
  WITH user_journey AS (
      SELECT
          u.user_id,
          u.username,
          pv.product_id,
          pv.view_date AS view_time,
          o.order_id,
          o.order_date
      FROM users u
      JOIN page_views pv ON u.user_id = pv.user_id
      LEFT JOIN orders o ON u.user_id = o.user_id
      LEFT JOIN order_items oi ON o.order_id = oi.order_id AND pv.product_id = oi.product_id
      WHERE pv.view_date <= o.order_date
      AND o.order_date <= DATE_ADD(pv.view_date, INTERVAL 7 DAY)
  )
  SELECT
      product_id,
      COUNT(DISTINCT user_id) AS viewer_count,
      COUNT(DISTINCT order_id) AS purchaser_count,
      COUNT(DISTINCT order_id) / COUNT(DISTINCT user_id) * 100 AS conversion_rate
  FROM user_journey
  GROUP BY product_id
  HAVING viewer_count >= 100
  ORDER BY conversion_rate DESC;
  ```

### 复购率分析  
  编写思路：  
  1. 创建 CTE 统计每个用户的订单数量
  2. 只计算状态为"completed"的订单
  3. 计算每个订单数量级别的用户数量
  4. 计算每个级别用户占总用户数的百分比
  5. 按订单数量升序排序，便于分析用户复购情况

  <img width="852" height="3644" alt="image" src="https://github.com/user-attachments/assets/80b2bc24-3e59-4a61-a3f9-0ed6ad8256e7" />  

  ```sql
  WITH customer_orders AS (
      SELECT 
          user_id,
          COUNT(DISTINCT order_id) AS order_count
      FROM orders
      WHERE status = 'completed'
      GROUP BY user_id
  )
  SELECT 
      order_count,
      COUNT(user_id) AS user_count,
      COUNT(user_id) / (SELECT COUNT(DISTINCT user_id) FROM orders) * 100 AS percentage
  FROM customer_orders
  GROUP BY order_count
  ORDER BY order_count;
  ```

## 练习 1：电商数据分析
  编程导航商城需要‌‌‌分析用户购买行为。请编写一个查⁢⁢⁢询，找出购买了特定产品（例如 ‏‏‏"Python 高级教程"）的‍‍‍用户还经常购买的其他产品（即常⁡⁡⁡见的产品组合），按关联度排序。  

  答案：  
  ```sql
  WITH target_product AS (
      SELECT product_id FROM products WHERE product_name = 'Python 高级教程'
  ),
  target_buyers AS (
      SELECT DISTINCT o.user_id
      FROM orders o
      JOIN order_items oi ON o.order_id = oi.order_id
      WHERE oi.product_id = (SELECT product_id FROM target_product)
  )
  SELECT 
      p.product_id,
      p.product_name,
      COUNT(DISTINCT o.user_id) AS buyer_count,
      COUNT(DISTINCT o.user_id) / (SELECT COUNT(*) FROM target_buyers) * 100 AS purchase_rate
  FROM orders o
  JOIN order_items oi ON o.order_id = oi.order_id
  JOIN products p ON oi.product_id = p.product_id
  WHERE o.user_id IN (SELECT user_id FROM target_buyers)
  AND oi.product_id <> (SELECT product_id FROM target_product)
  GROUP BY p.product_id, p.product_name
  HAVING COUNT(DISTINCT o.user_id) >= 3  -- 至少有3个共同购买者
  ORDER BY purchase_rate DESC, buyer_count DESC;
  ```

## 练习 2：用户行为分析
  面试鸭平台需‌‌‌要了解用户的学习习惯。请⁢⁢⁢编写一个查询，分析用户学‏‏‏习连续性，即计算每个用户‍‍‍最长的连续学习天数（连续⁡⁡⁡几天有学习活动记录）。  
  答案：  
  ```sql
  WITH daily_activity AS (
      -- 获取用户每天的活动记录
      SELECT 
          user_id, 
          DATE(activity_time) AS activity_date
      FROM learning_activities
      GROUP BY user_id, DATE(activity_time)
  ),
  activity_with_row_num AS (
      -- 为每个用户的活动日期分配序号
      SELECT 
          user_id,
          activity_date,
          ROW_NUMBER() OVER(PARTITION BY user_id ORDER BY activity_date) AS row_num
      FROM daily_activity
  ),
  activity_groups AS (
      -- 识别连续日期（连续日期的 activity_date - row_num 值相同）
      SELECT 
          user_id,
          activity_date,
          DATE_SUB(activity_date, INTERVAL row_num DAY) AS group_date
      FROM activity_with_row_num
  ),
  streak_lengths AS (
      -- 计算每个连续组的长度
      SELECT 
          user_id,
          group_date,
          COUNT(*) AS streak_length,
          MIN(activity_date) AS streak_start,
          MAX(activity_date) AS streak_end
      FROM activity_groups
      GROUP BY user_id, group_date
  )
  SELECT 
      u.user_id,
      u.username,
      MAX(sl.streak_length) AS longest_streak,
      (SELECT streak_start FROM streak_lengths 
       WHERE user_id = u.user_id AND streak_length = MAX(sl.streak_length)
       LIMIT 1) AS start_date,
      (SELECT streak_end FROM streak_lengths 
       WHERE user_id = u.user_id AND streak_length = MAX(sl.streak_length) 
       LIMIT 1) AS end_date
  FROM users u
  LEFT JOIN streak_lengths sl ON u.user_id = sl.user_id
  GROUP BY u.user_id, u.username
  ORDER BY longest_streak DESC;
  ```

## 练习 3：复杂报表生成

  WITH resume_completeness AS (

                   CASE WHEN r.awards IS NOT NULL THEN 1 ELSE 0 END +
                   CASE WHEN r.summary IS NOT NULL THEN 1 ELSE 0 END) >= 5 THEN '高度完善'
              WHEN (CASE WHEN r.education IS NOT NULL THEN 1 ELSE 0 END +
                   CASE WHEN r.experience IS NOT NULL THEN 1 ELSE 0 END +
                   CASE WHEN r.skills IS NOT NULL THEN 1 ELSE 0 END +
                   CASE WHEN r.projects IS NOT NULL THEN 1 ELSE 0 END +
                   CASE WHEN r.awards IS NOT NULL THEN 1 ELSE 0 END +
                   CASE WHEN r.summary IS NOT NULL THEN 1 ELSE 0 END) >= 3 THEN '中度完善'
              ELSE '低度完善'
          END AS completeness_level
      FROM resumes r
  ),
  interview_stats AS (
      -- 统计每份简历的面试结果
      SELECT 
          ir.resume_id,
          COUNT(*) AS total_interviews,
          SUM(CASE WHEN ir.result = 'success' THEN 1 ELSE 0 END) AS successful_interviews,
          SUM(CASE WHEN ir.result = 'success' THEN 1 ELSE 0 END) / COUNT(*) AS success_rate
      FROM interview_results ir
      GROUP BY ir.resume_id
  )
  SELECT 
      rc.completeness_level,
      COUNT(DISTINCT rc.resume_id) AS resume_count,
      AVG(COALESCE(is.success_rate, 0)) * 100 AS avg_success_rate,
      MIN(COALESCE(is.success_rate, 0)) * 100 AS min_success_rate,
      MAX(COALESCE(is.success_rate, 0)) * 100 AS max_success_rate,
      AVG(COALESCE(is.total_interviews, 0)) AS avg_interviews_per_resume,
      SUM(COALESCE(is.successful_interviews, 0)) / SUM(COALESCE(is.total_interviews, 1)) * 100 AS overall_success_rate
  FROM resume_completeness rc
  LEFT JOIN interview_stats is ON rc.resume_id = is.resume_id
  GROUP BY rc.completeness_level
  ORDER BY avg_success_rate DESC;
