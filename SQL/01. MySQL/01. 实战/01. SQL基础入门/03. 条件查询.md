# WHERE 子句基础
  WHERE 子句的基本语法是：  
  ```sql
  SELECT 列名1, 列名2, ...
  FROM 表名
  WHERE 条件;
  ```
  - WHERE 子句总是放在 FROM 子句之后，ORDER BY 子句之前。

# 空值处理(IS NULL/IS NOT NULL)

# 逻辑运算(AND, OR, NOT)
  逻辑运算符允许我们组合多个条件，创建更复杂的查询：  
  | 运算符 |	描述 |  
  | --- | --- |  
  | AND	| 如果两个条件都为真，结果为真 |  
  | OR | 如果至少一个条件为真，结果为真 |  
  | NOT | 反转条件的结果 |  

  <img width="2220" height="888" alt="image" src="https://github.com/user-attachments/assets/f8749088-a9b0-405f-9cd1-2fd391df3898" />

   ```sql
  SELECT course_name, price, rating
  FROM courses
  WHERE price < 200 AND rating > 4.7;
  ```

# 条件分支(CASE WHEN)
  CASE 表达式允‌许我们‌‌在 SQL 中进行条件分支处理⁢，类似于其他编程语言⁢⁢中的 if-el‏se 语句。CASE 表达式有两种‏‏基‍本形式：   
  
  简单 CASE 表达式：  
  ```sql
  CASE 表达式
      WHEN 值1 THEN 结果1
      WHEN 值2 THEN 结果2
      ...
      [ELSE 默认结果]
  END
  ```

  搜索 CASE 表达式：  
  ```sql
  CASE
      WHEN 条件1 THEN 结果1
      WHEN 条件2 THEN 结果2
      ...
      [ELSE 默认结果]
  END
  ```

  例如，我们可‌以使用 CASE 表达⁢式为课程添加难度分级:  
  ```sql
  SELECT 
      course_name,
      price,
      CASE
          WHEN price < 100 THEN '入门级'
          WHEN price >= 100 AND price < 200 THEN '进阶级'
          WHEN price >= 200 THEN '专家级'
          ELSE '未分类'
      END AS price_level
  FROM courses;
  ```
  结果：  
  | course_name | price | price_level |  
  |-------------|-------|-------------|  
  | AI零代码应用生成平台教程 | 599 | 专家级 |  
  | OJ在线判题项目教程 | 129 | 进阶级 |  
  | 2025年最新编程学习路线(汇总) | 259 | 专家级 |  
  | 🌲编程导航——职场指南 | 199 | 进阶级 |  
  | 用户中心项目教程 | 19.9 | 入门级 |  

  CASE 表达式可‌‌‌以在 SELECT 子句、WHER⁢⁢⁢E 子句、ORDER BY 子句等‏‏‏多个位置使用，使查询更加灵活和‍‍强大‍。  

# 练习题
## 练习 1：比较运算符应用
  假设有一个 users 表记录用户信息，包括 id, username, registration_date, last_login_date, articles_count（用户发表的文章数）等字段。请编写一个查询，找出 2025 年 1 月 1 日之后注册且发表文章数超过 5 篇的用户信息。  
  
  答案：  
  ```sql
  SELECT id, username, registration_date, articles_count
  FROM users
  WHERE registration_date > '2025-01-01' AND articles_count > 5;
  ```

## 练习 2：空值处理与逻辑运算
  使用上题中的 users 表，编写一个查询找出从未登录（last_login_date 为 NULL）或者最近一个月没有登录的用户。  

  答案：  
  ```sql
  SELECT id, username, last_login_date
  FROM users
  WHERE last_login_date IS NULL 
     OR last_login_date < CURRENT_DATE - INTERVAL 30 DAY;
  ```

## 练习 3：条件分支应用
  编写一个查询，根‌据用户‌的文‌章数为其划分等级：0⁢ 篇为"新手"，1-⁢5 篇为"‏进阶⁢用户"，6-20 篇为"活‍跃‏作者"，20 篇以上为"资深‏⁡专家"，并按等级‍降序排列结果。       
  ```sql
  SELECT 
      username,
      articles_count,
      CASE
          WHEN articles_count = 0 THEN '新手'
          WHEN articles_count BETWEEN 1 AND 5 THEN '进阶用户'
          WHEN articles_count BETWEEN 6 AND 20 THEN '活跃作者'
          WHEN articles_count > 20 THEN '资深专家'
          ELSE '未分类'
      END AS user_level
  FROM users
  ORDER BY 
      CASE
          WHEN articles_count = 0 THEN 1
          WHEN articles_count BETWEEN 1 AND 5 THEN 2
          WHEN articles_count BETWEEN 6 AND 20 THEN 3
          WHEN articles_count > 20 THEN 4
          ELSE 0
      END DESC;
  ```
