在设置页面，您可以设置编辑器首选项，以便在创建代码空间或打开现有代码空间时，选择使用以下编辑器打开：  
- Visual Studio Code（桌面应用程序）
- Visual Studio Code（Web 客户端应用程序）
- JupyterLab——Jupyter 项目的 Web 界面

从模板创建新的代码空间时，它始终在 Visual Studio Code Web 客户端中打开。您可以在任何受支持的编辑器中重新打开现有代码空间  

如果您想使用 Visual Studio Code 作为 GitHub Codespaces 的默认编辑器，则需要安装 Visual Studio Code 和 GitHub Codespaces 扩展。  

# 设置默认编辑器
  1. 在 GitHub 任意页面的右上角，点击你的个人资料图片，然后点击设置 。  
  2. 在侧边栏的“代码、规划和自动化”部分，点击代码空间 。  
  3. 在“编辑器首选项”下，选择您想要的选项。  
     - 如果您选择 Visual Studio Code （桌面应用程序），则必须确保您的本地计算机上已安装 VS Code。  
     - 如果您选择 Visual Studio Code （Web 客户端），下次创建或打开代码空间时，GitHub Codespaces 将自动在桌面应用程序中打开。  
       您可能需要同时允许浏览器和 Visual Studio Code 访问该程序才能成功打开  
     - 如果您选择 JupyterLab ，则必须将 JupyterLab 应用程序安装到您打开的代码空间中。默认的开发容器镜像包含 JupyterLab，  
       因此从默认镜像创建的代码空间始终会安装 JupyterLab。  
       如果您在开发容器配置中未使用默认镜像，则可以通过在 `devcontainer.json` 文件中添加 `ghcr.io/devcontainers/features/python` 功能来安装 JupyterLab。  
       您应该包含选项 `"installJupyterlab": true` 。有关更多信息，请参阅 `devcontainers/features` 仓库中 [`python`](https://github.com/devcontainers/features/tree/main/src/python#python-python) 功能的 README 文件。
