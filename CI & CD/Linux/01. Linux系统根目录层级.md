# Linux 根目录树状结构
  ```text
  /
  ├── bin -> /usr/bin
  ├── boot/
  ├── dev/
  ├── etc/
  ├── home/
  ├── lib -> /usr/lib
  ├── lib64 -> /usr/lib64
  ├── lost+found/
  ├── media/
  ├── mnt/
  ├── opt/
  ├── proc/
  ├── root/
  ├── run/
  ├── sbin -> /usr/sbin
  ├── snap/
  ├── srv/
  ├── sys/
  ├── tmp/
  ├── usr/
  │   ├── bin/
  │   ├── lib/
  │   ├── sbin/
  │   └── local/
  └── var/
  ```

# 主要目录功能说明
  | 目录 | 主要功能 |  
  | :--- | :--- |  
  | **/bin** | 普通用户使用的基本命令（如 `ls`, `cp`）。在合并系统中，它是指向 `/usr/bin` 的**符号链接**。 |  
  | **/bin.usr-is-merged** | 兼容性目录，通常也是指向 `/usr/bin` 的符号链接，用于标记系统已完成 /usr 合并。 |  
  | **/boot** | 引导加载程序使用的静态文件，包括 Linux 内核、initrd 镜像、Bootloader 配置文件（如 GRUB）。 |  
  | **/dev** | 设备文件系统（devtmpfs），包含所有硬件设备（磁盘、终端、输入设备等）的特殊文件。 |  
  | **/etc** | 系统配置文件目录，几乎所有的全局配置文件都存放在这里（如 `/etc/passwd`, `/etc/network/`）。 |  
  | **/home** | 普通用户的家目录。每个用户有一个对应的子目录（如 `/home/alice`）。 |  
  | **/init** | 内核启动后运行**第一个进程**（通常是 systemd 或 SysV init）的可执行文件。在合并系统中，它可能是指向 `/usr/lib/systemd/systemd` 的链接。 |  
  | **/lib** | 存放 `/bin` 和 `/sbin` 中命令所需的**共享库**及内核模块。现在通常是指向 `/usr/lib` 的符号链接。 |  
  | **/lib.usr-is-merged** | 兼容性目录，指向 `/usr/lib`，表明库文件已合并到 `/usr/lib`。 |  
  | **/lib64** | 64 位架构专用的库目录（如 x86_64 系统的动态链接器 `ld-linux-x86-64.so.2`），通常指向 `/usr/lib64`。 |  
  | **/lost+found** | 文件系统修复工具（如 `fsck`）存放恢复出来的**丢失数据片段**的目录。每个 ext 文件系统分区根下都有它。 |  
  | **/media** | 自动挂载的可移动介质挂载点（如 USB 闪存盘、DVD 驱动器通常挂载在 `/media/用户名/卷标`）。 |  
  | **/mnt** | 用于**临时手工挂载**文件系统的空目录（例如管理员临时挂载分区做维护）。 |  
  | **/opt** | 可选的第三方软件包安装目录（例如 Google Chrome、Java 可能安装在 `/opt` 下）。 |   
  | **/proc** | 虚拟文件系统（procfs），提供运行中的进程和内核信息（如 `/proc/cpuinfo`, `/proc/meminfo`）。 |  
  | **/root** | 超级用户 **root** 的家目录。普通用户无权访问。 |  
  | **/run** | tmpfs 文件系统，存放系统启动以来的**运行时变量数据**（如 `pid` 文件和套接字）。重启后清空。 |  
  | **/sbin** | 系统管理命令（如 `fdisk`, `mount`），通常只有 root 可使用。现在通常是指向 `/usr/sbin` 的符号链接。 |  
  | **/sbin.usr-is-merged** | 兼容性目录，指向 `/usr/sbin`，标记 `/sbin` 已合并到 `/usr/sbin`。 |  
  | **/snap** | Snap 软件包系统的默认安装目录（常见于 Ubuntu）。 |  
  | **/srv** | 本机提供的服务数据（如 Web 服务器的 `/srv/www/`，FTP 服务器的 `/srv/ftp/`）。 |  
  | **/sys** | 虚拟文件系统（sysfs），导出内核设备、驱动、总线等对象的信息，也是 `udev` 工作的地方。 |  
  | **/tmp** | 临时文件目录，任何用户都可以写，系统重启时通常会被清空（可配置）。 |  
  | **/usr** | **Unix系统资源**（Unix System Resources），包含大部分用户命令、库、文档、图标等。是合并后的核心目录。 |  
  | **/var** | **可变数据**目录，存放经常变化的信息（如日志 `/var/log`、邮件 `/var/mail`、缓存 `/var/cache`）。 |
