此方案为 配置客户端代理 (你已订阅或拥有其他代理服务)

# 使用 GitHub Container Registry (GHCR) 镜像
  v2rayA 官方除了 Docker Hub，也向 GitHub Container Registry (GHCR) 发布镜像。这通常是一个更稳定、下载速度更快的选择。  
  你可以直接用下面这行命令来拉取： 
  ```bash
  docker pull ghcr.io/v2raya/v2raya:latest
  ```

# 配置 docker-compose.yml
  ```bash
  mkdir -p v2raya
  nano docker-compose.yml
  ```

  复制下面的内容到docker-compose.yml：  
  ```bash
  services:
    v2raya:
      image: ghcr.io/v2raya/v2raya:latest
      container_name: v2raya
      restart: always
      volumes:
        - /etc/v2raya:/etc/v2raya  # 挂载配置文件目录，持久化 v2rayA 设置
      environment:
        - V2RAYA_ADDRESS=0.0.0.0:2017  # 让 v2rayA 监听所有网络接口的 2017 端口（否则默认只监听 127.0.0.1）
      privileged: true  # 赋予容器特权模式，允许修改宿主机网络规则（透明代理/全局代理需要）
      network_mode: host  # 使用宿主机网络栈，容器直接共享宿主机网络，端口无需映射，性能更高且支持透明代理
  ```

# 启动 v2rayA 容器
  在 docker-compose.yml 所在目录执行：  
  ```bash
  docker compose up -d
  ```

  v2rayA 图形界面默认端口是2017，别忘了去云服务商防火墙放行  

# 配置 v2rayA
  启动完成后，你可以通过 http://<你的服务器IP>:2017 访问 v2rayA 的 Web 管理界面进行后续配置。  

  1. 创建一个管理员账号
     账号信息位于本地，我们不会上传任何信息到服务器。  
     如不慎忘记密码，使用 v2raya --reset-password 重置。  
  2. 启动代理
     导入订阅链接  
     选择一个节点，点击“连接”  

# 为 Docker 守护进程配置代理
## daemon.json
  ```bash
  sudo nano /etc/docker/daemon.json
  ```

  在 /etc/docker/daemon.json 中填入：  
  ```bash
  {
    "proxies": {
      "http-proxy": "http://127.0.0.1:20172",
      "https-proxy": "http://127.0.0.1:20172",
      "no-proxy": "localhost,127.0.0.1,::1"
    }
  }
  ```
  - v2rayA 的三个核心端口
    | 端口 | 用途 | 说明 |
    |------|------|------|
    | 2017 | v2rayA 后端管理端口 | 这是你通过浏览器访问 v2rayA 图形化界面（Web GUI）时使用的端口。 |
    | 20171 | HTTP 代理端口（普通模式） | 这是一个标准的 HTTP 代理端口，所有流量都会被无条件发送到代理服务器。 |
    | 20172 | HTTP 代理端口（带分流规则） | 这也是一个 HTTP 代理端口，但内置了分流功能（例如 PAC 规则）。它会根据预设规则智能判断流量是直连还是走代理，因此通常效率更高。 |
  - 如果有代理认证，用 http://用户名:密码@代理IP:端口
  

## systemd drop-in
  创建 systemd 配置目录  
  ```bash
  sudo mkdir -p /etc/systemd/system/docker.service.d
  ```

  创建代理配置文件
  ```bash
  sudo nano /etc/systemd/system/docker.service.d/http-proxy.conf
  ```

  填入以下内容
  ```bash
  [Service]
  Environment="HTTP_PROXY=http://127.0.0.1:20172"
  Environment="HTTPS_PROXY=http://127.0.0.1:20172"
  Environment="NO_PROXY=localhost,127.0.0.1,::1"
  ```
  
## 重载并重启 Docker
  ```bash
  sudo systemctl daemon-reload && sudo systemctl restart docker
  ```

## 为什么 /etc/environment 没用
  - systemd 服务默认不读取 /etc/environment，除非服务文件中明确写了 EnvironmentFile=-/etc/environment
  - Docker 的 service 文件并没有这一行，所以你在 /etc/environment 里写再多代理变量，Docker 守护进程也看不到

# 设置用户级全局代理
  例如让 curl, wget, apt 等命令行工具（当你手动执行它们时）自动使用代理  
  ```bash
  sudo nano /etc/environment
  ```

  在 /etc/environment 里添加：  
  ```bash
  HTTP_PROXY="http://127.0.0.1:20172"
  HTTPS_PROXY="http://127.0.0.1:20172"
  NO_PROXY="localhost,127.0.0.1"
  ```
