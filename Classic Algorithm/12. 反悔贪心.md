# 思想
  整体追求最优解，每个阶段遇到不满足扩容条件(但当前子答案较已有子答案[存放在最小/大堆或已排序数组]更优)时，反悔，用前子答案替代较差子答案。  

## 问题描述：
  有 3 个任务，每个任务需要 1 个单位时间，且有一个截止时间（必须在截止时间之前完成）。每个任务有对应的利润。目标是选择若干任务使得总利润最大，且每个时间点只能做一个任务。  

  | 任务 | 利润 | 截止时间 |  
  |------|------|----------|  
  | A    | 100  | 1        |  
  | B    | 80   | 2        |  
  | C    | 90   | 2        |  

  - 时间从 0 开始，最多可以做 2 个任务（因为最大截止时间为 2）。

## 普通贪心（按利润从大到小排序，直接选）
  - 先选利润最大的 A（100），占用时间1。
  - 再选次大的 C（90），占用时间2，总利润 190。
  - 接着看 B（80），但时间2已被占用，且无法再增加时间，所以丢弃。
  结果：190（最优）  

## 反悔贪心（用小根堆维护已选任务）
  1. 将任务按利润从大到小排序：A(100,1), C(90,2), B(80,2)。
  2. 依次处理每个任务：
     - 选 A：堆 = [100]（堆中存已选任务的利润）
     - 选 C：当前已选数量 = 1，截止时间 2 允许做 2 个任务，直接加入，堆 = [90, 100]
     - 选 B：此时已选数量 = 2（达到当前任务截止时间允许的最大值）。B 的利润 80 与堆顶（最小利润 90）比较，80 < 90，不替换。
  3. 最后堆中利润和为 100+90 = 190。
  结果：190（最优）

## 为什么叫“反悔”？
  如果我们先选了 A 和 B（贪心得到 180），后来发现 C 比 B 更好，则“反悔”丢弃 B，改选 C。这个过程中，堆的作用就是随时准备用更大的利润替换已选中的最小利润，实现“反悔”操作。  

## 代码实现
  ```java
  import java.util.*;
  
  public class RegretGreedy {
      static class Task {
          int profit, deadline;
          Task(int profit, int deadline) {
              this.profit = profit;
              this.deadline = deadline;
          }
      }
  
      public static int maxProfit(Task[] tasks) {
          // 按利润从大到小排序
          Arrays.sort(tasks, (a, b) -> b.profit - a.profit);
          PriorityQueue<Integer> minHeap = new PriorityQueue<>();
          int time = 0;
          for (Task t : tasks) {
              if (time < t.deadline) {
                  // 当前时间未达到截止时间，直接选取
                  minHeap.offer(t.profit);
                  time++;
              } else if (!minHeap.isEmpty() && t.profit > minHeap.peek()) {
                  // 时间已满，但当前利润大于已选的最小利润，替换
                  minHeap.poll();
                  minHeap.offer(t.profit);
              }
          }
          int sum = 0;
          while (!minHeap.isEmpty()) sum += minHeap.poll();
          return sum;
      }
  
      public static void main(String[] args) {
          Task[] tasks = { new Task(100,1), new Task(80,2), new Task(90,2) };
          System.out.println(maxProfit(tasks));   // 输出 190
      }
  }
  ```
