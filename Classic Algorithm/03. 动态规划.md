- 记忆化搜索：自顶向底层层调用循环体，再自底向顶计算终值
- 递推：自底向顶计算终值（将“递归”改为只有“归”，去掉“递”）

多阶段决策最优解模型  
1. 最优子结构：子问题最优解 ——> 问题最优解
2. 无后效性：问题最优解只用到子问题最优解的值，不管子问题最优解怎么来的
3. 重复子问题

# 状态转移方程优化
  ```java
  int[] f = new int[n];
  f[i] = Math.max(f[i - 1], f[i - 2] + v[i]);
  ```
  - 此时i需从2开始，f[0], f[1]需赋初值

  优化：f[]中的i用i+2替换  
  ```java
  int[] f = new int[n + 2];
  f[i + 2] = Math.max(f[i + 1], f[i] + v[i]);
  ```
  - PS: 本质上是将前f扩容2列，变成后f。前f的值向右移2列。  

# f数组空间优化

## 用滚动数组代替f数组，修改第1维大小  

## 降维

### 0-1背包
  ```java
  f[i+1][c] = f[i][c] + f[i][c-w[i]];
  // 优化：去掉第1维
  f[c] = f[c] + f[c - w[i]];
  ```

  设w[2] = 2  
  |     |     |     |     |     |     |     |     |  
  | --- | --- | --- | --- | --- | --- | --- | --- |  
  | i | 1 | 2 | 3 | 5 | 6 | 7 | 9 |  
  | i+1 | 1 | 2 | 3 | 5 | 6 | 7 | 9 |  

  优化本质：直接在第i行上更新第i+1行  

  问题：从左到右遍历 导致 数据丢失
  ```java
  f[2] = f[2] + f[0] = 3 + 1 = 4
  // 原来的f[2]被新f[2]覆盖了，数据丢失
  f[4] = f[4] + f[2] = 6 + 4 = 10
  ```

  解决方案：从右到左遍历  
  ```java
  f[4] = f[4] + f[2]
  f[2] = f[2] + f[0]
  ```
  - 解决了数据丢失的问题
  - 0-1背包优化到一维空间需倒序遍历

### 完全背包
  例：
  ```java
  f[i+1][c] = Math.min( f[i][c], f[i+1][c-w[i]] + v[i] )
  // 优化：去掉第1维
  f[c] = Math.min( f[c], f[c-w[i]] + v[i] )
  ```

  设 w[i] = 2, v[i] = 1  
  |     |     |     |     |     |     |     |     |  
  | --- | --- | --- | --- | --- | --- | --- | --- |  
  | i | 0 | 2 | 2 | 5 | 6 | 6 | 9 |  
  | i+1 | 0 | 2 | 2 | 5 | 6 | 6 | 9 |  
  
  直接在第i行上更新第i+1行  

  此时从左到右遍历正确：  
  ```java
  f[2] = Math.min( f[2], f[0] + 1 )
  //  = min( 2, 1 ) = 1
  f[4] = Math.min( f[4], f[2] + 1 )
  // = min( 6, 2 ) = 2
  ```

  PS: 完全背包优化到一维空间需正序遍历  

### 总结：  
  | 下标 | 0 |  1  |  2  |  3  |  4  |  5  |  6  |  
  | --- | --- | --- | --- | --- | --- | --- | --- |  
  |  i  |  0  |  2  |  2  |  5  |  6  |  6  |  9  |  
  | i+1 |  0  |  2  |  2  |  5  |  6  |  6  |  9  |  

  假设遍历变量在i+1行下标4位置  
  如果状态转移方程是从i行0-4位置或i+1行4-6位置转移过来，则需倒序遍历  
  如果状态转移方程是从i行4-6位置或i+1行0-4位置转移过来，则需正序遍历  

# 0-1背包
  n个物品，第i个物品的体积为w[i]，价值为v[i]  
  每个物品至多选1个，体积不超过capacity，求最大价值和  

  回溯三问：  
  - 当前操作？  max( dfs(i-1, c), dfs(i-1, c-w[i]) + v[i] )
  - 当前问题？
  - 下一个子问题？
    - 选：前i-1个物体，剩余容量c-w[i]的最大价值和
    - 不选：前i-1个物体，剩余容量c的最大价值和
   
  常见变形：  
  - 至多装capacity，求方案数/最大价值和
  - 恰好装capacity，求方案数/最大/最小价值和
  - 至少装capacity，求方案数/最小价值和

## 恰好装capacity，求方案数

  ```java
  dfs(i - 1, c) + dfs(i - 1, c - w[i]);
  ```

# 完全背包
  有n种物品，第i种物品的体积为w[i]，价值为v[i]  
  每种物品无限次重复选，求体积不超过capacity的最大价值和  

  回溯三问：  
  - 当前操纵？    max( dfs(i-1, c), dfs(i, c-w[i]) + v[i] )
  - 当前问题？
  - 下一个子问题？
    - 选1个：前i种，剩余容量c-w[i]的最大价值和
    - 不选：前i-1种物品，剩余容量c的最大价值和
   
  常见变形：  
  - 至多装capacity，求方案数/最大价值和
  - 恰好装capacity，求方案数/最大/最小价值和
  - 至少装capacity，求方案数/最小价值和

## 恰好装capacity，求最小价值和
  ```java
  Math.min(dfs(i - 1, c), dfs(i, c - w[i]) + v[i]);
  if (i < 0) {    // 递归边界
      // Integer.MAX_VALUE表示不合法：无法恰好装满
      return c == 0 ? 0 : Integer.MAX_VALUE;    // 剩余容量 c == 0（恰好装capacity）时，价值才=0
  }
  ```

  递推：
  ```java
  int[][] f = new int[n + 1][c + 1];
  for (int[] row : f) {
    Arrays.fill(row, Integer.MAX_VALUE);
  }

  // 重点：初始值：f矩阵考虑前0个物品，装0的容量，价值=0
  f[0][0] = 0;    

  for (int i = 0; i < w.length; i++) {
    int x = w[i];
    for (int t = 0; t < c+1; t++) {
        if (t >= x && f[i + 1][t - x] != Integer.MAX_VALUE ) {    // 状态不允许从 非法值（无法恰好装满t-x） 转移过来
            f[i + 1][t] = Math.min(f[i][t], f[i + 1][t - x] + v[i]);
        } else {
            f[i + 1][t] = f[i][t];
        }
    }
  }

  return f[n][c];
  ```

## 至少装capacity，求最小价值和
  ```java
  Math.min(dfs(i - 1, c), dfs(i, c - w[i]) + v[i]);    //重点：考虑“至少装capacity”，即可以多装：剩余容量 c - w[i]<0 时，按剩余容量 =0 来算
  if (i < 0) {    // 递归边界
      // Integer.MAX_VALUE表示不合法：无法恰好装满
      return c <= 0 ? 0 : Integer.MAX_VALUE;    // 剩余容量 c <=0 (装满capacity或多装), 价值=0
  }
  ```

  递推：
  ```java
  int[][] f = new int[n + 1][c + 1];
  for (int[] row : f) {
    Arrays.fill(row, Integer.MAX_VALUE);
  }

  // 重点：初始值：f矩阵考虑前0个物品，装<=0的容量，价值=0
  f[0][0] = 0;    

  for (int i = 0; i < w.length; i++) {
    int x = w[i];
    for (int t = 0; t < c+1; t++) {

        // 先不选当前物品
        f[i + 1][t] = f[i][t];

        // 考虑可选当前物品——则取min（允许超额，t-x <= 0）
        if (f[i + 1][Math.max(t - x, 0)] != Integer.MAX_VALUE)     // 状态不允许从 非法值（无法恰好装满t-x） 转移过来
            f[i + 1][t] = Math.min(f[i + 1][t], f[i + 1][Math.max(t - x, 0)] + v[i]);
            //重点：考虑“至少装capacity”，即可以多装：f矩阵容量 t-x<0 时，按矩阵容量 =0 来算
    }
  }

  return f[n][c];
  ```

# 方案数 —— 状态转移方程
  以0-1背包为例  

## 恰好装 capacity，求方案数
  ```java
  dfs(i - 1, c) + dfs(i - 1, c - w[i]);
  // 结合
  if (i < 0) {    // 递归边界
      return c == 0 ? 1 : 0;    // c == 0（恰好装capacity）时，方案数才+1
  }
  ```

  递推：  
  ```java
  int[][] f = new int[n + 1][c + 1];

  // 重点：初始值：f矩阵考虑前0个物品，装0的容量，方案数=1
  f[0][0] = 1;    

  for (int i = 0; i < w.length; i++) {
    int x = w[i];
    for (int t = 0; t < c+1; t++) {
      if (t < x) {
        f[i + 1][t] = f[i][t];

        f[i + 1][t] = f[i][t] + f[i][t - x];
      }
    }
  }

  ```

  ```java
      return 1;    // 考虑前0个物品，剩余容量 >= 0 (不装), 方案数+1
  ```

  ```java
  int[][] f = new int[n + 1][c + 1];

    }

  ```

  ```java
  dfs(i - 1, c) + dfs(i - 1, Math.max(c - w[i], 0));    // 重点：考虑“至少装capacity”，即可以多装：剩余容量 c - w[i]<0时，按剩余容量 =0 来算
  if (i < 0) {    // 递归边界
  ```
