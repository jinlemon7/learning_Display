# BM(Boyer–Moore)算法
  <img width="2218" height="373" alt="7e2b28d5931918d2696ef0e6a047ed77" src="https://github.com/user-attachments/assets/b2f9ed0b-10b0-4767-a9ca-9f52cc7e82c0" />  

  坏字符不匹配位置，好后缀匹配位置：模式串进行匹配的字符在模式串中的位置  
  
  ①从模式串末尾开始匹配  
  不匹配时，坏字符'S'上的指针(模式串指针)所需移动(左移)数量：坏字符不匹配位置(6)-模式串中'S'上一次出现位置(-1) = 7  // -1：即's'在模式串中不存在  
  模式串指针移到了下标-1，下一步主串和模式串指针各+1后继续匹配  
  ②从模式串末尾开始匹配  
  不匹配时，好后缀'E'上的指针(模式串指针)所需移动(左移)数量：好后缀匹配位置(6)-模式串中上一次出现位置(0) = 6  
  模式串指针移到了下标0，继续匹配  
  好后缀：(MPLE),(PLE),(LE),(E)中在模式串中上一次出现的最长后缀：(E)  

  时间复杂度：T(n) = O(n/m)  

# Z函数
  Z 值（Z[i]）的定义：    
  从第 i 个字符（i 从 0 开始）开始，往后数，最多能连续数出几个字符，正好和这个字符串的 开头 一样。  
  例：  `ababa`  
  Z = [0,0,3,0,1]  // Z[0] 不管，一般设 0（因为整个字符串和自己比没意义）  

  Z-box ：  
  就是目前找到的最靠右的、能与字符串开头精确匹配的一段区间。  
  用上面 `ababa` 举例子：  
  当 i=2 时，发现从位置 2 开始的一段 aba 与开头的 aba 完全相同。这段区间是 [2,4]（位置 2,3,4 三个字符）。  
  这个区间就是 Z-box。它的左端位置（2）叫 left，右端位置（4）叫 right。  
  
  <img width="2210" height="524" alt="15c07caed5fed372c73874cfd5cda31e" src="https://github.com/user-attachments/assets/10f6125d-b34f-4d72-b5c6-68137ebe6841" />  

  1. 将模式串，分隔字符(不在模式串和主串中的自定义字符)，主串按顺序串连起来
  2. 从下标1开始遍历
  3. 当前i在Z-box范围外，则进行暴力匹配（Z[i]初始值为0，Z[i]值作为当前i进行公共前缀匹配的前缀指针），若匹配成功，更新Z-box: 窗口右滑。
  4. 当前i在Z-box范围内，用O(1)时间得到s[i, right]与s的LCP长度(min(z[i-left], r-i+1) )，再继续跟Z-box范围外字符匹配。

## 获得Z数组
  ```java
  import java.util.ArrayList;
  import java.util.List;
  
  public class ZAlgorithmConverter {
      public static int[] computeZ(List<Character> pattern, List<Character> s) {
          int m = pattern.size();
          pattern.add('x');          // 自定义分隔字符，这里用'x'，保证任意z[i] <= m，因为'x'是s没有的字符，公共前缀数量 <= m
          pattern.addAll(s);
          int n = pattern.size();

          int[] z = new int[n];
          int left = 0, right = 0;
          for (int i = 1; i < n; i++) {  // 从1开始更新Z数组，Z[0] 不管，一般设 0（因为整个字符串和自己比没意义）
              // 当前i在Z-box范围内，利用Z-box的对称性:
              // 这个区间内的所有字符，与字符串开头同样长度的区间 [0, right-left] 完全一样。也就是说：
              // 对于任意 i 在 [left, right] 中(不包括left，因为z[left]更新后才会更新new Z-box，且z[0]=0 )，一定有 str[i] == str[i - left]。
              if (i <= right)
                  // Z-box只保证 [left, right] 这一段与 [0, right - left] 一样。
                  // 镜像位置k = i - left, 如果 i + z[k] - 1 超过了 right，那么超出的部分我们还没有验证过！
                  // 所以从 i 开始的匹配长度，最多只能先取到 right 为止，即长度 right - i + 1。
                  z[i] = Math.min(right - i + 1, z[i - left]);    // z[i - left]（镜像位置的部分匹配值）和right - i + 1（当前Z-box的剩余长度）都有可能是更小的那个

              // 当前i在Z-box范围外, 暴力匹配新的Z-box
              while (i + z[i] < n && pattern.get(z[i]) == pattern.get(i + z[i]) )
                  z[i]++;
              if (i + z[i] - 1 > right) {    // 更新一次Z-box，会在匹配前缀的子串的第0个下标就确定匹配的区间范围(Z-box)，并右滑Z-box
                  left = i;
                  right = i + z[i] - 1;
              }
          }
          return z;
      }
  }
  ```

# KMP(Knuth–Morris–Pratt)算法
  <img width="1671" height="376" alt="4617483eb52ac70da7b6cac283150535" src="https://github.com/user-attachments/assets/3b6e0d3e-b113-4421-8e90-26f3e30f8e29" />  

  从模式串开头开始比  
  移动位置 = 已匹配的字符数 - 对应的[部分匹配值]  
          = 6 - 2 = 4  

  部分匹配表PMT：  
  | 词 | A | B | C | D | A | B | D |  
  | --- | --- | --- | --- | --- | --- | --- | --- |  
  | 部分匹配值 | 0 | 0 | 0 | 0 | 1 | 2 | 0 |  

  部分匹配值：模式串的前缀与后缀中最长相同缀串的长度，用来在模式串下标i不与主串匹配时，模式串的指针左移的位置(调用PMT[i-1])。  
  因为模式串下标i之前的部分字符(当前前缀)在模式串前缀中已经出现过，直接在当前前缀的对应前缀+1的下标(最长相同缀串的长度：PMT[i-1])继续匹配  
  - 当前前缀：模式串下标i之前的部分字符, 与模式串的某个前缀相同  
  - 对应前缀：与当前前缀相同的，模式串的某个前缀

  | 前缀 | 后缀 |  
  | --- | --- |  
  | ABCDA | BCDAB |  
  | ABCD | CDAB |  
  | ABC | DAB |  
  | AB | AB |  
  | A | B |  

  - 模式串ABCDAB的部分匹配值为2，相同缀串为AB

  优化为next数组  
  | A | B | C | D | A | B | D |  
  | --- | --- | --- | --- | --- | --- | --- |  
  | -1 | 0 | 0 | 0 | 0 | 1 | 2 |  

  > [!NOTE]  
  > 部分匹配表右移1位：当发生不匹配：D与主串字符不匹配。直接访问next[j]，无须访问PMT[j-1]，模式串指针移至next[j]位置继续与主串匹配  

  ```java
  // 获得模式串next数组
  void getNext(char[] pattern, int[] next){
    // next.size()  = pattern.size()+1
    next[0] = -1;    // 模式串第0位不与主串匹配时，主串指针右移一位
    int i=0, j=-1;
    while(i<pattern.size()){    
      if(j==-1){    // 当前i下标之前，没有任何的对应前缀
        i++;  
        j++;
      }else if(pattern[i] == pattern[j]){  // 当前前缀指针和后缀指针所指字符相同
        i++;
        j++;
        next[i] = j;  // 将部分匹配值 赋给 当前前缀下标+1 的 next[]
      }else
        j = next[j];  // 当前前缀指针和后缀指针所指字符不匹配，前缀指针回退到可能的对应前缀+1下标或 下标-1
    }
  }
  ```

  ```java
  int search(char[] str, char[] pattern, int[] next){
    int i=0;
    int j=0;
    while(i<str.length && j<pattern.length){
      if(j==-1 || str[i] == pattern[j]){  // 当前匹配或j回退到-1
        i++;
        j++;
      }else{
        j=next[j];    // 当前不匹配，回退j
      }

      if(j==pattern.length)    // 成功匹配（完整）
        return i-j;
      else
        return -1;
    }
  }
  ```

## 获得next数组（简易版）
  ```java
  public static int[] computeNext(String pattern) {
      int m = pattern.length();
      int[] next = new int[m + 1];    // 模式串的部分匹配值表右移（长度+1，可连续匹配）
      next[0] = -1;

      // j:前缀指针，表示下标，也表示当前前缀已匹配长度（+1之后）
      // j = -1: 初值或者 连前缀第一个字符都不匹配，后缀指针需+1
      int i = 0, j = -1;
      while (i < m) {
          // 当前前缀指针和后缀指针所指字符不匹配（保证j>-1）
          while (j > -1 && pattern.charAt(i) != pattern.charAt(j)) {
              // 回退到可能的对应前缀+1下标或 下标-1
              j = next[j];
          }
          // j = -1 或 pattern[i] = pattern[j], 都要执行j+1, i+1
          j++;
          i++;
          // 将部分匹配值 赋给 当前前缀下标+1 的 next[]
          next[i] = j;
      }
      return next;
  }
  ```

## 字符串匹配（使用next数组）（简易版）
  ```java
  public static int countOccurrences(String s, String pattern, int[] next) {
      int m = pattern.length();
      int j = 0;  // 当前匹配的模式串长度
      int c = 0;  // 匹配次数
      
      for (int i = 0; i < s.length(); i++) {
          char v = s.charAt(i);
          // 不匹配时回退
          while (j > -1 && v != pattern.charAt(j)) {
              j = next[j];
          }
          j++;
          // 找到一个完整匹配
          if (j == m) {
              c++;
              j = next[j];  // 继续寻找下一个匹配
          }
      }
      return c;
  }
  ```

## 获得PMT表（简易版）
  ```java
  public static int[] computePi(String pattern) {
      int m = pattern.length();
      int[] pi = new int[m];
      // 前缀指针：表示下标，也表示当前前缀已匹配长度
      int j = 0;
      for (int i = 1; i < m; i++) {
          while (j > 0 && pattern.charAt(j) != pattern.charAt(i)) {
              j = pi[j - 1];
          }
          if (pattern.charAt(j) == pattern.charAt(i)) { // 前后缀指针所指字符相同，j才+1
              j++;
          }
          pi[i] = j;
      }
      return pi;
  }
  ```

## 字符串匹配（使用PMT数组）（简易版）
  ```java
  public static int countOccurrences(String s, String pattern, int[] pi) {
      int m = pattern.length();
      int j = 0;        // 当前已匹配的模式串长度（同时也是模式串的索引）
      int c = 0;        // 匹配次数
  
      for (int i = 0; i < s.length(); i++) {
          char v = s.charAt(i);
          // 当 j > 0 且当前字符不匹配时，根据 pi 数组回退
          while (j > 0 && pattern.charAt(j) != v) {
              j = pi[j - 1];
          }
          // 如果当前字符匹配，则增加匹配长度
          if (pattern.charAt(j) == v) {
              j++;
          }
          // 达到完整匹配
          if (j == m) {
              c++;
              j = pi[j - 1];   // 回退，继续寻找下一个重叠匹配
          }
      }
      return c;
  }
  ```
