# Java导包处理

## Java.lang
  Java默认已导入Java.lang包（无需程序员导入），包含：  
  | 类别 | 包含的核心内容 | 简要说明 |  
  | :--- | :--- | :--- |  
  | 核心根基类 | `Object`, `Class` | `Object` 是所有类的祖先；`Class` 则代表了正在运行的类或接口的元数据 。 |  
  | 数据类型与包装类 | `String`, `StringBuffer`, `StringBuilder`, `Number`,`Boolean`, `Integer`, `Double`, `Void` 等 | `String` 系列处理字符串；`Number`抽象类提供了各种包装类：为基本类型（如 `int`, `boolean`）提供了对象形式的表示 。 |  
  | 数学计算类 | `Math`, `StrictMath` | 提供常用的数学函数，如正弦、余弦、平方根、随机数、四舍五入等 。 |  
  | 系统与运行时类 | `System`, `Runtime`, `ClassLoader`, `Process` | 用于与系统环境交互，如标准输入/输出、加载类、执行外部进程、查询时间等 。 |  
  | 异常与错误类 | `Throwable`, `Exception`, `Error`, `RuntimeException` 等 | 构成了 Java 异常处理机制的基础，`Throwable` 是所有可以被抛出和捕获的异常类的父类 。 |  
  | 线程与并发类 | `Thread`, `Runnable`, `ThreadLocal` | 提供对多线程编程的基础支持，用于创建、管理和运行线程 。 |  
  | 枚举与注解类 | `Enum`, 以及 `java.lang.annotation` 子包 | `Enum` 是所有枚举类型的公共基类。`java.lang.annotation` 子包为 Java 的注解功能提供支持 。 |  

## java.util
  `import java.util.*;`  使用Collection，Map等容器/接口时，导包包括不限于Arrays.Collections等特殊类 
  | 类别 | 包含的核心内容 | 简要说明 |  
  | :--- | :--- | :--- |  
  | 集合框架 | `Collection`, `List`, `Set`, `Queue`, `Map` 等接口，以及 `ArrayList`, `LinkedList`, `HashSet`, `TreeSet`, `HashMap`, `TreeMap` 等实现类 | 提供了一套完整的数据结构和算法，用于存储、检索和操作对象组，是 Java 编程中最常用的部分之一。 |  
  | 工具类 | `Collections`, `Arrays`, `Objects` | `Collections` 和 `Arrays` 提供了对集合和数组进行排序、查找、比较等操作的静态方法。`Objects` 提供了操作对象引用的实用方法（如空值检查）。 |  
  | 日期和时间 | `Date`, `Calendar`, `TimeZone`, `GregorianCalendar` | 用于处理日期、时间和时区的类。注意：这些类大多较老，新项目中推荐使用 `java.time` 包，但 `java.util.Date` 等仍很常见。 |  
  | 国际化 | `Locale`, `ResourceBundle`, `Currency` | 用于编写可适应不同语言、地区和文化习惯的应用程序。 |  
  | 其他实用工具 | `Random`, `Scanner`, `UUID`, `Optional`, `Base64`, `BitSet`, `Properties` | 提供随机数生成、文本扫描和解析、生成唯一标识符、优雅处理可能为 `null` 的值、Base64 编解码、位操作、读写属性文件等功能。 |  
  | 遗留集合类 | `Vector`, `Stack`, `Hashtable`, `Enumeration`, `Dictionary` | Java 早期版本的集合类，虽然仍被支持（例如 `Stack` 在某些场景下仍在使用），但新代码中通常建议使用 Java 集合框架中的对应实现以获得更好的性能和设计。 |  

## java.math
  | 类别 | 包含的核心内容 | 简要说明 |  
  | :--- | :--- | :--- |  
  | 高精度数字类 | `BigDecimal`, `BigInteger` | `BigDecimal` 用于任意精度的有符号十进制数，适合精确的货币计算。`BigInteger` 用于任意精度的不可变整数，常用于加密等场景。 |  
  | 运算上下文与舍入 | `MathContext`, `RoundingMode` | `MathContext` 封装了数值运算的精度和舍入规则等上下文设置(全局设置)。`RoundingMode` 是一个枚举，为可能丢弃精度的运算指定舍入行为（如四舍五入）。 |

### RoundingMode 枚举值
  | 枚举值 | 核心行为 | 示例 (`5.5` 精确到个位) | 示例 (`-5.5` 精确到个位) |  
  | :--- | :--- | :--- | :--- |  
  | `UP` | 远离零取整（绝对值变大） | `6` | `-6` |  
  | `DOWN` | 向零取整（直接截断） | `5` | `-5` |  
  | `CEILING` | 天花板取整（向正无穷大） | `6` | `-5` |  
  | `FLOOR` | 地板取整（向负无穷大） | `5` | `-6` |  
  | `HALF_UP` | 四舍五入（最常用） | `6` | `-6` |  
  | `HALF_DOWN` | 五舍六入 | `5` | `-5` |  
  | `HALF_EVEN` | 银行家舍入法（向邻近偶数靠拢） | `6` | `-6` |  
  | `UNNECESSARY` | 断言精确（不能精确表示则抛异常） | 抛出 `ArithmeticException` | 抛出 `ArithmeticException` |

  | 常量 | 精度（有效数字位数） | 舍入模式 | 适用场景 |

# 入口函数：public static void main(String[] args)
  main函数作为Java应用的入口函数，JVM在第一时间访问它。  
  public保证可以被访问  
  static保证无需实例对象，可在第一时间被访问  

  类c语言应用入口函数的一种命名约定————必须是main()  
  是Java语言规范和JVM规范的规定，属于语言规范的约束  

## main()方法无法直接调用方法外的类成员(static成员除外)
  JVM第一时间访问main(),此时没有创建任何对象实例，自然无法访问  

# 对象图(Object Graph)
  一组通过引用相互连接的对象所构成的网络结构  
  - 点：代表一个个对象实例
  - 线：代表对象之间的引用关系（一个对象持有另一个对象的引用，即组合）

  关键特性：  
  - 引用关系
  - 可达性：从某个“根对象”出发，通过引用链能访问到的所有对象，都属于对象图的一部分
  - 复杂结构：树形结构、网状结构、环形结构

  应用场景  
  1. 序列化Serialization：深拷贝，序列化机制需要遍历整个对象图  
     序列化引擎会处理循环引用，避免无限循环  
  2. 垃圾回收Garbage Collection  
     GC从一组“GC Roots”（如静态变量、活动线程中的局部变量）开始，遍历对象图标记所有可达对象为“存活”，不可达的对象则被回收。  
  3. 深拷贝Deep Copy
  4. 数据遍历与处理  
     社交网络分析，文件系统遍历等算法，本质上都是处理对象图

  挑战：  
  1. 循环引用（Circular Reference）  使用set记录访问对象
  2. 性能问题  遍历/复制耗时大
  3. 内存泄漏  错误且长期持有某个对象引用，导致整个对象图无法被回收
