通过官方 QQ Bot API (v2) 将 Hermes 连接到 QQ — 支持私人 (C2C)、群组 @ 提及、公会和带有语音转录的直接消息。  

# 概述
  QQ Bot 适配器使用官方 QQ Bot API 来实现以下功能：  
  - 通过与 QQ 网关建立的持久 WebSocket 连接接收消息
  - 通过 REST API 发送文本和 Markdown 回复
  - 下载并处理图像、语音消息和文件附件
  - 使用腾讯内置的自动语音识别 (ASR) 或可配置的语音转文本 (STT) 服务提供商转录语音消息

# 先决条件
  1. QQ Bot 应用 — 在 q.qq.com 注册：
     - 创建一个新应用，并记下您的应用 ID 和应用密钥。
     - 启用所需意图：C2C 消息、群组 @ 消息、公会消息
     - 在沙盒模式下配置机器人进行测试，或发布到生产环境。
  2. 依赖项 — 该适配器需要 aiohttp 和 httpx ：
     `pip install aiohttp httpx`

# 配置
  
## 交互式设置  
  ```bash
  hermes gateway setup
  ```
  - 从平台列表中选择 QQ Bot ，然后按照提示操作。

## 手动配置
  在 ~/.hermes/.env 中设置所需的环境变量：  
  ```bash
  QQ_APP_ID=your-app-id
  QQ_CLIENT_SECRET=your-app-secret
  ```

# 环境变量
  | Variable | 描述 | 默认 |
  |----------|------|------|
  | QQ_APP_ID | QQ Bot 应用 ID（必填） | — |
  | QQ_CLIENT_SECRET | QQ Bot 应用密码（必填） | — |
  | QQBOT_HOME_CHANNEL | 用于定时任务/通知传递的 OpenID | — |
  | QQBOT_HOME_CHANNEL_NAME | 家庭频道的显示名称 | Home |
  | QQ_ALLOWED_USERS | 以逗号分隔的用户 OpenID 用于 DM 访问 | open (all users) |
  | QQ_GROUP_ALLOWED_USERS | 以逗号分隔的组 OpenID 用于组访问 | — |
  | QQ_ALLOW_ALL_USERS | 设置为 true 以允许所有私信 | false |
  | QQ_PORTAL_HOST | 覆盖 QQ 门户主机（沙盒路由设置为 sandbox.q.qq.com） | q.qq.com |
  | QQ_STT_API_KEY | 语音转文本提供商的 API 密钥 | — |
  | QQ_STT_BASE_URL | STT 提供商的基本 URL | https://open.bigmodel.cn/api/coding/paas/v4 |
  | QQ_STT_MODEL | STT 模型名称 | glm-asr |


# 高级配置
  如需进行更精细的控制，请将平台设置添加到 ~/.hermes/config.yaml 中：  
  ```bash
  platforms:
    qq:
      enabled: true
      extra:
        app_id: "your-app-id"
        client_secret: "your-secret"
        markdown_support: true       # enable QQ markdown (msg_type 2). Config-only; no env-var equivalent.
        dm_policy: "open"          # open | allowlist | disabled
        allow_from:
          - "user_openid_1"
        group_policy: "open"       # open | allowlist | disabled
        group_allow_from:
          - "group_openid_1"
        stt:
          provider: "zai"          # zai (GLM-ASR), openai (Whisper), etc.
          baseUrl: "https://open.bigmodel.cn/api/coding/paas/v4"
          apiKey: "your-stt-key"
          model: "glm-asr"
  ```

# 语音留言（STT）
  语音转录分两个阶段进行：  
  1. QQ 内置 ASR （免费，始终优先尝试）——QQ 在语音消息附件中提供 asr_refer_text ，该功能使用腾讯自家的语音识别技术。
  2. 配置的 STT 提供程序 （备用）— 如果 QQ 的 ASR 不返回文本，则适配器会调用与 OpenAI 兼容的 STT API：
     - Zhipu/GLM (zai): 默认提供程序，使用 glm-asr 模型
     - OpenAI Whisper ：设置 QQ_STT_BASE_URL 和 QQ_STT_MODEL
     - 任何与 OpenAI 兼容的 STT 端点
    
# 故障排除
  
## 机器人立即断开连接（快速断开连接）
  这通常意味着：  
  - 无效的应用 ID/密钥 ——请在 q.qq.com 上仔细检查您的凭据。
  - 权限不足 — 请确保机器人已启用所需的意图。
  - 仅限沙盒模式的机器人 ——如果机器人处于沙盒模式，则只能接收来自 QQ 沙盒测试频道的消息。

## 未转录的语音留言
  - 检查附件数据中是否存在 QQ 内置的 asr_refer_text
  - 如果使用自定义 STT 提供商，请确认 QQ_STT_API_KEY 设置正确。
  - 检查网关日志中是否存在 STT 错误消息

## 消息未送达
  - 请在 q.qq.com 上验证机器人的意图功能是否已启用。
  - 检查 QQ_ALLOWED_USERS 是否限制了私信访问权限。
  - 对于群组消息，请确保已 @提及机器人（群组策略可能需要将其列入允许列表）。
  - 检查 QQBOT_HOME_CHANNEL 是否已发送 cron/通知

## 连接错误
  - 确保已安装 aiohttp 和 httpx ： pip install aiohttp httpx
  - 检查与 api.sgroup.qq.com 和 WebSocket 网关的网络连接
  - 查看网关日志以获取详细的错误消息和重新连接行为
