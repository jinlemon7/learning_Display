# 快速设置
  配置消息平台最简单的方法是使用交互式向导：  
  ```bash
  hermes gateway setup        # Interactive setup for all messaging platforms
  ```
  - 这会引导您使用箭头键选择来配置每个平台，显示哪些平台已经配置好，并在完成后提供启动/重启网关的选项。

  网关命令  
  ```bash
  hermes gateway              # Run in foreground
  hermes gateway setup        # Configure messaging platforms interactively
  hermes gateway install      # Install as a user service (Linux) / launchd service (macOS)
  # 此命令需要前提：Hermes以ROOT用户安装
  sudo hermes gateway install --system   # Linux only: install a boot-time system service 
  hermes gateway start        # Start the default service
  hermes gateway stop         # Stop the default service
  hermes gateway status       # Check default service status
  # 此命令需要前提：Hermes以ROOT用户安装
  hermes gateway status --system         # Linux only: inspect the system service explicitly
  ```

# 会话管理

## 会话持久性
  会话会在消息之间持续存在，直到会话重置。agent会记住您的对话上下文。  

## 重置策略
  会话根据可配置策略重置：  
  | Policy | 默认 | 描述 |
  |--------|------|------|
  | Daily | 4:00 AM | 每天在特定时间重置 |
  | Idle | 1440 min | N 分钟无活动后重置 |
  | Both | (combined) | 无论哪个先触发，都先触发。 |

  在 ~/.hermes/gateway.json 中配置每个平台的覆盖设置：  
  ```bash
  {
    "reset_by_platform": {
      "telegram": { "mode": "idle", "idle_minutes": 240 },
      "discord": { "mode": "idle", "idle_minutes": 60 }
    }
  }
  ```

# 安全
  默认情况下，网关会拒绝所有不在允许列表中或未通过私信配对的用户。 对于拥有终端访问权限的机器人来说，这是安全的默认设置。  
  ```bash
  # Restrict to specific users (recommended):
  TELEGRAM_ALLOWED_USERS=123456789,987654321
  DISCORD_ALLOWED_USERS=123456789012345678
  SIGNAL_ALLOWED_USERS=+155****4567,+155****6543
  SMS_ALLOWED_USERS=+155****4567,+155****6543
  EMAIL_ALLOWED_USERS=trusted@example.com,colleague@work.com
  MATTERMOST_ALLOWED_USERS=3uo8dkh1p7g1mfk49ear5fzs5c
  MATRIX_ALLOWED_USERS=@alice:matrix.org
  DINGTALK_ALLOWED_USERS=user-id-1
  FEISHU_ALLOWED_USERS=ou_xxxxxxxx,ou_yyyyyyyy
  WECOM_ALLOWED_USERS=user-id-1,user-id-2
  WECOM_CALLBACK_ALLOWED_USERS=user-id-1,user-id-2
  TEAMS_ALLOWED_USERS=aad-object-id-1,aad-object-id-2
  
  # Or allow
  GATEWAY_ALLOWED_USERS=123456789,987654321
  
  # Or explicitly allow all users (NOT recommended for bots with terminal access):
  GATEWAY_ALLOW_ALL_USERS=true
  ```

# DM 配对（允许列表的替代方案）
  无需手动配置用户 ID，未知用户在私信机器人时会收到一次性配对码：  
  ```bash
  # The user sees: "Pairing code: XKGH5N7P"
  # You approve them with:
  hermes pairing approve telegram XKGH5N7P
  
  # Other pairing commands:
  hermes pairing list          # View pending + approved users
  hermes pairing revoke telegram 123456789  # Remove access
  ```
  - 配对码在 1 小时后过期，有速率限制，并使用加密随机性。

# 打断 agent
  在agent工作期间发送任何消息都会干扰其运行。主要行为：  
  - 正在进行的终端命令会被立即终止 （先发出 SIGTERM 信号，1 秒后发出 SIGKILL 信号）。
  - 工具调用被取消 ——只有当前正在执行的工具调用才会运行，其余的都会被跳过。
  - 多条消息合并 ——中断期间发送的消息会合并成一条提示信息。
  - /stop 命令 — 中断操作而不排队发送后续消息

# 队列 vs 中断 vs 转向（忙输入模式）
  默认情况下，向忙碌的 agent 发送消息会中断其工作。还有两种其他模式可用：  
  - queue — 后续消息等待并在当前任务完成后作为下一回合运行。
  - steer ——后续消息通过 /steer 注入到当前运行中，并在下一次工具调用后到达agent。不会中断，也不会进行新的转向。如果agent尚未启动，则回退到 queue 行为
    ```bash
    display:
      busy_input_mode: steer   # or queue, or interrupt (default)
      busy_ack_enabled: true   # set to false to suppress the ⚡/⏳/⏩ chat reply entirely
    ```

  当您首次通过任何平台向忙碌的 agent 发送消息时，Hermes 会在忙碌回复中添加一行提示信息，解释该提示的用途（ "💡 First-time tip — …" ）。此提示信息每次安装后只会触发一次 onboarding.seen.busy_input_prompt 下的一个标志会锁定此提示。删除该标志即可再次看到提示信息。  

  如果您觉得忙音提示音很吵——尤其是在语音输入或快速发送消息时——请将 display.busy_ack_enabled: false 。这样，您的输入仍然会像往常一样被排队/引导/中断，只有聊天回复会被静音。  

# 工具进度通知
  在 ~/.hermes/config.yaml 中控制显示多少工具活动：  
  ```bash
  display:
    tool_progress: all    # off | new | all | verbose
    tool_progress_command: false  # set to true to enable /verbose in messaging
  ```

  启用后，机器人会在运行过程中发送状态消息：  
  ```bash
  💻 `ls -la`...
  🔍 web_search...
  📄 web_extract...
  🐍 execute_code...
  ```

# 后台会话
  在单独的后台会话中运行提示，以便 agent 可以独立处理提示，同时您的主聊天保持响应：  
  ```bash
  /background 检查集群中的所有服务器，并报告任何宕机的服务器。
  ```

  Hermes立即确认：  
  ```bash
  🔄 Background task started: "Check all servers in the cluster..."
     Task ID: bg_143022_a1b2c3
  ```

## 工作原理
  每个 /background 提示符都会生成一个异步运行的独立agent实例 ：  
  - 独立会话 ——后台agent拥有独立的会话和对话历史记录。它不了解您当前的聊天内容，仅接收您提供的提示信息。
  - 配置相同 ——继承当前网关设置的模型、提供商、工具集、推理设置和提供商路由。
  - 非阻塞式 ——您的主聊天界面将保持完全交互。您可以在聊天运行时发送消息、运行其他命令或启动更多后台任务。
  - 结果交付 ——任务完成后，结果会发送到您发出命令的同一聊天或频道 ，并以“✅ 后台任务完成”为前缀。如果任务失败，您将看到“❌ 后台任务失败”以及错误信息。

## 后台进程通知
  当运行后台会话的 agent 使用 terminal(background=true) 启动长时间运行的进程（服务器、构建等）时，网关可以将状态更新推送至您的聊天窗口。您可以通过 ~/.hermes/config.yaml 中的 display.background_process_notifications 来控制此功能：  
  ```bash
  display:
    background_process_notifications: all    # all | result | error | off
  ```

  | Mode（模式） | What you receive（您将收到） |
  |--------------|----------------------------|
  | all |  运行输出更新和最终完成消息（默认） |
  | result  | 仅显示最终完成消息（无论退出代码如何） |
  | error | 仅当退出代码非零时才显示最终消息 |
  | off  | 完全没有进程监视器消息 |

  您也可以通过环境变量进行设置：  
  ```bash
  HERMES_BACKGROUND_NOTIFICATIONS=result
  ```

## 用例
  - 服务器监控 — “/background 检查所有服务的运行状况，如有任何服务出现故障，请通知我”
  - 长时间构建 ——“/background 构建并部署暂存环境”，同时您可以继续聊天
  - 研究任务 ——“/背景 研究竞争对手的定价并汇总成表格”
  - 文件操作 — "/background 将 ~/Downloads 文件夹中的照片按日期整理到文件夹中"

  > [!IMPORTANT]
  > 即时通讯平台上的后台任务是“即发即弃”的——您无需等待或查看。任务完成后，结果会自动显示在同一聊天窗口中。

# 服务管理

##  Linux（systemd）
  ```bash
  hermes gateway install               # Install as user service
  hermes gateway start                 # Start the service
  hermes gateway stop                  # Stop the service
  hermes gateway status                # Check status
  journalctl --user -u hermes-gateway -f  # View logs
  
  # Enable lingering (keeps running after logout)
  sudo loginctl enable-linger $USER

  # 以下命令需要前提：Hermes以ROOT用户安装
  # Or install a boot-time system service that still runs as your user
  sudo hermes gateway install --system
  sudo hermes gateway start --system
  sudo hermes gateway status --system
  journalctl -u hermes-gateway -f
  ```

  在笔记本电脑和开发机上使用用户服务。在 VPS 或无头主机上使用系统服务，这些主机应该在启动时自动恢复，而无需依赖 systemd 的延迟。  

  除非您确实需要，否则请避免同时安装用户网关单元和系统网关单元。如果 Hermes 检测到两者同时存在，将会发出警告，因为启动/停止/状态行为会变得不明确。  

  > [!WARNING]
  > 如果在同一台机器上运行多个 Hermes 安装（使用不同的 HERMES_HOME 目录），每个安装都会获得自己的 systemd 服务名称。默认的 ~/.hermes 使用 hermes-gateway ；其他安装使用 hermes-gateway-<hash> hermes gateway 命令会自动根据您当前的 HERMES_HOME 指定正确的服务。
