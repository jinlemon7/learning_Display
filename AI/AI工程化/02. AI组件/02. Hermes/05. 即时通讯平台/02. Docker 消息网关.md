用 Docker Compose 启动 Hermes 和 hermes-dashboard 后。  

进入容器终端：  
```bash
docker exec -it hermes bash
```

退出容器终端:  
```bash
exit
```

# 配置消息平台
  配置config.yaml:  
  ```bash
  gateway:
    platforms:
      qq:
        enabled: true
        extra:
          app_id: "你的QQ机器人AppID"
          client_secret: "你的QQ机器人AppSecret"
          markdown_support: true          # 是否启用QQ Markdown消息
          dm_policy: "allowlist"          # 私聊策略：open | allowlist | disabled
          allow_from:                     # 私聊白名单（仅 dm_policy=allowlist 时生效）
            - "用户openid_1"
            - "用户openid_2"
          group_policy: "allowlist"       # 群聊策略：open | allowlist | disabled
          group_allow_from:               # 群聊白名单（仅 group_policy=allowlist 时生效）
            - "群openid_1"
            - "群openid_2"
          stt:                            # 可选，语音识别配置
            provider: "zai"
            baseUrl: "https://open.bigmodel.cn/api/coding/paas/v4"
            apiKey: "你的stt-key"
            model: "glm-asr"
  ```

  字段说明：  
  | 字段 | 说明 |
  |------|------|
  | enabled | 是否启用 QQ 平台，设为 true |
  | extra.app_id | QQ 机器人的 AppID（来自 QQ 开放平台） |
  | extra.client_secret | QQ 机器人的 AppSecret（客户端密钥） |
  | extra.markdown_support | 是否开启 QQ Markdown 消息类型（true 会使用 msg_type=2） |
  | dm_policy | 私聊（Direct Message）策略<br>open – 允许所有用户私聊机器人<br>allowlist – 仅白名单中的用户可私聊（名单见 allow_from）<br>disabled – 完全禁止私聊 |
  | allow_from | 私聊白名单，填入用户的 OpenID（非 QQ 号），仅在 dm_policy = allowlist 时生效 |
  | group_policy | 群聊策略<br>open – 允许所有群<br>allowlist – 仅白名单中的群可交互（名单见 group_allow_from）<br>disabled – 完全禁止群聊 |
  | group_allow_from | 群聊白名单，填入群的 OpenID，仅在 group_policy = allowlist 时生效 |
  | stt | （可选）语音识别服务配置，用于处理 QQ 语音消息 |
  | stt.provider | 语音识别供应商，如 zai（GLM-ASR）、openai（Whisper） |
  | stt.baseUrl | 语音识别 API 地址 |
  | stt.apiKey | 语音识别 API 密钥 |
  | stt.model | 使用的模型名称，例如 glm-asr |

# 配置后操作
  删除 .env 中的 QQ_ALLOWED_USERS（如果有）  
  
  重启 Gateway：  
  ```bash
  docker compose restart gateway
  ```

# 启动 gateway
  进入容器终端  

  ```bash
  hermes gateway run
  ```
