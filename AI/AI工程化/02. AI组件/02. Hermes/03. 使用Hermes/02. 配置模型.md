# 配置模型
  Hermes 使用两种类型的模型插槽：  
  - 主模型 ——智能体思考的依据。每条用户消息、每一次工具调用循环、每一次流式响应都会经过这个模型。
  - 辅助模型 —— agent 程序卸载的小型辅助任务。包括上下文压缩、视觉（图像分析）、网页摘要、会话搜索、审批评分、MCP 工具路由、会话标题生成和技能搜索。每个模型都有自己的插槽，并且可以独立进行覆盖

# 模型页面
  打开控制面板，点击侧边栏中的 “模型” 。您会看到两个部分：  
  - 模型设置 — 顶部面板，用于将模型分配到插槽。
  - 使用情况分析 ——排名卡片显示在选定时间段内运行过会话的每个模型，包括令牌计数、成本和功能徽章。

  <img width="1280" height="720" alt="image" src="https://github.com/user-attachments/assets/21a0c5b6-a29d-480e-98d8-32cf6fd33a9a" />  

  顶部的卡片是 “模型设置” 面板。主行始终显示代理将在新会话中启动的模型。单击 “更改” 打开选择器。  

# 设置主模型
  点击主模型行上的 “更改” 按钮  
  <img width="1280" height="720" alt="image" src="https://github.com/user-attachments/assets/e20192d6-1716-4263-9448-2748af75f710" />  

  选择器有两列：  
  - 左侧 — 已认证的提供商。此处仅显示您已设置的提供商（已设置 API 密钥、已通过 OAuth 认证或已定义为自定义端点）。如果缺少某个提供商，请前往 “密钥”页面添加其凭据。
  - 右侧 ——这是为所选提供商精心挑选的模型列表。这些是 Hermes 为该提供商推荐的代理模型，而不是原始的 /models 转储（在 OpenRouter 上，该转储包含 400 多个模型，包括 TTS、图像生成器和重排序器）。

  在筛选框中输入内容，按提供商名称、别名或型号 ID 进行筛选。  

  选择一个模型，点击 “切换” ，Hermes 会将该模型写入 ~/.hermes/config.yaml 文件中的 model 部分。 此操作仅适用于新会话 ——任何已打开的聊天标签页仍会运行其初始模型。要热切换当前聊天，请在聊天中使用 /model 斜杠命令。  

# 设置辅助模型
  点击 “显示辅助任务” 以显示八个任务槽：  
  <img width="1280" height="720" alt="image" src="https://github.com/user-attachments/assets/cd338129-a89d-4b64-bb01-a91223a84245" />  

  所有辅助任务默认设置为 auto ——这意味着 Hermes 也会使用您的主模型来完成该任务。如果您想为某个辅助任务选择更便宜或更快捷的模型，可以手动覆盖特定任务的设置。  

## 常用覆盖模式
  | Task | When to override |
  |------|------------------|
  | Title Gen | 几乎总是如此。0.10 美元/兆的闪存型号可以写入会话标题以及 Opus 格式。OpenRouter 上的默认配置将其设置为 google/gemini-3-flash-preview 。 |
  | Vision | 当你的主要模型是一个没有视觉功能的编码模型（例如 Kimi、DeepSeek）时，请将其指向 google/gemini-2.5-flash 或 gpt-4o-mini 。 |
  | Compression | 当你在 Opus/M2.7 上消耗大量推理代币仅仅为了概括上下文时，一个快速聊天模型就能以 1/50 的成本完成同样的工作。 |
  | Session Search | 当 召回查询 分散进行 时——默认最大并发数为 3。低成本模型可使账单可预测。 |
  | Approval | 对于 approval_mode: smart ，系统会使用快速且成本低廉的模型（例如 Haiku、Flash 或 GPT-5-mini）来决定是否自动批准低风险命令。在这里使用昂贵的模型是浪费。 |
  | Web Extract | 当你大量使用 web_extract 时，逻辑与压缩相同——摘要不需要推理。 |
  | Skills Hub | hermes skills search 使用此功能。通常在 auto 下运行良好。 |
  | MCP | MCP 工具路由。通常在 auto 下运行良好。 |

## 每个任务的覆盖
  点击任意辅助行上的 “更改” 按钮。打开相同的选择器，操作相同——选择提供商 + 型号，然后点击“切换”按钮。该行将更新为显示 provider · model 而不是 auto (use main model) 。  

## 全部重置为自动
  如果你过度调整了设置，想要重新开始，请点击辅助设置顶部的 “全部重置为自动” 。所有槽位都会恢复使用你的主模型  

# “用作”快捷方式
  页面上的每个模型卡片都有一个 “用作” 下拉菜单。这是快捷方式——选择您在分析中看到的模型，点击 “用作” ，然后一键将其分配给主槽位或任何特定的辅助任务：  

  <img width="1280" height="720" alt="image" src="https://github.com/user-attachments/assets/7b47d9ac-a69d-4ccc-9592-6383c0965a93" />  

  下拉菜单包：  
  - 主模型 ——与点击主行上的“更改”按钮相同。
  - 所有辅助任务 ——将此型号同时分配给所有 8 个辅助插槽。当您只想用廉价闪存型号完成所有辅助任务时非常有用。
  - 单个任务选项 ——视觉、网络提取、压缩等。每个任务当前分配的模型标记为 current 。

  当卡片当前被分配给某个任务时，卡片上会标有 main 或 aux · <task> 的标记——这样你就可以一目了然地看到哪些历史模型连接到了哪些位置。  

# config.yaml 文件会写入什么内容？
  当您通过控制面板保存时，Hermes 会将配置写入 ~/.hermes/config.yaml 文件：  
  Main model:  主要型号：  
  ```bash
  model:
    provider: openrouter
    default: anthropic/claude-opus-4.7
    base_url: ''        # cleared on provider switch
    api_mode: chat_completions
  ```

  辅助功能覆盖（例如——gemini-flash的vision）：  
  ```bash
  auxiliary:
    vision:
      provider: openrouter
      model: google/gemini-2.5-flash
      base_url: ''
      api_key: ''
      timeout: 120
      extra_body: {}
      download_timeout: 30
  ```

  辅助功能自动开启（默认）：  
  ```bash
  auxiliary:
    compression:
      provider: auto
      model: ''
      base_url: ''
      # ... other fields unchanged
  ```
  - provider: auto with model: '' 告诉 Hermes 使用主模型来完成该任务。  

# 何时生效？
  - CLI （ hermes chat ）：下一次 hermes chat 调用。
  - 网关 （Telegram、Discord、Slack 等）：用于下一个新会话。现有会话将保留其模型。如果您想强制所有会话应用此更改，请重启网关（ hermes gateway restart ）。
  - 仪表盘聊天标签页 （ /chat ）：下一个新的 PTY。当前打开的聊天窗口会保留其模型——在聊天窗口内使用 /model 可以进行热切换。
  
  更改不会使运行会话中的提示符缓存失效。这是有意为之：在会话中切换主模型需要重置缓存（系统提示符包含特定于模型的内容），而我们只在聊天窗口中使用显式的 /model 命令来执行此操作。  

# 故障排除

## 选择器中“无已验证的提供商”  
  Hermes 只会在提供商拥有有效凭证时才会将其列出。请查看侧边栏中的 “密钥”部分 ——您应该会看到以下选项之一：API 密钥、成功的 OAuth 凭证或自定义端点 URL。如果您需要的提供商不在列表中，请运行 hermes setup 进行配置，或者前往 “密钥”部分添加环境变量。  

## 我的运行聊天中主模型没有改变
  这是预期行为。控制面板会写入 config.yaml ，新会话会读取该文件。当前打开的聊天窗口是一个实时agent进程——它会保留创建时使用的模型。在聊天窗口中使用 /model <name> 可以快速切换当前会话。  

## 辅助覆盖“未生效”
  三项需要检查的内容：  
  - 你是不是新建了一个会话？ 已有的聊天记录不会重新读取配置。
  - provider 是否设置为除 auto 以外的其他值？ 如果该字段显示为 auto ，则任务仍在使用您的主模型。单击 “更改” 并选择一个实际的提供程序。
  - 提供者是否已通过身份验证？ 如果您将 minimax 分配给某个任务，但没有 MiniMax API 密钥，则该任务将回退到 openrouter 默认值，并在 agent.log 中记录警告。

## 我选好了一款型号，但 Hermes 却给我换了供应商
  在 OpenRouter（或任何聚合器）中，裸模型名称首先在聚合器内部解析。因此，在 OpenRouter 中， claude-sonnet-4 会解析为 anthropic/claude-sonnet-4.6 ，并保留在您的 OpenRouter 身份验证中。但是，如果您在 Anthropic 原生身份验证中输入 claude-sonnet-4 ，它将保持为 claude-sonnet-4-6 。如果您看到意外的提供商切换，请检查当前提供商是否符合预期——选择器始终在对话框顶部显示当前主提供商。  

# 其他方法

## CLI 斜杠命令  
  在任何 hermes chat 会话中：  
  ```bash
  /model gpt-5.4 --provider openrouter             # session-only
  /model gpt-5.4 --provider openrouter --global    # also persists to config.yaml
  ```
  `--global` 作用与仪表板上的 “更改” 按钮相同，此外，它还会就地切换正在运行的会话。  

## 自定义别名
  为常用模型定义自己的简短名称，然后在 CLI 或任何消息平台中使用 /model <alias> ：  
  ```bash
  # ~/.hermes/config.yaml
  model_aliases:
    fav:
      model: claude-sonnet-4.6
      provider: anthropic
    grok:
      model: grok-4
      provider: x-ai
  ```

  或者在 shell（简写形式， provider/model ）设置：  
  ```bash
  hermes config set model.aliases.fav anthropic/claude-opus-4.6
  hermes config set model.aliases.grok x-ai/grok-4
  ```

  然后，在聊天框中输入 /model fav 或 /model grok 。用户别名会覆盖内置的短名称（ sonnet 、 kimi 、 opus 等）。  

## hermes model 子命令
  ```bash
  hermes model list                   # list authenticated providers + models
  hermes model set anthropic/claude-opus-4.7 --provider openrouter
  ```

## 直接配置编辑
  编辑 ~/.hermes/config.yaml 并重启读取该文件的服务。  

## REST API
  该仪表盘使用三个端点，便于编写脚本：  
  ```bash
  # List authenticated providers + curated model lists
  curl -H "X-Hermes-Session-Token: $TOKEN" http://localhost:PORT/api/model/options
  
  # Read current main + auxiliary assignments
  curl -H "X-Hermes-Session-Token: $TOKEN" http://localhost:PORT/api/model/auxiliary
  
  # Set the main model
  curl -X POST -H "Content-Type: application/json" -H "X-Hermes-Session-Token: $TOKEN" \
    -d '{"scope":"main","provider":"openrouter","model":"anthropic/claude-opus-4.7"}' \
    http://localhost:PORT/api/model/set
  
  # Override a single auxiliary task
  curl -X POST -H "Content-Type: application/json" -H "X-Hermes-Session-Token: $TOKEN" \
    -d '{"scope":"auxiliary","task":"vision","provider":"openrouter","model":"google/gemini-2.5-flash"}' \
    http://localhost:PORT/api/model/set
  
  # Assign one model to every auxiliary task
  curl -X POST -H "Content-Type: application/json" -H "X-Hermes-Session-Token: $TOKEN" \
    -d '{"scope":"auxiliary","task":"","provider":"openrouter","model":"google/gemini-2.5-flash"}' \
    http://localhost:PORT/api/model/set
  
  # Reset all auxiliary tasks to auto
  curl -X POST -H "Content-Type: application/json" -H "X-Hermes-Session-Token: $TOKEN" \
    -d '{"scope":"auxiliary","task":"__reset__","provider":"","model":""}' \
    http://localhost:PORT/api/model/set
  ```

  会话令牌会在启动时注入到仪表盘的 HTML 中，并在每次服务器重启时轮换。如果您正在对正在运行的仪表盘编写脚本，请从浏览器开发者工具中获取它（ window.__HERMES_SESSION_TOKEN__ ）。
