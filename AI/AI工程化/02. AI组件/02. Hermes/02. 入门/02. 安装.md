# 先决条件
  - uv （快速 Python 包管理器）
  - Python 3.11 （通过 uv 安装，无需 sudo）
  - Node.js v22 （用于浏览器自动化和 WhatsApp 桥接）
  - ripgrep （快速文件搜索）
  - ffmpeg （用于 TTS 的音频格式转换）

  > [!WARNING]
  > 您无需手动安装 Python、Node.js、ripgrep 或 ffmpeg。安装程序会自动检测缺失的组件并为您安装。只需确保 git 可用（ git --version ）。

# 安装程序的功能
  安装程序会自动处理所有事项——所有依赖项（Python、Node.js、ripgrep、ffmpeg）、仓库克隆、虚拟环境、全局 hermes 命令设置以及 LLM 提供程序配置。安装完成后，即可开始聊天。  

## 安装布局
  安装程序将文件放置的位置取决于您是以普通用户身份还是以 root 用户身份进行安装：  
  | 安装程序 | 代码存在于 | hermes 二进制 | 数据目录 |
  |----------|------------|---------------|----------|
  | 按用户（正常） | ~/.hermes/hermes-agent/ | ~/.local/bin/hermes （符号链接） | ~/.hermes/ |
  | 以 root 用户身份运行（ sudo curl … | sudo bash ） | /usr/local/lib/hermes-agent/ | /usr/local/bin/hermes | /root/.hermes/ （或 $HERMES_HOME） |

  根模式 FHS 布局 （ /usr/local/lib/… ， /usr/local/bin/hermes ）与 Linux 上其他系统级开发工具的安装位置一致。这对于共享机器部署非常有用，因为一个系统安装应该服务于所有用户。每个用户的配置（身份验证、技能、会话）仍然位于每个用户的 ~/.hermes/ 下或显式指定的 HERMES_HOME 下。
