# 安装 Hermes Agent
  运行一行安装程序：  
  ```bash
  # Linux / macOS / WSL2 / Android (Termux)
  curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
  ```

  完成后，重载shell：  
  ```bash
  source ~/.bashrc   # or source ~/.zshrc
  ```

# 最快路径
  选择符合您目标的行：  
  | 目标 | 先做这件事 | 然后这样做 |  
  |------|------------|------------|  
  | 我只想让 Hermes 在我的机器上运行。 | 运行 `hermes setup` | 运行一次真实的聊天测试，并验证其响应情况。 |  
  | 我已经知道我的服务提供商了。 | 运行 `hermes model` | 保存配置，然后开始聊天。 |  
  | 我想要一个机器人或始终在线的设置。 | 在 CLI 成功设置 Hermes 后，运行 `hermes gateway setup` | 连接 Telegram、Discord、Slack 或其他平台。 |  
  | 我想要一个本地或自托管的模型。 | 运行 `hermes model` 并选择自定义端点 | 验证端点、模型名称和上下文长度。 |  
  | 我想要多提供商fallback。 | 首先运行 `hermes model` | 只有在基本聊天功能正常运行后，才添加路由和fallback机制。 |  

  经验法则： 如果 Hermes 无法完成正常的聊天，就不要添加更多功能。先确保能够流畅地进行对话，然后再逐步添加网关、定时任务、技能、语音或路由等功能。  


# 选择服务提供商
  最重要的设置步骤。使用 hermes model 交互式地完成选择过程：  
  ```bash
  hermes model
  ```

  良好的默认值：  
  | Provider | 它是什么 | 如何设置 |
  |----------|----------|----------|
  | Nous Portal | 基于订阅的零配置 | 通过 `hermes model` 进行 OAuth 登录 |
  | OpenAI Codex | ChatGPT OAuth，使用 Codex 模型 | 通过 `hermes model` 进行设备代码认证 |
  | Anthropic | 直接使用 Claude 模型 — 最高套餐 + 额外使用额度（OAuth），或按令牌付费的 API 密钥 | 通过 `hermes model` 进行 OAuth 登录（需要最高套餐和额外积分），或使用 Anthropic API 密钥 |
  | OpenRouter | 跨多种模型的多提供商路由 | 输入您的 API 密钥 |
  | Z.AI | GLM / 智谱托管模型 | 设置 `GLM_API_KEY` / `ZAI_API_KEY` |
  | Kimi / Moonshot | Moonshot 托管的编码和聊天模型 | 设置 `KIMI_API_KEY` |
  | Kimi / Moonshot China | 中国区域 Moonshot 端点 | 设置 `KIMI_CN_API_KEY` |
  | Arcee AI | Trinity 模型 | 设置 `ARCEEAI_API_KEY` |
  | GMI Cloud | 多模型直接 API | 设置 `GMI_API_KEY` |
  | MiniMax (OAuth) | 通过浏览器 OAuth 使用 MiniMax-M2.7 — 无需 API 密钥 | 运行 `hermes model` 并选择 MiniMax (OAuth) |
  | MiniMax | 国际 MiniMax 端点 | 设置 `MINIMAX_API_KEY` |
  | MiniMax China | 中国区 MiniMax 端点 | 设置 `MINIMAX_CN_API_KEY` |
  | Alibaba Cloud | 通过 DashScope 使用通义千问模型 | 设置 `DASHSCOPE_API_KEY` |
  | Hugging Face | 通过统一路由器支持 20 多种开放模型（Qwen、DeepSeek、Kimi 等） | 设置 `HF_TOKEN` |
  | AWS Bedrock | 通过原生 Converse API 使用 Claude、Nova、Llama、DeepSeek | IAM 角色或运行 `aws configure`（参考指南） |
  | Kilo Code | KiloCode 托管模型 | 设置 `KILOCODE_API_KEY` |
  | OpenCode Zen | 按需付费访问精选模型 | 设置 `OPENCODE_ZEN_API_KEY` |
  | OpenCode Go | 开放模型订阅，每月 10 美元 | 设置 `OPENCODE_GO_API_KEY` |
  | DeepSeek | 直接访问 DeepSeek API | 设置 `DEEPSEEK_API_KEY` |
  | NVIDIA NIM | 通过 build.nvidia.com 或本地 NIM 使用 Nemotron 模型 | 设置 `NVIDIA_API_KEY`（可选 `NVIDIA_BASE_URL`） |
  | GitHub Copilot | GitHub Copilot 订阅（GPT-5.x、Claude、Gemini 等） | 通过 `hermes model` 进行 OAuth 认证，或设置 `COPILOT_GITHUB_TOKEN` / `GH_TOKEN` |
  | GitHub Copilot ACP | Copilot ACP 代理后端（启动本地 copilot CLI） | 运行 `hermes model`（需要 `copilot` CLI 并已执行 `copilot login`） |
  | Vercel AI Gateway | Vercel AI 网关路由 | 设置 `AI_GATEWAY_API_KEY` |
  | Custom Endpoint | VLLM、SGLang、Ollama 或任何与 OpenAI 兼容的 API | 设置基础 URL 和 API 密钥 |

  对于大多数首次使用的用户：选择一个提供商，除非您知道为什么要更改默认设置，否则请接受默认设置。  

  > [!WARNING]
  > Hermes Agent 要求模型至少包含 64,000 个上下文标记 。窗口较小的模型无法为多步骤工具调用工作流程维护足够的内存，因此会在启动时被拒绝。大多数托管模型（例如 Claude、GPT、Gemini、Qwen 和 DeepSeek）都能轻松满足此要求。如果您运行的是本地模型，请将其上下文大小设置为至少 64K

# 设置如何存储
  Hermes 将密钥与普通配置分开：  
  - secret 和 token → ~/.hermes/.env
  - 非机密设置 → ~/.hermes/config.yaml

  设置值的最简单方法是通过命令行界面 (CLI)：  
  ```bash
  hermes config set model anthropic/claude-opus-4.6
  hermes config set terminal.backend docker
  hermes config set OPENROUTER_API_KEY sk-or-...
  ```
  - 正确的值会自动写入正确的文件。

# 进行你的第一次聊天
  ```bash
  hermes            # classic CLI
  hermes --tui      # modern TUI (recommended)
  ```

  您会看到一个欢迎横幅，其中包含您的模型、可用工具和技能。请使用具体且易于验证的提示：  

  > [!IMPORTANT]
  > Hermes 自带两个终端界面：经典的 prompt_toolkit CLI 和带有模态覆盖层、鼠标选择和非阻塞输入的新型 TUI 。两者共享相同的会话、斜杠命令和配置——分别使用 hermes vs hermes --tui 进行尝试。


  验证示例：  
  ```text
  请用 5 个要点概括这个仓库，并告诉我它的主要入口点是什么。
  请检查我的当前目录，并告诉我哪个文件看起来像是主项目文件。
  请帮我为这个代码库建立一个简洁的GitHub PR工作流程。
  ```

  成功是什么样子：  
  - 横幅显示您选择的型号/提供商。
  - Hermes毫无差错地回复
  - 必要时可以使用工具（终端、文件读取、网络搜索）

  如果像这样，你就已经度过了最难的部分。  

# 验证会话是否正常工作
  在继续之前，请确保resume功能正常：  
  ```bash
  hermes --continue    # Resume the most recent session
  hermes -c            # Short form
  ```

  这样应该能让你回到刚才的会话。如果不行，请检查你是否在同一个用户配置文件中，以及会话是否已保存。这一点在你以后管理多个设置或机器时很重要。  

# 试用主要功能
  
## 使用终端  
  ```text
  我的磁盘使用情况如何？显示占用空间最大的 5 个目录。
  ```

  该代理程序会代表您运行终端命令并显示结果。  

## 斜杠命令
  输入 / 可查看所有命令的自动完成下拉菜单：  
  | Command | 它的作用 |
  |---------|----------|
  | /help | 显示所有可用命令 |
  | /tools | 列出可用工具 |
  | /model | 交互式切换模型 |
  | /personality pirate | 尝试培养一种有趣的个性 |
  | /save | 保存对话 |

## 多行输入
  按 Alt+Enter 或 Ctrl+J 添加新行。非常适合粘贴代码或编写详细提示。  

## 打断 agent
  如果代理处理时间过长，请输入新消息并按 Enter 键——这将中断当前任务并切换到您的新指令。Ctrl+C 也有效。  

# 添加下一层

## 机器人或共享助手
  ```bash
  hermes gateway setup    # Interactive platform configuration
  ```
  连接 Telegram 、 Discord 、 Slack 、 WhatsApp 、 Signal 、 电子邮件 、 Home Assistant 或 Microsoft Teams 。  

## 自动化和工具
  - hermes tools — 按平台调整工具访问权限
  - hermes skills ——浏览和安装可重用的工作流程
  - 定时任务（Cron）——仅在您的机器人或命令行界面（CLI）设置稳定后使用

## 沙盒终端
  为了安全起见，请在 Docker 容器或远程服务器上运行代理：  
  ```bash
  hermes config set terminal.backend docker    # Docker isolation
  hermes config set terminal.backend ssh       # Remote server
  ```

## 语音模式
  ```bash
  pip install "hermes-agent[voice]"
  # Includes faster-whisper for free local speech-to-text
  ```
  然后在命令行界面 (CLI) 中输入： /voice on 。按 Ctrl+B 开始录音  

## 技能
  ```bash
  hermes skills search kubernetes
  hermes skills install openai/skills/k8s
  ```

  或者在聊天会话中使用 /skills 。  

## MCP 服务
  ```bash
  # Add to ~/.hermes/config.yaml
  mcp_servers:
    github:
      command: npx
      args: ["-y", "@modelcontextprotocol/server-github"]
      env:
        GITHUB_PERSONAL_ACCESS_TOKEN: "ghp_xxx"
  ```

## 编辑器集成（ACP）
  ```bash
  pip install -e '.[acp]'
  hermes acp
  ```

# 常见故障模式
  以下是一些最浪费时间的问题：  
  | 症状 | 可能原因 | 修复 |
  |------|----------|------|
  | Hermes打开了信箱，但只给了空洞或不完整的回复。 | 提供商身份验证或型号选择错误 | 再次运行 `hermes model` 并确认提供商、模型和身份验证。 |
  | 自定义端点“有效”，但返回垃圾数据。 | 基础 URL、模型名称错误，或者实际上与 OpenAI 不兼容。 | 首先在另一个客户端中验证端点。 |
  | 网关已启动，但无人可以向其发送消息。 | 机器人token、白名单或平台设置不完整 | 重新运行 `hermes gateway setup` 并检查 `hermes gateway status`。 |
  | `hermes --continue` 找不到旧会话 | 切换配置文件或会话未保存 | 查看 `hermes sessions list` 并确认您登录的是正确的个人资料。 |
  | 模型不可用或出现异常的备用行为 | 提供商路由或fallback设置过于激进。 | 保持路由关闭状态，直到基础服务提供商稳定为止。 |
  | `hermes doctor` 标志配置问题 | 配置值缺失或已过期。 | 修复配置，在添加功能之前重新测试一下简单的聊天功能。 |

# 恢复工具包
  当感觉不对劲时，请按以下顺序操作：  
  - hermes doctor
  - hermes model
  - hermes setup
  - hermes sessions list
  - hermes --continue
  - hermes gateway statu

  这个方法能让你迅速从“糟糕的情绪”恢复到熟悉的状态。
