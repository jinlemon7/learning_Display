# 顶层部署层次图
  ```text
  ~/hl-engineer/                    # ★ HL‑Engineer 系统根目录，所有相关文件集中于此
  ├── bin/                           # 自定义可执行脚本（启动/停止/同步）
  ├── config/                        # 组件配置文件（从默认位置链接或拷贝而来）
  │   ├── openclaw/                    # OpenClaw 配置（等效 ~/.openclaw）
  │   └── hermes/                      # Hermes 配置（等效 ~/.hermes）
  ├── data/                          # 运行时产生的数据（技能、规则、进化产物）
  │   ├── openclaw/                    # OpenClaw 的技能、规则存放处
  │   │   ├── rules/                     # 四大原则等 rules
  │   │   └── skills/                    # 本地技能库
  │   ├── hermes/                      # Hermes 沉淀的技能、历史对话
  │   │   └── skills/                    # 进化出的 agentskills.io 标准技能
  │   └── acp/                         # ACP 桥接状态文件、任务队列等
  ├── logs/                          # 所有日志集中存放
  │   ├── openclaw.log                 # OpenClaw 后台日志
  │   ├── hermes.log                   # Hermes 后台日志
  │   └── acp-bridge.log               # ACP 桥接服务日志
  ├── projects/                      # 测试/生产 Git 仓库（Hermes 的工作区）
  │   └── test-hl-engineer/            # 示例测试仓库（后续阶段二会用到）
  ├── acp-bridge/                    # ACP 桥接服务代码（阶段二创建，但目录先预留）
  ├── scripts/                       # 辅助脚本（比如阶段三的技能同步脚本）
  └── README.md                      # 整体说明
  ```

  重要文件和目录详细注释：  
  - `~/hl-engineer/bin/`：存放所有自制管理脚本，比如 start-all.sh、stop-all.sh、sync-skills.sh，赋予系统级的可执行权限，便于日常运维。
  - config/openclaw/：计划用软链接替换默认的 ~/.openclaw，这样任何对 OpenClaw 配置的修改都会发生在统一目录中，备份时只备份 ~/hl-engineer 即可。
  - data/openclaw/rules/：存放阶段二要注入的《四大原则》，文件名例如 hl-engineer-principles.md。
  - data/openclaw/skills/：存放 OpenClaw 本地技能，阶段二会创建 delegate-to-hermes.md，阶段三会通过 data/hermes/skills/ 同步进化技能过来。
  - data/hermes/skills/：Hermes 完成任务后自动生成的 agentskills.io 标准技能文件，由定时脚本 sync-skills.sh 单向同步到 data/openclaw/skills/。
  - logs/：统一收集所有日志，方便调试整个系统链路。
  - projects/：Hermes 进行代码生成和提交的 Git 仓库根目录，避免与个人其他项目混淆。

# 清理旧组件
  请按顺序执行以下命令，确保环境干净。  

## 停止并卸载守护进程
  OpenClaw 会在系统中注册持久化后台服务（systemd 用户服务），若残留，重启后会自动复活。  
  先尝试优雅停止：  
  ```bash
  openclaw gateway stop
  openclaw gateway uninstall
  ```

  若上述命令无效，手动清理 systemd 服务：  
  ```bash
  systemctl --user stop openclaw-gateway.service
  systemctl --user disable openclaw-gateway.service
  rm -f ~/.config/systemd/user/openclaw-gateway.service
  systemctl --user daemon-reload
  ```

## 第二步：执行官方卸载命令（核心）
  ```bash
  openclaw uninstall --all --yes --non-interactive
  ```

  这将删除服务、配置、数据库等全部本地组件。  
  若 CLI 命令已不可用，请直接跳至手动删除阶段。  

## 手动删除配置目录
  ```bash
  rm -rf ~/.openclaw
  ```
  - 这是最关键的一步，该目录包含凭证、API 密钥、会话记忆和 Skills，确保无残留数据遗留。

## 卸载 CLI 工具 
  若通过 npm 安装，请执行：  
  ```bash
  npm uninstall -g openclaw
  ```

  验证是否已移除：  
  ```bash
  which openclaw          # 应无输出
  openclaw --version      # 应报 "command not found"
  ```
